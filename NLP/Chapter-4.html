Preliminary Concepts (Read Once)

You type "best pizza Pune" into Google → it returns the most relevant web pages from billions. That's INFORMATION RETRIEVAL — the science of finding documents matching a user's information need. NLP makes IR smarter — understanding queries, recognizing entities, handling synonyms, working across languages.

What is Information Retrieval (IR)?

IR is the process of obtaining relevant information (documents, web pages, images) from a large collection in response to a user's query. The core challenge: matching user's INTENT with relevant content.

Main components of an IR system

  • Document collection (corpus).
  • Indexing — preprocess documents into searchable structures (inverted index, embeddings).
  • Query processing — tokenize, normalize, expand the user query.
  • Ranking — compute relevance scores and order documents.
  • Evaluation — precision, recall, F1, MAP, NDCG.

Q1 — Vector Space Model (VSM)

Asked 5 times(May'23, Nov'23, May'24, May'25, Nov'25 — usually 6–9 marks)
How this question appeared in past papers:

(May'23, Nov'23, 6 marks) Explain the concept of the Vector Space Model and describe how it is used in Information Retrieval.

(May'24, 9 marks) Describe VSM. How does VSM represent documents and queries, and how are similarities calculated? Discuss strengths and weaknesses.

(May'25, 8 marks) Describe the Vector Space Model (VSM) for information retrieval.

(Nov'25, 9 marks) Explain VSM in IR. How are documents and queries represented, how is relevance computed?

Answer
Imagine every document is a POINT in a giant space with one axis for each word in the vocabulary. A document about cricket lies near other cricket documents. A query is also placed in this space; the documents CLOSEST to the query are the most relevant.

1. Definition

Vector Space Model (VSM) represents documents and queries as vectors in a high-dimensional space. Each dimension corresponds to a term in the vocabulary. Relevance is computed by measuring the similarity (typically cosine) between the query vector and document vectors.

2. How documents are represented

  • Vocabulary = all unique terms across the corpus.
  • Each document = vector with one dimension per term.
  • Each component = weight (TF, TF-IDF, binary, etc.) of that term in the document.
  • If V = 10,000 terms, each document is a 10,000-dimensional vector.

3. How queries are represented

  • The query is treated like a short document.
  • Converted into the same vector space using the same weighting scheme.

4. Cosine similarity (computing relevance)

similarity(q, d) = (q · d) / (||q|| · ||d||) = cos θ
  • q · d = dot product of query and document vectors.
  • ||q||, ||d|| = magnitudes (Euclidean norms).
  • Result: a number in [0, 1] (since vectors are non-negative).
  • 1 = perfect match; 0 = no overlap.

5. Steps of VSM-based retrieval

  1. Build vocabulary from corpus.
  2. Build document vectors using TF-IDF weighting.
  3. When a query arrives, build its vector.
  4. Compute cosine similarity with each document.
  5. Rank documents by similarity (highest first).
  6. Return top-K results to the user.

6. Example

Vocabulary: {ball, team, bank, money}

Documents (TF-IDF vectors):
  Doc1: [3, 4, 0, 0]    → about sports
  Doc2: [0, 0, 5, 6]    → about finance

Query: "team ball"
  q = [2, 2, 0, 0]

cos(q, Doc1) = (3·2 + 4·2 + 0 + 0) / (||q|| · ||Doc1||) = 14 / (√8 · √25) = 14 / 14.14 ≈ 0.99
cos(q, Doc2) = 0 / anything = 0

→ Doc1 is returned as the best match.

7. Diagram (2-D conceptual)

       y-axis "team"
        ↑
        |       Doc1 (sports)
        |     /
        |   /
        | /
        |
        |  q (query)
        |
   ─────|──────────────→ x-axis "ball"
        |
        |________________________
                         Doc2 (finance: orthogonal direction)

8. Strengths of VSM

  • Simple, intuitive, easy to implement.
  • Supports partial matching (not all terms need to match).
  • Produces ranked results (not just binary).
  • Foundation for many advanced IR techniques.
  • Works well with TF-IDF weighting.

9. Weaknesses / Limitations

  • Assumes terms are INDEPENDENT (ignores word order, syntax).
  • Cannot handle SYNONYMS (e.g., "car" vs "automobile" treated as different).
  • Cannot handle POLYSEMY (one word with multiple meanings).
  • Sparse high-dimensional vectors → computational cost.
  • No semantic understanding.
  • Solved by LSA, neural embeddings, BERT.

10. Applications

  • Web search engines (Google's early ranker).
  • Document retrieval systems.
  • Plagiarism detection.
  • Recommender systems (content-based filtering).
  • Email spam filtering.

Key points to remember

  • VSM = documents & queries as vectors in vocabulary space.
  • Weights usually TF-IDF.
  • Relevance = cosine similarity.
  • Strengths: simple, ranked results, partial matching.
  • Weaknesses: no synonymy/polysemy handling, no semantic understanding.

Q2 — Named Entity Recognition (NER)

Asked 5 times(May'23, Nov'23, May'24, May'25, Nov'25 — usually 4–9 marks)
How this question appeared in past papers:

(May'23, Nov'23, 4–8 marks) What is NER? Describe the various metrics used for evaluation.

(May'24, 9 marks) Methods for evaluating NER systems; common metrics.

(May'25, 2 marks) What is Named Entity Recognition (NER)?

(Nov'25, 9 marks) Describe NER system building process; outline steps using supervised learning.

Answer
When you read "Sundar Pichai is the CEO of Google in Mountain View", your brain instantly tags: Person = Sundar Pichai, Organization = Google, Location = Mountain View. NER teaches a computer to do this automatically.

1. Definition

Named Entity Recognition (NER) is a sub-task of NLP that automatically identifies and classifies named entities (persons, organizations, locations, dates, numerical values, etc.) within text.

2. Common entity types

  • PERSON — names of people (Sundar Pichai, Mahatma Gandhi).
  • ORG — organizations (Google, SPPU, ISRO).
  • LOC / GPE — locations (Pune, India, Asia).
  • DATE / TIME — temporal expressions (2026, 5 PM, Monday).
  • MONEY — currency amounts (₹500, $10).
  • PERCENT — percentages (50%).
  • PRODUCT — products (iPhone 15, Tesla Model 3).
  • EVENT — named events (Olympics, Diwali).

3. NER System Building Process (steps)

  1. Data collection: gather labeled text data (e.g., CoNLL-2003 dataset).
  2. Text preprocessing: tokenization, sentence splitting, lowercase (sometimes), POS tagging.
  3. Feature extraction:
    • Word-level features (capitalization, suffixes, prefixes).
    • Context features (previous/next words, POS tags).
    • Gazetteer features (lists of known entities).
    • Word embeddings (word2vec, BERT).
  4. Model training: use supervised algorithms like CRF (Conditional Random Field), BiLSTM-CRF, or transformer-based (BERT).
  5. Tagging scheme: usually IOB or BIO format
    • B = Beginning of entity
    • I = Inside entity
    • O = Outside any entity
  6. Evaluation on test set using precision, recall, F1.
  7. Deployment and continuous improvement.

4. Example with BIO tagging

Sentence:  Sundar  Pichai  is  the  CEO  of  Google  in  Mountain  View
Tags:      B-PER   I-PER   O   O    O    O   B-ORG   O   B-LOC     I-LOC

5. Evaluation Metrics for NER

MetricFormulaMeaning
PrecisionTP / (TP + FP)Of predicted entities, how many are correct.
RecallTP / (TP + FN)Of true entities, how many were caught.
F1 Score2·P·R / (P + R)Harmonic mean.
Accuracy(TP + TN) / TotalLess useful in NER (highly imbalanced).

Special considerations for NER evaluation:

  • Exact match: the entity boundaries AND type must be exactly correct.
  • Partial / Type match: partial credit if some tokens overlap or type matches.
  • Per-entity-type metrics: precision/recall computed separately for each entity type.
  • MUC / CoNLL scoring schemes are widely used in NER benchmarks.

6. Approaches to NER

  • Rule-based: regex patterns, dictionaries. Works for simple cases.
  • Statistical / ML: HMM, CRF, MaxEnt with hand-crafted features.
  • Deep Learning: BiLSTM-CRF, Transformer-based (BERT-NER, spaCy).
  • Hybrid: combines rule-based + ML.

7. Challenges of NER

  • Ambiguity: "Apple" → company or fruit?
  • Capitalization variation (especially in informal text).
  • Nested entities (e.g., "University of California, Los Angeles").
  • Cross-domain transfer (medical, legal terms differ).
  • Multilingual NER.

8. Applications

  • Search engines (extract names, locations from queries).
  • News aggregation (people, places, organizations in news).
  • Customer support (identify product names).
  • Medical NER (drug names, diseases).
  • Legal NER (laws, cases, jurisdictions).
  • Knowledge graph construction.

Key points to remember

  • NER = identify and classify named entities (PER, ORG, LOC, DATE, …).
  • Tagging scheme: BIO (Beginning, Inside, Outside).
  • Metrics: Precision, Recall, F1 (per type).
  • Approaches: rule-based, statistical (CRF), deep learning (BiLSTM-CRF, BERT).
  • Challenges: ambiguity, nested entities, domain transfer.

Q3 — Entity Extraction and Relation Extraction

Asked 5 times(May'23, Nov'23, May'24, May'25, Nov'25 — usually 8 marks)
How this question appeared in past papers:

(May'23 & Nov'23, 8 marks) Describe entity extraction and relation extraction with help of examples.

(May'24, 9 marks) Importance of entity extraction. How does it differ from NER?

(May'25, 8 marks) Process of Entity Extraction; techniques and algorithms.

(Nov'25, 8 marks) Compare Entity Extraction, Relation Extraction, and Coreference Resolution.

Answer

1. Entity Extraction

Entity Extraction is the task of identifying and extracting structured pieces of information (entities) from unstructured text. It is a SUPERSET of NER — it includes named entities AND other entities like phone numbers, product specs, custom domain entities, etc.

Difference from NER

AspectNEREntity Extraction
ScopeNamed entities (PER, ORG, LOC, etc.)Broader: includes NER + custom entities (phone, email, product, attribute)
DomainGenericOften domain-specific (medical, legal, finance)
OutputTagged tokensStructured records / key-value pairs

Techniques used for Entity Extraction

  • Rule-based (regex for emails, phone numbers).
  • Dictionary lookup (e.g., known product names list).
  • Statistical models (CRF, HMM).
  • Deep learning (BiLSTM-CRF, BERT).
  • Hybrid approaches.

Example of Entity Extraction

Text: "Please contact John Doe at john.doe@example.com or +91-9876543210
       for a quote on iPhone 15 (256GB) priced at ₹95,000."

Extracted entities:
  - PERSON: John Doe
  - EMAIL: john.doe@example.com
  - PHONE: +91-9876543210
  - PRODUCT: iPhone 15
  - STORAGE: 256GB
  - PRICE: ₹95,000

2. Relation Extraction

Relation Extraction (RE) is the task of identifying SEMANTIC RELATIONSHIPS between pairs of entities mentioned in text.

Common relation types

  • Person-Org: works-for, founded, CEO-of.
  • Person-Person: spouse, parent, sibling.
  • Person-Location: born-in, lives-in.
  • Org-Location: headquartered-in.
  • Other: part-of, contains, manufactured-by.

Example of Relation Extraction

Text: "Sundar Pichai is the CEO of Google."

Entities:  Sundar Pichai (PER),  Google (ORG)
Relation:  CEO-of(Sundar Pichai, Google)

Approaches

  • Pattern-based: handcrafted templates ("X is the CEO of Y").
  • Supervised: train classifier on labeled relation pairs (using SVM, neural nets).
  • Semi-supervised: bootstrapping starting from a few seed examples.
  • Distant supervision: automatically label sentences using existing knowledge bases.
  • Neural approaches: CNN, LSTM, transformer-based (BERT).

3. Entity Extraction vs Relation Extraction (comparison)

AspectEntity ExtractionRelation Extraction
What it findsIndividual entitiesConnections between entities
OutputList of entitiesTriples (Entity1, Relation, Entity2)
Comes firstEarlier in pipelineAfter entity extraction
Example"Pichai", "Google"CEO-of(Pichai, Google)

4. Applications

  • Knowledge graph construction (Wikidata, Freebase).
  • Question answering systems.
  • Resume parsing for HR.
  • Medical record extraction.
  • Legal document analysis.
  • Business intelligence (extract financial entities and relations).

Key points to remember

  • Entity Extraction = identify named + custom entities (broader than NER).
  • Relation Extraction = find semantic relationships between entities.
  • Output of RE: triples (Entity1, Relation, Entity2).
  • Approaches: pattern-based, supervised, semi-supervised, neural.
  • Used in knowledge graphs, QA, medical, legal, business apps.

Q4 — Cross-Lingual Information Retrieval (CLIR)

Asked 5 times(May'23, Nov'23, May'24, May'25, Nov'25 — usually 6–9 marks)
How this question appeared in past papers:

(May'23 & Nov'23, 6 marks) What is Cross-Lingual Information Retrieval? Provide an example.

(May'24, 9 marks) Define CLIR, discuss challenges. How do MT techniques assist?

(May'25, 8 marks) Explain CLIR. Challenges and techniques.

(Nov'25, 8 marks) What is CLIR? Discuss challenges and evaluate different approaches.

Answer
You type "best biryani recipe" in English, but the best recipe is on a Hindi blog. You won't find it with English-only search. CLIR translates the query (or the documents) so search can cross language boundaries.

1. Definition

Cross-Lingual Information Retrieval (CLIR) is the task of retrieving relevant documents written in one or more languages DIFFERENT from the language of the user's query. It enables users to access information across language barriers.

2. Example

  • User query (English): "best mango farming techniques"
  • Document collection: Marathi, Hindi, Tamil agriculture documents.
  • CLIR system: returns relevant documents in any of these languages.
  • Optionally translates the results back to English for the user.

3. Three main approaches to CLIR

(a) Query Translation

  • Translate the user's query into the document's language.
  • Then use standard monolingual IR.
  • Most common approach (queries are short, so MT is cheap).

(b) Document Translation

  • Translate the entire document collection into the user's language at index time.
  • Then use standard monolingual IR.
  • Expensive but improves search quality.

(c) Interlingual / Concept-based

  • Map both queries and documents into a language-INDEPENDENT representation (concepts, embeddings).
  • Compare in the shared space.
  • Uses multilingual embeddings like mBERT, LASER, multilingual BERT.

4. Challenges of CLIR

  • Translation ambiguity — words can have multiple meanings; wrong translation hurts retrieval.
  • Out-of-vocabulary terms — proper nouns, slang may not be in dictionary.
  • Morphological richness — Indian languages have many word forms.
  • Limited parallel corpora for low-resource language pairs.
  • Cultural and idiomatic expressions hard to translate.
  • Different writing scripts (Devanagari, Tamil, Arabic).
  • Different word orders (English SVO vs Hindi SOV).
  • Evaluation — needs multilingual test sets.

5. How Machine Translation assists CLIR

  • MT systems translate queries or documents automatically.
  • SMT (Statistical MT) — uses parallel corpora and statistical models.
  • NMT (Neural MT) — modern deep-learning approach (Transformer-based).
  • For better CLIR, MT systems can return MULTIPLE translation candidates (n-best lists).
  • Improvements: integrate translation probability into ranking, use query expansion.

6. Resources and tools for CLIR

  • Bilingual dictionaries.
  • Parallel corpora (EuroParl, IIT Bombay English-Hindi).
  • Pre-trained multilingual models (mBERT, XLM-R, MUSE).
  • Indic NLP tools (for Indian languages).

7. Applications

  • Multilingual search engines (Google, Bing).
  • Cross-language news aggregation.
  • Patent and scientific literature retrieval.
  • Cultural heritage digitization.
  • E-commerce — find products listed in other languages.
  • Tourism — find local information in a foreign country.

Key points to remember

  • CLIR = retrieve documents in a different language from the query.
  • Three approaches: query translation, document translation, interlingual.
  • MT (SMT or NMT) helps translate queries/docs.
  • Challenges: ambiguity, OOV terms, morphology, low-resource pairs, scripts.
  • Applications: multilingual search, news, patents, e-commerce.

Q5 — Reference Resolution & Coreference Resolution

Asked 4 times(May'23, Nov'23, May'25, Nov'25 — usually 4–8 marks)
How this question appeared in past papers:

(May'23, 4 marks) What is Coreference Resolution? Give examples.

(Nov'23, 8 marks) Explain reference resolution and coreference resolution with example.

(May'25, 8 marks) Reference and coreference resolution; relationships between entities.

(Nov'25, 8 marks) Compare entity extraction, relation extraction, coreference resolution.

Answer
Read: "Sundar Pichai joined Google in 2004. He became CEO in 2015." You instinctively know "He" refers back to "Sundar Pichai". Computers must be TAUGHT to make this link — that's coreference resolution.

1. Reference Resolution

Reference Resolution = the process of determining what real-world entity (or piece of text) a referring expression points to. It is the general task of linking referring expressions (pronouns, names, descriptions) to their referents.

2. Coreference Resolution

Coreference Resolution is a sub-task of reference resolution. It identifies all mentions in a text that refer to the SAME real-world entity, grouping them into "coreference chains".

3. Types of referring expressions

  • Pronominal: "he", "she", "it", "they", "this", "that".
  • Definite noun phrases: "the CEO", "the company".
  • Demonstrative: "that man", "those documents".
  • Proper names: "Sundar Pichai", "Google".
  • Anaphora: referring back ("Mary likes cats. She has three of them.").
  • Cataphora: referring forward ("When she arrived, Mary opened the door.").

4. Example of coreference resolution

Text:  "Sundar Pichai joined Google in 2004. He led product development. In 2015,
        Pichai became its CEO. The company's success is largely due to him."

Coreference chains:
  Chain 1 (Sundar Pichai):  Sundar Pichai → He → Pichai → him
  Chain 2 (Google):         Google → its → The company

5. Steps in Coreference Resolution

  1. Identify all mentions (entities, pronouns, noun phrases).
  2. For each pair of mentions, predict whether they refer to the same entity.
  3. Cluster mentions into coreference chains.

6. Approaches

  • Rule-based — gender, number, syntactic agreement, recency.
  • Mention-pair classifier — supervised ML; for each pair, predict coreferent or not.
  • Mention-ranking model — rank candidate antecedents.
  • Cluster-based — incrementally build clusters.
  • Neural / Deep learning — BERT-based coreference (state-of-the-art).

7. Challenges

  • Ambiguous pronouns ("he" could refer to multiple men in context).
  • Long-distance references.
  • World knowledge ("Joan asked Sue if she could help" — who is "she"?).
  • Gender ambiguity, especially in singular "they".
  • Nested coreference chains.

8. Comparison: Entity Extraction vs Relation Extraction vs Coreference Resolution

TaskGoalOutput
Entity ExtractionFind entitiesList of entities (with types)
Relation ExtractionFind connections between entitiesTriples (Entity1, Relation, Entity2)
Coreference ResolutionGroup mentions of same entityCoreference chains / clusters

9. Applications

  • Question Answering (resolve "she" in question to a specific person).
  • Text summarization (avoid duplicated mentions).
  • Machine Translation (correct pronoun translation).
  • Information Extraction (build knowledge graphs).
  • Dialogue systems (chatbots tracking referents).

Key points to remember

  • Reference resolution = link referring expressions to entities (general task).
  • Coreference resolution = group all mentions of same entity into a chain.
  • Types of referring expressions: pronouns, NPs, demonstratives, anaphora, cataphora.
  • Approaches: rule-based, mention-pair classifier, neural (BERT).
  • Used in QA, summarization, MT, dialogue systems.

Q6 — Concept of Information Retrieval

Asked 3 times(May'23, Nov'23, May'25 — usually 2–4 marks)
How this question appeared in past papers:

(May'23, 4 marks) Describe the concept of IR system in NLP.

(Nov'23, 4 marks) Describe IR; significance of NLP in IR.

(May'25, 2 marks) What is Information Retrieval, and how is it used in NLP?

Answer

1. Definition

Information Retrieval (IR) is the process of obtaining relevant information from a large collection (documents, web pages, images, etc.) in response to a user's information need expressed as a query.

2. Components of an IR system

  • Document corpus — collection of documents to search.
  • Indexing — preprocess and store documents in efficient data structures.
  • Query processor — analyze and normalize user's query.
  • Ranking algorithm — score documents by relevance.
  • User interface — display results.

3. IR pipeline

   User query  →  Tokenize  →  Remove stopwords  →  Stem  →  Match against index
                                                                 ↓
                                                              Rank docs
                                                                 ↓
                                                           Return top K

4. Role of NLP in IR

  • Query understanding — interpret synonyms, paraphrases, intent.
  • Document preprocessing — tokenization, lemmatization, POS tagging.
  • Named entity recognition — identify people, places in queries and docs.
  • Semantic matching — embeddings, BERT, beyond exact word match.
  • Question answering — answer questions directly, not just return docs.
  • Personalization — adapt to user language and context.
  • Cross-lingual retrieval — handle queries in one language, docs in another.

5. Models of IR

  • Boolean model — match terms using AND/OR/NOT.
  • Vector Space Model (VSM) — similarity in vector space (see Q1).
  • Probabilistic model (BM25) — probabilistic ranking.
  • Language model approach — model query as generated from document.
  • Neural IR — use BERT, DPR for dense retrieval.

6. Evaluation metrics

  • Precision — fraction of retrieved docs that are relevant.
  • Recall — fraction of relevant docs that are retrieved.
  • F1 score.
  • Mean Average Precision (MAP).
  • NDCG (Normalized Discounted Cumulative Gain).

7. Applications

  • Web search (Google, Bing, DuckDuckGo).
  • Library catalog search.
  • E-commerce product search.
  • Email search.
  • Patent and academic literature search.
  • Question answering systems.

Key points to remember

  • IR = finding relevant documents from a large collection given a query.
  • Components: corpus, indexing, query processing, ranking, UI.
  • NLP helps with query understanding, preprocessing, semantic matching, NER, QA, multilingual.
  • Models: Boolean, VSM, BM25, language model, neural IR.
  • Metrics: Precision, Recall, F1, MAP, NDCG.
