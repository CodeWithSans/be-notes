Answer
Translating "I love mangoes" вҶ’ Hindi can be done in 3 ways: (1) A linguist makes hand-crafted RULES ("subject + verb + object вҶ’ kartДҒ + karma + kriyДҒ") вҖ” Rule-Based MT. (2) A computer reads MILLIONS of pairs of English-Hindi sentences and learns statistical patterns of what word usually translates to what вҖ” Statistical MT. (3) A neural network learns translations end-to-end from huge data вҖ” Neural MT (modern, e.g., Google Translate).

1. What is Machine Translation (MT)?

Machine Translation is the use of software to translate text or speech from one natural language (source) to another (target). MT is one of the oldest and most challenging NLP applications.

2. Three main approaches

(A) Rule-Based Machine Translation (RBMT)

  • Uses HAND-CRAFTED linguistic rules and bilingual dictionaries.
  • Built by language experts.
  • Pipeline:
    1. Morphological analysis of source language.
    2. Syntactic parsing вҶ’ parse tree.
    3. Semantic transfer using rules and dictionary.
    4. Target language syntactic generation.
    5. Morphological generation in target language.
  • Three sub-types:
    • Direct (word-by-word): dictionary lookup + minimal reordering.
    • Transfer-based: parse вҶ’ transfer rules вҶ’ generate.
    • Interlingua-based: source вҶ’ language-independent meaning representation вҶ’ target.

(B) Statistical Machine Translation (SMT)

  • Learns translation probabilities from large parallel corpora.
  • Famous IBM models (1990s); Google Translate used SMT until ~2016.
  • Based on Bayes' rule:
    argmaxt P(t | s) = argmaxt P(s | t) В· P(t)
    • P(s | t) = translation model.
    • P(t) = target language model.
  • Two main types:
    • Word-based SMT: aligns single words.
    • Phrase-based SMT: aligns multi-word phrases (more accurate, dominant approach).
  • Three components:
    • Translation model вҖ” learned from parallel corpus.
    • Language model вҖ” fluency of target sentence.
    • Decoder вҖ” searches for best translation.

(C) Neural Machine Translation (NMT) вҖ” modern

  • Uses deep neural networks (encoder-decoder + attention; now Transformers).
  • End-to-end trained on parallel corpora.
  • Google Translate switched to NMT in 2016; quality jumped dramatically.
  • Examples: Google Translate, DeepL, OpenNMT.

7. Cross-Lingual Translation

  • Translation between language pairs where direct training data is scarce.
  • Uses pivot languages (e.g., Hindi вҶ’ English вҶ’ Tamil).
  • Modern multilingual NMT models like Google's M4 can translate among 100+ languages.

8. Evaluation of MT

  • BLEU (Bilingual Evaluation Understudy) вҖ” most common.
  • METEOR вҖ” improves on BLEU with synonyms and stemming.
  • TER (Translation Edit Rate).
  • Human evaluation вҖ” gold standard but expensive.

Key points to remember

  • 3 MT approaches: Rule-Based, Statistical (SMT), Neural (NMT).
  • RBMT = hand-crafted rules; SMT = learned from parallel corpora; NMT = end-to-end deep learning.
  • SMT formula: argmaxt P(s | t) В· P(t).
  • Modern systems (Google, DeepL) use NMT.
  • Evaluation: BLEU, METEOR, TER, human evaluation.

Answer
When you ask Siri "what's the weather?" and she replies "It will rain in Pune today, 25В°C, mostly cloudy", that response was GENERATED by an NLG system, starting from raw weather data. NLG produces natural-sounding human language from structured data or internal representations.

1. Definition

Natural Language Generation (NLG) is the NLP task of automatically producing human-readable text from data, knowledge bases, or other structured input. It is the OPPOSITE of Natural Language Understanding (NLU).

2. NLG vs Text-to-Speech (TTS)

OutputWritten textSpoken audio

ToolsGPT, Templates, T5Tacotron, WaveNet

NLG often precedes TTS in a voice assistant pipeline: NLG decides what to say вҶ’ TTS speaks it.

4. Detailed explanation of each stage

(1) Content Planning (Content Selection)

  • Decide WHICH facts/information to include in the output.
  • Filter important data from a large input.
  • Example: from a weather database, pick only temperature, rain, wind.

(2) Document Planning (Structuring)

  • Decide the OVERALL STRUCTURE of the output (paragraphs, sections, order).
  • Example: forecast first, then current conditions, then advice.

(3) Sentence Planning (Microplanning)

  • Combine related facts into sentences.
  • Choose voice (active/passive), tense, tone.
  • Example: combine "temperature = 25В°C" + "weather = rainy" вҶ’ "It will be rainy with 25В°C".

(4) Lexicalization (Word Choice)

  • Choose specific words/phrases.
  • Example: "rainy" vs "wet" vs "showery".

(5) Referring Expression Generation

  • Decide how to refer to entities (full name, pronoun, etc.).
  • Example: "Pune" first time вҶ’ "the city" later вҶ’ "it" after.

(6) Surface Realization (Linguistic Generation)

  • Apply grammar rules, morphological inflections, punctuation.
  • Produce the final grammatically correct sentence.

5. Approaches to NLG

  • Template-based: fill blanks in pre-written templates (simple but rigid).
  • Rule-based / grammar-based: use linguistic rules (controlled output).
  • Statistical NLG: learn from corpora.
  • Neural NLG: use seq2seq, transformers, GPT (modern, flexible).

6. Applications of NLG

  • Automated journalism вҖ” sports recaps, financial reports.
  • Weather forecasts from meteorological data.
  • Business intelligence dashboards вҖ” narrative summaries.
  • Chatbots and virtual assistants (Siri, Alexa, Google Assistant).
  • E-commerce вҖ” product descriptions auto-generated.
  • Medical reports from patient data.
  • Data storytelling for analytics.
  • Personalized email / SMS generation.
  • Game dialogue generation.
  • Code documentation generation.

7. Modern NLG (transformer-based)

  • GPT, T5, BART have largely replaced traditional pipeline NLG for many tasks.
  • End-to-end: take input data, output natural text directly.
  • State-of-the-art for creative writing, summarization, dialogue.

Key points to remember

  • NLG = generate human-readable text from data / structured input.
  • Different from TTS: NLG produces text; TTS produces audio.
  • 6 stages: Content Planning вҶ’ Document Planning вҶ’ Sentence Planning вҶ’ Lexicalization вҶ’ Referring Expressions вҶ’ Surface Realization.
  • Approaches: template, rule-based, statistical, neural (GPT, T5).
  • Applications: automated journalism, weather reports, chatbots, BI, e-commerce.

Q3 вҖ” Question Answering (QA) System

Asked 3 times(Nov'23, May'25, Nov'25 вҖ” usually 7вҖ“9 marks)
How this question appeared in past papers:

(Nov'23, 7 marks) Explain three stages of Question Answering system with neat diagram.

(May'25, 8 marks) Explain fundamental components of a QA system.

(Nov'25, 9 marks) Explain architecture and challenges of building a QA system.

Answer

1. Definition

A Question Answering (QA) system is an NLP system that automatically answers questions posed in natural language by retrieving information from a knowledge source (text corpus, structured database, or knowledge graph).

2. Types of QA systems

  • Closed-domain QA: answers questions in a specific domain (medical, legal).
  • Open-domain QA: answers questions on any topic (Wikipedia, web).
  • Factoid QA: answer is a specific fact (date, name).
  • Non-factoid QA: longer explanations or opinions.
  • Reading comprehension: answer based on a given passage.
  • Knowledge-based QA: answer using a structured knowledge graph.

3. Three main stages of QA (classical pipeline)

Stage 1: Question Analysis / Processing

  • Tokenize and parse the question.
  • Classify question type (who, what, where, when, how, why).
  • Identify expected answer type (PERSON, LOCATION, DATE, NUMBER, вҖҰ).
  • Extract keywords / entities.
  • Generate a query for the retrieval step.

Stage 2: Document / Passage Retrieval

  • Use IR techniques (TF-IDF, BM25, or neural retrievers) to fetch documents relevant to the question.
  • Optionally split documents into smaller passages.
  • Rank passages by relevance.

Stage 3: Answer Extraction

  • From the top passages, identify the exact ANSWER SPAN.
  • For factoid questions, look for entity matching the expected type.
  • Modern systems use BERT-based reading comprehension models.
  • Rank candidate answers; return the most confident.

5. Modern neural QA systems

  • BERT-based QA вҖ” fine-tune BERT on SQuAD-like datasets; predict answer start and end positions.
  • Retriever-Reader вҖ” dense passage retrieval (DPR) + BERT reader.
  • End-to-end transformer models вҖ” T5, BART.
  • Knowledge graph QA вҖ” answer using entity-relation graph.
  • Conversational QA вҖ” multi-turn (context-aware).

6. Challenges in QA

  • Ambiguous questions ("How long is вҖҰ?" вҖ” length or duration?).
  • Multi-hop reasoning (combining facts from multiple sources).
  • Out-of-domain or open-ended questions.
  • Limited training data for non-English languages.
  • Handling conflicting information from different sources.
  • Evaluating answer correctness (exact match, F1, semantic similarity).
  • Generating fluent natural answers (not just extracting spans).
  • Real-time response speed.

7. Real-world examples

  • IBM Watson (won Jeopardy! in 2011).
  • Google Featured Snippets.
  • Apple Siri, Amazon Alexa, Google Assistant.
  • ChatGPT (LLM-based QA).

Key points to remember

  • QA = automatically answer questions in natural language.
  • 3 stages: Question Analysis вҶ’ Document Retrieval вҶ’ Answer Extraction.
  • Types: closed-domain, open-domain, factoid, non-factoid, reading comprehension.
  • Modern QA uses BERT and retriever-reader architectures.
  • Challenges: ambiguity, multi-hop reasoning, evaluation, multilingual.

Q4 вҖ” Conversational Agents / Chatbots

Asked 2 times(May'24, Nov'25 вҖ” usually 8 marks)
How this question appeared in past papers:

(May'24, 8 marks) Key components of a conversational agent (chatbots, virtual assistants). Role of NLG and NLU.

(Nov'25, 8 marks) Discuss key components of a conversational agent. Role of NLG and NLU in creating effective conversational agents.

Answer

1. Definition

A Conversational Agent (also called chatbot, virtual assistant, or dialog system) is an AI system designed to interact with humans through natural language, either via text or voice. Examples: Siri, Alexa, Google Assistant, ChatGPT, customer-support chatbots.

2. Types of Conversational Agents

  • Rule-based / Pattern-matching: simple if-else, regex (ELIZA, AIML).
  • Retrieval-based: selects best response from predefined templates.
  • Generative: generates novel responses (GPT-based).
  • Task-oriented: helps with specific tasks (booking flights, banking).
  • Open-domain / chitchat: general conversation.

3. Key Components

(a) Natural Language Understanding (NLU)

  • Convert user's input text into structured representation.
  • Subtasks:
    • Intent recognition (e.g., "book flight" intent).
    • Entity / slot extraction (e.g., source = Pune, destination = Mumbai, date = tomorrow).
    • Sentiment / tone detection.

(b) Dialog Manager

  • The "brain" вҖ” tracks conversation state and decides next action.
  • Maintains context across turns.
  • Implements policy: ask for missing info, confirm, execute action.

(c) Knowledge Base / Backend

  • Database, API, or knowledge graph that holds factual information.
  • Examples: flight schedule API, FAQ database, weather service.

(d) Natural Language Generation (NLG)

  • Convert system's response into fluent natural language.
  • Either template-based or neural.

(e) Speech Components (for voice assistants)

  • Automatic Speech Recognition (ASR): speech вҶ’ text.
  • Text-to-Speech (TTS): text вҶ’ speech.

5. Role of NLU and NLG in effectiveness

  • NLU is the FRONT door. If it misinterprets the user's intent, the rest goes wrong.
    • Good NLU = correctly understands varied phrasings ("Book me a ticket" = "Reserve a seat" = "I want to fly").
    • Strong entity extraction вҶ’ fewer follow-up clarifications.
  • NLG is the BACK door. Even with perfect understanding, a clunky response feels robotic.
    • Good NLG = natural, varied, polite responses.
    • Personalization, context-awareness, emotional tone matter.
  • Together they make the conversation feel HUMAN-LIKE.

6. Challenges in building chatbots

  • Handling unexpected input ("out-of-domain" questions).
  • Maintaining context across many turns.
  • Personalization (different users need different responses).
  • Multilingual support.
  • Emotional intelligence and empathy.
  • Avoiding offensive or biased responses.
  • Integrating with backend systems.

7. Examples

  • Siri (Apple), Alexa (Amazon), Google Assistant.
  • Cortana (Microsoft).
  • ChatGPT (OpenAI), Claude (Anthropic) вҖ” large language model based.
  • Customer support chatbots on banking, e-commerce websites.

Key points to remember

  • Conversational agent = chatbot / virtual assistant.
  • Components: NLU, Dialog Manager, Knowledge Base, NLG, (ASR + TTS for voice).
  • NLU understands intent + entities; NLG produces fluent response.
  • Both NLU and NLG quality directly affect user experience.
  • Modern systems use LLMs (GPT, Claude) for end-to-end conversations.

Q5 вҖ” Text Entailment

Asked 2 times(Nov'23, May'25 вҖ” usually 9вҖ“10 marks)
How this question appeared in past papers:

(Nov'23, 10 marks) Describe Text Entailment as an NLP application.

(May'25, 9 marks) Define text entailment; importance in natural language understanding. How does it differ from text similarity or paraphrasing?

Answer

1. Definition

Text Entailment (also called Recognizing Textual Entailment, RTE) is the NLP task of determining whether the MEANING of one piece of text (called the hypothesis) can be INFERRED from another piece of text (called the premise).

2. The three possible relations

  • Entailment: hypothesis logically follows from the premise.
  • Contradiction: hypothesis is logically opposite to the premise.
  • Neutral: hypothesis is neither entailed nor contradicted.

3. Examples

PremiseHypothesisLabel
"A man is playing guitar.""A person is making music."Entailment
"A man is playing guitar.""A man is silent."Contradiction
"A man is playing guitar.""The man enjoys jazz."Neutral

4. Importance in Natural Language Understanding (NLU)

  • Tests a system's ability to do REASONING with language.
  • Underlies many NLP tasks:
    • Question answering (answer must entail the question).
    • Information retrieval (results must entail query intent).
    • Summarization (summary must entail the source).
    • Machine translation (translation should entail source).
    • Plagiarism detection.

5. Text Entailment vs Similarity vs Paraphrasing

ConceptDefinitionDirection
EntailmentHypothesis follows from premiseOne-way (premise вҶ’ hypothesis)
SimilarityTwo texts share meaning to some degreeSymmetric
ParaphrasingTwo texts have the same meaning, different wordsSymmetric (bidirectional entailment)

So: paraphrasing = entailment in BOTH directions. Similarity is a soft, gradient measure (not yes/no).

6. Approaches to Text Entailment

  • Lexical overlap вҖ” count shared words.
  • Logical / rule-based вҖ” convert to formal logic.
  • Machine learning with hand-crafted features.
  • Deep learning вҖ” LSTM, BERT, transformer-based classifiers.
  • NLI benchmarks: SNLI, MultiNLI, ANLI datasets.

7. Applications

  • Search engine validation (does the result answer the query?).
  • Fact-checking (does an article support a claim?).
  • Summarization evaluation.
  • Question answering pipelines.
  • Educational systems (grading student answers).
  • Legal document analysis.

Key points to remember

  • Text Entailment = does hypothesis follow from premise?
  • Three labels: Entailment, Contradiction, Neutral.
  • Different from similarity (gradient) and paraphrasing (bidirectional entailment).
  • Modern approach: BERT-based classifier.
  • Applications: QA, fact-checking, summarization eval, plagiarism.

Q6 вҖ” Sentiment Analysis

Asked 1 time(May'23 вҖ” 10 marks)
How this question appeared in past papers:

(May'23, 10 marks) Write a note on Sentiment Analysis.

Answer

1. Definition

Sentiment Analysis (also called Opinion Mining) is the NLP task of automatically determining the EMOTIONAL TONE or OPINION expressed in a piece of text вҖ” typically positive, negative, or neutral.

2. Granularity levels

  • Document-level: overall sentiment of a whole document/review.
  • Sentence-level: sentiment of individual sentences.
  • Aspect-based: sentiment toward specific aspects ("the camera is great, but the battery is poor").
  • Entity-level: sentiment toward specific entities (people, products).

3. Categories of sentiment

  • Binary: positive vs negative.
  • Ternary: positive / negative / neutral.
  • Fine-grained: 1-5 star rating.
  • Emotion classification: joy, anger, sadness, fear, surprise, disgust.

4. Approaches

(a) Lexicon-based / Rule-based

  • Use a sentiment lexicon (e.g., SentiWordNet, VADER, AFINN).
  • Count positive and negative words; output the difference.
  • Handle negation ("not good" вҶ’ flip sentiment).
  • Pros: simple, interpretable, no training data needed.
  • Cons: misses context and sarcasm.

(b) Machine Learning

  • Features: bag-of-words, TF-IDF, n-grams.
  • Classifiers: Naive Bayes, SVM, Logistic Regression.
  • Pros: better than rules; needs labeled training data.

(c) Deep Learning

  • LSTM, CNN, Transformer (BERT) based sentiment classifiers.
  • State-of-the-art accuracy.
  • Handle context, negation, complex syntax well.

5. Challenges

  • Sarcasm and irony ("Oh great, another Monday!").
  • Negation ("not bad" = positive).
  • Domain dependence (positive in movies вү  positive in food).
  • Multilingual sentiment.
  • Aspect-level sentiment (different aspects = different sentiments).
  • Emoji and slang handling.
  • Contextual / discourse-level sentiment.

6. Applications

  • Product review analysis (Amazon, Flipkart).
  • Social media monitoring (Twitter sentiment around brands, politicians, events).
  • Customer feedback analysis.
  • Stock market prediction from news sentiment.
  • Movie / restaurant rating prediction.
  • Election prediction.
  • Hate-speech detection.

7. Tools and resources

  • VADER (Python library) вҖ” for social media.
  • TextBlob вҖ” simple polarity score.
  • SentiWordNet вҖ” WordNet with sentiment scores.
  • Hugging Face transformers вҖ” pre-trained BERT sentiment models.

Key points to remember

  • Sentiment analysis = determine emotional tone (positive / negative / neutral).
  • Granularity: document, sentence, aspect, entity.
  • Approaches: lexicon, ML (NB, SVM), deep learning (LSTM, BERT).
  • Challenges: sarcasm, negation, domain, multilingual.
  • Applications: reviews, social media, finance, customer feedback.

Q8 вҖ” Cross-Lingual Translation Challenges

Asked 1 time(May'24 вҖ” 8 marks)
How this question appeared in past papers:

(May'24, 8 marks) Discuss the challenges in cross-lingual translation and benefits in real-world applications.

Answer

1. What is Cross-Lingual Translation?

Translation between two languages where the system may not have abundant direct training data. Often handled by pivoting through a high-resource language (typically English).

2. Challenges

  • Limited parallel corpora for low-resource language pairs (e.g., Marathi вҶ’ Tamil).
  • Differences in grammar / word order (English SVO vs Hindi SOV).
  • Morphological richness in Indian languages (one English word = many Hindi forms).
  • Cultural and idiomatic expressions hard to translate directly.
  • Out-of-vocabulary words (proper nouns, technical terms).
  • Different writing scripts (Latin, Devanagari, Tamil, Arabic).
  • Ambiguity propagation through multiple pivot translations.
  • Quality evaluation for non-English languages is hard.
  • Code-mixing / code-switching (Hinglish, Tamglish) in real text.
  • Polysemy вҖ” words with multiple meanings.

3. Techniques to address these challenges

  • Pivot translation: source вҶ’ English вҶ’ target.
  • Multilingual NMT models (mBART, M2M-100, NLLB) вҖ” single model for many language pairs.
  • Transfer learning from high-resource to low-resource languages.
  • Back-translation (synthetic parallel data).
  • Subword tokenization (BPE) for morphologically rich languages.
  • Romanization / script normalization.

4. Benefits in real-world applications

  • Global communication вҖ” break language barriers.
  • E-commerce вҖ” translate product listings to local languages.
  • Healthcare вҖ” translate medical instructions.
  • Education вҖ” translate learning content (Khan Academy in multiple languages).
  • News вҖ” translate international news.
  • Government services вҖ” make official documents accessible in regional languages.
  • Tourism and travel вҖ” Google Translate camera mode for menus and signs.
  • Customer support вҖ” multilingual chatbots.
  • Cultural preservation вҖ” translate texts of endangered languages.

Key points to remember

  • Challenges: limited corpora, grammar differences, morphology, cultural idioms, OOV, scripts, code-mixing.
  • Solutions: pivot translation, multilingual NMT, transfer learning, back-translation.
  • Benefits: global communication, e-commerce, education, news, government.
