Preliminary Concepts (Read Once)

Building NLP from scratch (tokenization, POS tagging, parsing) is like writing your own car engine before driving. Most developers use READY-MADE NLP LIBRARIES (NLTK, spaCy) just like drivers use ready-made cars. They also use LEXICAL RESOURCES like WordNet (a giant dictionary of word meanings) the way a writer uses a thesaurus.

Two main categories in this chapter

  1. NLP Libraries / Tools — software you import in your Python code (NLTK, spaCy, TextBlob, Gensim).
  2. Lexical Knowledge Networks — pre-built linguistic databases (WordNet, IndoWordNet, VerbNet, PropBank, Treebanks).

Plus one important technique: Word Sense Disambiguation (WSD) using Lesk and Walker's algorithms.

Q1 — Lesk Algorithm and Walker's Algorithm for Word Sense Disambiguation

Asked 5 times(May'23, Nov'23, May'24, May'25, Nov'25 — usually 8–10 marks)
How this question appeared in past papers:

(May'23 & Nov'23, 10 marks) Describe in detail the Lesk algorithm and Walker's algorithm for word sense disambiguation.

(May'24, 9 marks) Describe Walker's algorithm for WSD; differences from Lesk; scenarios where most effective.

(May'25, 8 marks) Explain the Lesk Algorithm for WSD. How does it work; advantages and limitations.

(Nov'25, 9 marks) Describe Walker's algorithm for WSD. Differences from Lesk.

Answer
The word "bank" can mean a financial institution OR the side of a river. When you read "He sat on the bank watching fish jump", you instantly know it's the river bank — because of the surrounding words ("fish jump"). Word Sense Disambiguation (WSD) teaches a computer to make this decision.

1. Word Sense Disambiguation (WSD) — Definition

WSD is the NLP task of automatically identifying which SENSE (meaning) of a word is being used in a given context, when the word has multiple possible meanings (polysemy).

2. Why WSD matters

  • Improves machine translation accuracy.
  • Critical for information retrieval.
  • Helps in question answering.
  • Important for text summarization and sentiment analysis.

3. Lesk Algorithm

Definition

The Lesk Algorithm (proposed by Michael Lesk, 1986) is a dictionary-based WSD method. It selects the sense whose dictionary definition (called the GLOSS) has the most word OVERLAP with the surrounding context of the ambiguous word.

Steps of Lesk Algorithm

  1. Take the ambiguous word w and its surrounding context (the words near it in the sentence).
  2. Look up w in a dictionary (e.g., WordNet) → retrieve all its senses with their glosses.
  3. For each sense, count how many words in the gloss also appear in the context.
  4. Choose the sense with the HIGHEST overlap count.

Example

Sentence: "He went to the bank to deposit money."

Word to disambiguate: "bank"

Sense 1: "a financial institution that accepts deposits and channels money"
   Gloss words: {financial, institution, accepts, deposits, channels, money}
   Context: {went, deposit, money}
   Overlap: {deposits, money} → 2 words

Sense 2: "sloping land especially beside a body of water"
   Gloss words: {sloping, land, beside, body, water}
   Context: {went, deposit, money}
   Overlap: {} → 0 words

→ Choose Sense 1 (financial institution).

Advantages of Lesk

  • Simple, fast, easy to implement.
  • Requires only a dictionary (e.g., WordNet); no training data needed.
  • Domain-independent.

Limitations of Lesk

  • Dictionary glosses are SHORT — small overlaps mean low accuracy.
  • Sensitive to exact word matches (no synonym handling).
  • Best sense often missed if context words differ from gloss.
  • Variants: Simplified Lesk, Extended Lesk (uses related senses too) improve accuracy.

4. Walker's Algorithm

Definition

Walker's Algorithm (Walker & Amsler 1986) is a knowledge-based WSD method that uses a thesaurus organized by SUBJECT CATEGORIES (semantic codes). It assigns each ambiguous word to the sense whose category appears most often in the context.

Idea

  • Each word sense is associated with one or more subject categories (e.g., FINANCE, NATURE, SPORTS).
  • Look at all the words in the context → see which subject categories they belong to.
  • Choose the sense of the ambiguous word whose category is most frequent in the context.

Steps of Walker's Algorithm

  1. For each sense of the ambiguous word, list its subject categories from the thesaurus.
  2. For each surrounding context word, find its possible subject categories.
  3. Count how many context words "vote" for each subject category.
  4. Pick the sense of the ambiguous word whose category received the most votes.

Example

Word: "bank"
Senses:
  Sense 1 (financial) → category: FINANCE
  Sense 2 (river side) → category: NATURE

Context: "He went to the bank to deposit money."
Categories of context words:
   "deposit" → FINANCE
   "money"   → FINANCE
   "went"    → MOTION (generic)
   "the"     → (function word, ignored)

Votes: FINANCE = 2, NATURE = 0
→ Choose Sense 1 (financial bank).

Advantages

  • Uses broader semantic information than Lesk.
  • Works even when exact words don't overlap (uses category, not surface words).
  • Can leverage existing thesauri (Roget's Thesaurus).

Limitations

  • Depends on quality of subject-category mapping.
  • Some words have ambiguous categories.
  • Not all words have category tags in a thesaurus.
  • Less popular today than statistical or neural WSD.

5. Lesk vs Walker — Comparison

AspectLesk AlgorithmWalker's Algorithm
Resource usedDictionary glosses (WordNet)Thesaurus with subject categories
Comparison basisWord overlap between gloss and contextSubject category match between sense and context
GranularityWord-level matchingSemantic category-level voting
Synonym handlingPoor (exact match only)Better (uses semantic categories)
SimplicityVery simpleSlightly more complex
Best forShort, informal textText where domain categories are clear

6. Other WSD approaches (briefly)

  • Supervised: train on sense-annotated corpora (SVM, neural).
  • Semi-supervised / Bootstrapping (Yarowsky algorithm).
  • Unsupervised clustering of contexts.
  • Neural WSD using BERT embeddings.

Key points to remember

  • WSD = pick the right sense of a polysemous word using context.
  • Lesk: compare dictionary GLOSS with surrounding context; choose max word overlap.
  • Walker's: use SUBJECT CATEGORIES from thesaurus; choose category with most votes.
  • Lesk = word matching; Walker = category matching.
  • Modern systems use BERT-based WSD with much higher accuracy.

Answer

1. Major NLP libraries / tools

  • NLTK (Natural Language Toolkit)
  • spaCy
  • TextBlob
  • Gensim
  • Stanford CoreNLP
  • Hugging Face Transformers
  • OpenNLP (Java)
  • Pattern
  • Polyglot (multilingual)
  • AllenNLP
  • Indic NLP Library (for Indian languages)

2. NLTK (Natural Language Toolkit)

  • Created in 2001 at the University of Pennsylvania.
  • One of the OLDEST and most popular Python NLP libraries.
  • Originally designed for teaching/research.
  • Features:
    • Tokenization (word, sentence, regex-based, punctuation-based).
    • POS tagging.
    • Stemming (Porter, Lancaster) and Lemmatization (WordNet).
    • Parsing (CFG, chart parsers).
    • Built-in corpora (Brown, Reuters, etc.).
    • WordNet interface for word senses.
    • Classification (Naive Bayes, Decision Tree).
    • n-gram modeling.
    • Concordance and frequency analysis.
  • Pros: comprehensive, educational, free; large built-in corpora.
  • Cons: slower than spaCy; less suitable for production.

3. spaCy

  • Released in 2015. Modern, production-grade NLP library.
  • Written in Cython for speed.
  • Features:
    • Fast tokenization, POS tagging, NER.
    • Dependency parsing.
    • Pre-trained models for many languages.
    • Word vectors and similarity.
    • Pipeline architecture (configurable).
    • Easy integration with deep learning frameworks.
  • Pros: very fast, accurate, production-ready, good documentation.
  • Cons: fewer educational helpers than NLTK; harder to customize.

4. TextBlob

  • Built on top of NLTK and Pattern.
  • Provides a SIMPLE, beginner-friendly API.
  • Features:
    • Tokenization, POS tagging, noun-phrase extraction.
    • Sentiment analysis (polarity and subjectivity).
    • Translation (uses Google Translate API).
    • Spell correction.
    • Pluralize/singularize words.
    • Tokenizing and N-grams.
  • Pros: very easy to use; good for quick prototypes.
  • Cons: less powerful for advanced tasks; depends on external APIs for translation.

5. Gensim

  • Specialized library for topic modeling and document similarity.
  • Features:
    • Word embeddings (word2vec, doc2vec, FastText).
    • Topic modeling: LSI, LDA, HDP, NMF.
    • TF-IDF model.
    • Document similarity using cosine, Jaccard.
    • Streaming corpus support (works on huge data without loading all in memory).
    • Pre-trained Word2Vec models.
  • Pros: excellent for topic modeling and embeddings; memory-efficient.
  • Cons: doesn't do general-purpose NLP (no parsing, NER).

6. How NLTK / spaCy / Gensim handle Tokenization + POS Tagging

Key points to remember

  • NLP libraries: NLTK, spaCy, TextBlob, Gensim, Stanford CoreNLP, Hugging Face.

Q3 — Lexical Knowledge Networks (WordNet, IndoWordNet, VerbNet, PropBank, Treebanks)

Asked 5 times(Nov'23, May'24, May'25, Nov'25 — usually 8–10 marks)
How this question appeared in past papers:

(Nov'23, 10 marks) Explain the following lexical knowledge networks: WordNet, IndoWordNet, VerbNet, PropBank, Treebanks.

(May'24, 8 marks) Compare IndoWordNet with traditional WordNet. Differences and advantages for Indian languages.

(May'24, 8 marks) Significance of PropBank and VerbNet; how they extract semantic information.

(May'25, 9 marks) Role of Treebanks and Universal Dependency Treebanks in NLP research.

(Nov'25, 9 marks) Compare IndoWordNet with WordNet for Indian languages.

Answer

1. WordNet

  • Developed at Princeton University by George A. Miller (1985).
  • A large lexical database of English words.
  • Words are organized into synsets (sets of synonymous words).
  • Synsets are linked by semantic relations:
    • Synonymy: car ≈ automobile.
    • Hypernymy/Hyponymy: "is-a" relations (dog is-a mammal).
    • Meronymy/Holonymy: "part-of" (wheel is part of car).
    • Antonymy: hot ↔ cold.
  • Contains nouns, verbs, adjectives, adverbs.
  • Used in WSD, IR, machine translation.

2. IndoWordNet

  • Developed by IIT Bombay (Centre for Indian Language Technology).
  • WordNet for INDIAN LANGUAGES (Hindi, Marathi, Bengali, Tamil, Telugu, Punjabi, etc.).
  • Connects synsets across multiple Indian languages.
  • Linked to English WordNet for cross-lingual applications.

3. WordNet vs IndoWordNet — comparison

AspectWordNetIndoWordNet
LanguageEnglishMultiple Indian languages
Coverage~150,000 wordsVaries per language
Use caseEnglish NLPIndian language NLP, multilingual
Built byPrinceton (Miller)IIT Bombay (Dr. Pushpak Bhattacharyya)
Cross-lingual linksIndirectly via mappingsBuilt-in across Indian languages
MorphologySimple English morphologyRich morphology of Indian languages

Advantages of IndoWordNet

  • Enables NLP for low-resource Indian languages.
  • Supports cross-lingual NLP (Hindi → Marathi).
  • Handles complex morphology of Indian languages.
  • Aids in machine translation between Indian languages.
  • Useful for Indian language WSD and IR.

4. VerbNet

  • A lexical resource for English VERBS, organized by Levin classes.
  • Each verb class has:
    • Syntactic frames (how verb is used grammatically).
    • Semantic predicates (meaning components).
    • Member verbs (verbs sharing similar behavior).
  • Example: the verb "give" has frames like "Subject gives Object to Recipient".
  • Used for semantic role labeling, parsing.

5. PropBank (Proposition Bank)

  • A corpus of texts annotated with verbal propositions and their arguments.
  • Each verb has labeled arguments (Arg0 = agent, Arg1 = patient, Arg2 = instrument, …).
  • Used for Semantic Role Labeling (SRL).
  • Example: "John broke the window with a hammer."
    • Arg0 (agent) = John
    • Arg1 (patient) = the window
    • Arg2 (instrument) = a hammer

6. Treebanks

  • A corpus of sentences annotated with their SYNTACTIC STRUCTURE (parse trees).
  • Most famous: Penn Treebank (~3 million words of English).
  • Other examples: Chinese Treebank, Hindi Treebank.
  • Used to train parsers (statistical and neural).
  • Two types:
    • Constituency Treebank (phrase structure trees).
    • Dependency Treebank (word-to-word dependency relations).

7. Universal Dependencies (UD) Treebanks

  • A cross-lingual framework for annotating grammar.
  • Uses a single, consistent set of dependency relations across 100+ languages.
  • Allows ONE parser architecture to work on many languages.
  • Crucial for multilingual NLP research.
  • Indian languages like Hindi, Tamil, Urdu have UD treebanks.

8. Role of Treebanks in NLP

  • Training and benchmarking dependency / constituency parsers.
  • POS tagger evaluation.
  • Linguistic research on grammar.
  • Building cross-lingual NLP models (via UD).
  • Used for machine translation and grammar checking.

9. Comparison of all resources

ResourceWhat it containsUsed for
WordNetSynsets + semantic relationsWSD, IR, MT
IndoWordNetSynsets in Indian languagesIndian language NLP
VerbNetVerb classes + framesSemantic parsing, role labeling
PropBankAnnotated verb argumentsSemantic Role Labeling
TreebankAnnotated parse treesParser training, grammar analysis
UD TreebankMulti-lingual dependency annotationsCross-lingual NLP

Key points to remember

  • WordNet = English synsets + semantic relations.
  • IndoWordNet = WordNet for Indian languages, by IIT Bombay.
  • VerbNet = English verb classes with syntactic and semantic info.
  • PropBank = annotated verb arguments (Arg0, Arg1, …) for SRL.
  • Treebanks = corpora with parse-tree annotations; UD = cross-lingual.

Q4 — WordNet (detailed)

Asked 2 times(May'23, May'25 — usually 8–10 marks)
How this question appeared in past papers:

(May'23, 10 marks) Write a note on WordNet.

(May'25, 8 marks) Explain WordNet as a lexical knowledge network. How can it be used in NLP, and what are its limitations?

Answer

1. Definition

WordNet is a large lexical database of English where nouns, verbs, adjectives, and adverbs are grouped into sets of cognitive synonyms (synsets), each expressing a distinct concept. Synsets are interlinked by semantic and lexical relations.

2. Key concepts

  • Synset: a set of synonymous words (e.g., {car, automobile, motorcar}).
  • Gloss: a short definition of the synset.
  • Examples: usage examples for the synset.
  • Lexical relations: between words (synonymy, antonymy).
  • Semantic relations: between synsets (hypernymy, meronymy).

3. Semantic relations in WordNet

  • Synonymy: words with similar meaning (car / automobile).
  • Antonymy: opposites (hot / cold).
  • Hypernymy: more general ("vehicle" is hypernym of "car").
  • Hyponymy: more specific ("car" is hyponym of "vehicle").
  • Meronymy: "part-of" (wheel is part of car).
  • Holonymy: "whole" relation (car is whole of wheel).
  • Troponymy: for verbs (to whisper is a way of speaking).
  • Entailment: verb A implies verb B (snore entails sleep).

4. Structure (text diagram)

                    Entity (root)
                       │
                  ─────┴─────
                  │           │
                Object      Abstraction
                  │
              Vehicle ←── Hypernym of "Car"
                  │
                  Car ──── (synonym: automobile)
                  │            │
                  │         meronyms: wheel, engine
                  │
                  ↓ (hyponyms)
                Sedan, SUV, Hatchback, ...

5. How WordNet is used in NLP

  • Word Sense Disambiguation — Lesk algorithm uses WordNet glosses.
  • Information Retrieval — query expansion using synonyms.
  • Machine Translation — finding semantic equivalents.
  • Text Summarization — identifying related concepts.
  • Sentiment Analysis — SentiWordNet adds polarity scores.
  • Question Answering — finding hypernyms / hyponyms of query terms.
  • Semantic similarity measures based on WordNet path distance.

6. Limitations of WordNet

  • English-only (other languages need WordNets like IndoWordNet, EuroWordNet).
  • Limited coverage of new / rare words.
  • No frequency information (all senses treated equally).
  • No context-dependent senses (BERT handles this better).
  • Sparse coverage of multi-word expressions.
  • No proper nouns (people, places, organizations) — needs other resources.
  • Manually built, expensive to maintain.
  • Verb-relations less detailed than noun-relations.

Key points to remember

  • WordNet = English lexical database; words grouped into synsets.
  • Relations: synonymy, antonymy, hypernymy/hyponymy, meronymy/holonymy, troponymy, entailment.
  • Uses: WSD, IR, MT, summarization, sentiment analysis, QA.
  • Limitations: English-only, no frequency info, no proper nouns.

Q5 — Gensim Library

Asked 2 times(May'23, Nov'25 — usually 7–9 marks)
How this question appeared in past papers:

(May'23, 7 marks) Which types of tasks are performed by the Gensim library? Give an example.

(Nov'25, 9 marks) Tasks performed by Gensim with an example.

Answer

1. What is Gensim?

Gensim (Generate Similar) is an open-source Python library for topic modeling, document indexing, and similarity retrieval. It is specialized in working with large text corpora using streaming algorithms (no need to load all data in memory).

2. Main tasks performed by Gensim

  • Topic modeling: LDA, LSI/LSA, HDP, NMF.
  • Word embeddings: Word2Vec, FastText, Doc2Vec.
  • TF-IDF model.
  • Document similarity using cosine, Jaccard.
  • Corpus streaming — processes huge corpora in chunks (memory-efficient).
  • Phrase detection (n-gram models).
  • Random projections (dimension reduction).
  • Loading and saving pre-trained models.

3. Example code (Word2Vec)

from gensim.models import Word2Vec

sentences = [
    ["I", "love", "NLP"],
    ["NLP", "is", "fun"],
    ["I", "study", "machine", "learning"]
]

# Train Word2Vec model
model = Word2Vec(sentences, vector_size=100, window=5, min_count=1, workers=4)

# Get vector for a word
print(model.wv["NLP"])

# Find most similar words to "NLP"
print(model.wv.most_similar("NLP"))

4. Example for LDA topic modeling

from gensim import corpora
from gensim.models import LdaModel

texts = [
    ["bank", "money", "loan", "interest"],
    ["river", "water", "bank", "flow"],
    ["company", "profit", "market", "share"]
]

dictionary = corpora.Dictionary(texts)
corpus = [dictionary.doc2bow(text) for text in texts]

lda = LdaModel(corpus=corpus, id2word=dictionary, num_topics=2, passes=10)

# Print top words for each topic
for idx, topic in lda.print_topics(-1):
    print(f"Topic {idx}: {topic}")

5. Advantages of Gensim

  • Memory-efficient (streaming algorithms).
  • Easy to use.
  • Fast (Cython under the hood for some models).
  • Excellent for embeddings and topic modeling.
  • Pre-trained models available.

6. Limitations

  • NOT a general-purpose NLP library (no NER, parsing).
  • Less suitable for syntactic / morphological tasks.
  • API can change between versions.

Key points to remember

  • Gensim = library for topic modeling, embeddings, similarity.
  • Main features: LDA, LSI, Word2Vec, Doc2Vec, FastText, TF-IDF.
  • Memory-efficient streaming.
  • Not for general NLP (no NER, parsing) — use NLTK / spaCy for that.

Q6 — NLTK Python Tokenization Code

Asked 1 time(Nov'23 — 7 marks)
How this question appeared in past papers:

(Nov'23, 7 marks) Write Python code using NLTK to split text into tokens using whitespace, punctuation-based and default tokenization methods.

Answer

Python Code

import nltk
from nltk.tokenize import (
    word_tokenize,           # default tokenizer
    WhitespaceTokenizer,     # splits only on whitespace
    WordPunctTokenizer       # punctuation-based
)

# Download required NLTK data (only first time)
nltk.download('punkt')

text = "Hello, World! Welcome to NLP. Let's learn tokenization."

# 1. DEFAULT tokenization (uses Treebank tokenizer under the hood)
default_tokens = word_tokenize(text)
print("Default tokens:", default_tokens)
# Output: ['Hello', ',', 'World', '!', 'Welcome', 'to', 'NLP', '.', 'Let', "'s", 'learn', 'tokenization', '.']

# 2. WHITESPACE tokenization
whitespace_tokenizer = WhitespaceTokenizer()
ws_tokens = whitespace_tokenizer.tokenize(text)
print("Whitespace tokens:", ws_tokens)
# Output: ['Hello,', 'World!', 'Welcome', 'to', 'NLP.', "Let's", 'learn', 'tokenization.']

# 3. PUNCTUATION-BASED tokenization
punct_tokenizer = WordPunctTokenizer()
punct_tokens = punct_tokenizer.tokenize(text)
print("Punctuation-based tokens:", punct_tokens)
# Output: ['Hello', ',', 'World', '!', 'Welcome', 'to', 'NLP', '.', 'Let', "'", 's', 'learn', 'tokenization', '.']

Explanation of differences

MethodHow it splits"Let's"
Default (Treebank)Smart rules; separates punctuation['Let', "'s"]
WhitespaceOnly on spaces/tabs/newlines["Let's"]
Punctuation-basedSplits on every non-alphanumeric['Let', "'", 's']

Key points to remember

  • NLTK provides multiple tokenizers — choose based on need.
  • Default = word_tokenize() (Treebank).
  • Whitespace = splits only on spaces.
  • Punctuation-based = WordPunctTokenizer; splits on every punctuation.
