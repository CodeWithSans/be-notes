Preliminary Concepts (Read Once)

Imagine teaching a dog new tricks. You don't show the dog all possible actions; instead, when the dog does something good (sits when told), you give a TREAT (reward). When it does something bad, you don't reward it. Over time, the dog learns by trial and error which actions lead to treats. That's exactly how Reinforcement Learning works — an AGENT learns by interacting with an ENVIRONMENT, getting REWARDS for good actions and figuring out a strategy that maximizes long-term reward.

1. What is Reinforcement Learning?

Reinforcement Learning (RL) is a type of machine learning where an agent learns to make decisions by interacting with an environment, receiving rewards for good actions and penalties for bad ones, with the goal of maximizing the total reward over time.

3. Five basic elements of RL

  • Agent — the learner / decision-maker.
  • Environment — the world the agent interacts with.
  • State (s) — current situation of the agent.
  • Action (a) — what the agent can do.
  • Reward (r) — feedback from the environment after each action.

Q1 — Markov Decision Process (MDP)

Asked 4 times(May'23, Nov'23, May'25, Nov'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23, 6 marks) Explain Markov Decision Process with Markov property.

(Nov'23, 6 marks) Explain Markov decision process.

(May'25, 6 marks) Explain Markov Decision Process.

(Nov'25, 5 marks) What is a Markov Decision Process (MDP)? Define its components.

Answer
A grandmaster chess player doesn't need to remember every single move from the start of the game — looking at the CURRENT board position alone is enough to plan the next move. The current position fully describes the game state. This idea — "the FUTURE depends only on the PRESENT, not on the past" — is called the Markov Property, and decisions made in such systems are modeled as a Markov Decision Process.

1. Definition

A Markov Decision Process (MDP) is a mathematical framework used to model decision-making in situations where outcomes are partly random and partly controlled by the agent. It is the foundation of reinforcement learning.

2. The Markov Property

"The future depends ONLY on the present state, NOT on the past history."

P(st+1 | st, at, st-1, at-1, …) = P(st+1 | st, at)

In simple words: to decide what happens next, we only need to know the CURRENT state and CURRENT action — past history doesn't matter.

3. Components of an MDP (5-tuple)

An MDP is formally a tuple (S, A, P, R, γ):

  • S — Set of states. All possible situations the agent can be in. Example: chess positions.
  • A — Set of actions. All possible moves the agent can take. Example: moving a chess piece.
  • P — Transition probability function. P(s' | s, a) = probability of moving to state s' if action a is taken in state s.
  • R — Reward function. R(s, a) = reward received for taking action a in state s.
  • γ — Discount factor. γ ∈ [0, 1]; how much we value future rewards vs immediate. γ near 1 = far-sighted; γ near 0 = short-sighted.

4. Key concepts in MDP

  • Policy (π): a mapping from states to actions. π(s) = action to take in state s.
  • Value function (Vπ(s)): expected total discounted reward starting from state s and following policy π.
  • Action-value function (Qπ(s, a)): expected reward starting from s, taking action a, then following π.
  • Optimal policy (π*): the policy that maximizes total reward.

5. Bellman equation

V(s) = maxa [ R(s, a) + γ · Σs' P(s' | s, a) · V(s') ]

The value of a state = best immediate reward + discounted value of likely next states.

6. Diagram (text)

                action a
   state s ──────────────►  state s'    (with probability P(s' | s, a))
                │
                │  reward r = R(s, a)
                ▼
           Agent observes r, s'
           and decides next action a'

7. Example — simple grid world

  • States: each cell of a 4×4 grid.
  • Actions: up, down, left, right.
  • Rewards: +10 for reaching the goal; −1 for each step; −100 for falling into a hole.
  • Transitions: 80% you go in chosen direction, 10% slip left, 10% slip right (stochastic).
  • Discount factor γ = 0.9 (slightly far-sighted).
  • Solution: find policy π that maximizes expected total reward.

8. Applications of MDP

  • Robotics (path planning, motor control).
  • Game-playing AI (Chess, Go, Atari).
  • Autonomous driving.
  • Inventory and resource management.
  • Healthcare treatment optimization.
  • Recommendation systems.

Key points to remember

  • MDP = (S, A, P, R, γ) — 5-tuple.
  • Markov property: future depends ONLY on current state, not past.
  • Policy π(s) = action to take in each state.
  • Goal: find optimal policy π* that maximizes total expected reward.
  • Bellman equation: V(s) = maxa[R(s,a) + γ Σ P(s'|s,a) V(s')].
  • γ = discount factor (0 to 1); controls how much we value future rewards.

Q2 — Dynamic Programming for Reinforcement Learning

Asked 5 times(May'23, Nov'23, May'24, May'25, Nov'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23, 6 marks) Explain in detail Dynamic programming algorithms for reinforcement learning.

(Nov'23, 6 marks) Explain dynamic programming algorithms for reinforcement learning.

(May'24, 5 marks) Explain in detail Dynamic programming algorithms for reinforcement learning.

(May'25, 5 marks) Explain Dynamic programming algorithm for reinforcement learning.

(Nov'25, 6 marks) What is dynamic programming in RL? How does it differ from traditional DP?

Answer

1. What is Dynamic Programming (DP) in RL?

Dynamic Programming in RL refers to a class of algorithms that compute the optimal policy and value function of an MDP by breaking down the problem into smaller sub-problems and solving them recursively using the Bellman equations. It REQUIRES complete knowledge of the MDP (transition probabilities P and rewards R).

2. Two main DP algorithms in RL

(a) Policy Iteration

Repeatedly evaluates a policy and improves it until it converges.

  1. Initialize a policy π randomly.
  2. Policy Evaluation: compute Vπ(s) for all states by repeatedly applying: 
    V(s) = Σ P(s' | s, π(s)) · [R + γ V(s')]
    until convergence.
  3. Policy Improvement: for each state s, choose the action that maximizes the Bellman expectation: 
    π(s) = argmaxa [Σ P(s' | s, a) · (R + γ V(s'))]
  4. Repeat steps 2–3 until policy stops changing.

(b) Value Iteration

Combines policy evaluation and improvement in one update.

  1. Initialize V(s) = 0 for all states.
  2. Repeat until convergence: 
    V(s) ← maxa [Σ P(s' | s, a) · (R + γ V(s'))]
  3. Once V converges, extract the optimal policy: 
    π*(s) = argmaxa [Σ P(s' | s, a) · (R + γ V(s'))]

3. Comparison: Policy Iteration vs Value Iteration

AspectPolicy IterationValue Iteration
Update styleFull evaluation, then one improvementOne-step improvement each iteration
Convergence speedFaster in iterations (fewer)Faster per iteration (cheaper)
MemoryStores V and πStores only V

4. How RL's DP differs from traditional (CS) Dynamic Programming

AspectTraditional DP (CS)DP in Reinforcement Learning
Problem domainOptimization (shortest path, knapsack, etc.)Sequential decision making in MDPs
Recurrence typeBellman recursion on indicesBellman equation over states
Deterministic vs StochasticUsually deterministicStochastic (transition probabilities)
Knowledge requiredProblem definitionFull MDP (states, actions, P, R, γ) — model-based
OutputOptimal value / decision pathOptimal policy + value function
Iterative processBottom-up table fillIterative policy/value updates until convergence

5. Limitations of DP in RL

  • Needs FULL knowledge of MDP (P and R) — not realistic for most real problems.
  • Computational cost grows quickly with state-space size ("curse of dimensionality").
  • Not suitable for very large or continuous state spaces.
  • For unknown MDPs, we use model-free RL (Q-learning, Monte Carlo, TD).

6. Applications

  • Small grid worlds, gambling problems, simple games.
  • Inventory management with known demand distributions.
  • Sometimes used as a sub-routine in more complex algorithms (e.g., AlphaZero-style planning).

Key points to remember

  • DP in RL = Policy Iteration + Value Iteration; solves MDPs using Bellman equations.
  • Policy Iteration: alternate evaluation and improvement until convergence.
  • Value Iteration: iteratively apply Bellman optimality update on V.
  • Requires FULL MDP knowledge (model-based).
  • Differs from traditional DP: stochastic, sequential, uses Bellman over states.
  • Limitations: curse of dimensionality, needs known model.

Q3 — Q-Learning and Deep Q-Networks

Asked 5 times(May'23, Nov'23, May'24, May'25, Nov'25 — usually 6 marks)
How this question appeared in past papers:

(May'23, 6 marks) Write Short Note on Q Learning and Deep Q-Networks.

(Nov'23, 6 marks) Write Short Note on Q Learning and Deep Q-Networks.

(May'24, 6 marks) Write short note on Deep Q-Learning.

(May'25, 6 marks) Explain the process of Deep Q-learning.

(Nov'25, 6 marks) What is Q-learning? How does it differ from other RL algorithms?

Answer
Imagine a child learning a video game without instructions. The child tries random moves; sometimes they get points (reward), sometimes they lose. Over time, the child builds a mental table: "If I'm in this game-screen, and I press THIS button, I usually do well." This mental table is the Q-table — Q stands for QUALITY of an action in a state.

Part A — Q-Learning

1. Definition

Q-Learning is a popular model-free, off-policy reinforcement learning algorithm. It learns the optimal action-value function Q*(s, a) — the expected total future reward of taking action a in state s and then following the best policy.

2. Q-table

  • A table with rows = states, columns = actions, entries = Q-values.
  • Initially all Q-values are 0 (or random).
  • Updated as the agent interacts with the environment.

3. Q-Learning update rule

Q(s, a) ← Q(s, a) + α · [ r + γ · maxa' Q(s', a') − Q(s, a) ]
  • α = learning rate (e.g., 0.1)
  • γ = discount factor (e.g., 0.9)
  • r = immediate reward received
  • maxa' Q(s', a') = best future Q-value from the next state s'

4. Algorithm

  1. Initialize Q(s, a) = 0 for all s, a.
  2. For each episode:
    • Start in some state s.
    • Choose action a (often using ε-greedy: random with probability ε, best Q-value otherwise).
    • Take action; observe reward r and new state s'.
    • Update Q(s, a) using the formula above.
    • s ← s'.
    • Repeat until episode ends.
  3. Repeat for many episodes until Q-values converge.
  4. The optimal policy is π*(s) = argmaxa Q*(s, a).

5. Why Q-Learning is special (differs from other RL)

  • Model-free: doesn't need to know transition probabilities or rewards in advance.
  • Off-policy: learns the optimal policy even while following a different (exploratory) policy.
  • Value-based: learns value of actions (not policy directly).
  • Guaranteed convergence (to Q*) under certain conditions.
  • Other RL types: policy gradient (REINFORCE, PPO), actor-critic, model-based.

Part B — Deep Q-Networks (DQN)

6. Why we need DQN

  • Q-table works fine for small state spaces, but BLOWS UP for large/continuous state spaces.
  • Example: Atari games — millions of possible pixel states.
  • Solution: APPROXIMATE Q(s, a) with a neural network.

7. What is DQN?

A Deep Q-Network (DQN) uses a deep neural network to approximate Q(s, a). The network takes the state as input and outputs Q-values for all possible actions.

8. DQN architecture

   State (e.g., game frame) → CNN layers → FC layers → Q-values for each action
                                                          ─────────────────
                                                          [Q(s, a₁), Q(s, a₂), …]

9. Key innovations of DQN (DeepMind 2013/2015)

  • Experience replay: store past experiences in a buffer; randomly sample batches for training. Breaks temporal correlations.
  • Target network: use a SEPARATE slowly-updated network for computing target Q-values. Stabilizes training.
  • ε-greedy exploration: mix of random actions and greedy actions.

10. DQN training step

  1. Observe current state s.
  2. Choose action a using ε-greedy on Q-network.
  3. Execute action; observe reward r and next state s'.
  4. Store (s, a, r, s') in experience replay buffer.
  5. Sample random mini-batch from buffer.
  6. Compute target: y = r + γ · maxa' Qtarget(s', a').
  7. Compute loss: L = (y − Q(s, a))².
  8. Backpropagate and update Q-network weights.
  9. Periodically update target network: Qtarget ← Q.

11. Famous applications of DQN

  • Atari game playing (DeepMind 2015) — superhuman performance on many Atari games.
  • AlphaGo (combined with policy networks and tree search).
  • Robotics manipulation and control.
  • Resource allocation in data centres.
  • Autonomous driving sub-tasks.

Key points to remember

  • Q-Learning = model-free, off-policy, value-based RL.
  • Updates Q(s, a) using: Q ← Q + α[r + γ max Q(s', a') − Q].
  • DQN = neural network approximating Q-values for large/continuous state spaces.
  • Innovations: experience replay + target network + ε-greedy.
  • Famous for: superhuman Atari game playing.

Q4 — Reinforcement Learning for Tic-Tac-Toe

Asked 4 times(May'23, Nov'23, May'25, Nov'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23, 5 marks) Explain Simple reinforcement learning for Tic-Tac-Toe.

(Nov'23, 5 marks) Explain Simple reinforcement learning for Tic-Tac-Toe.

(May'25, 6 marks) Explain reinforcement learning for Tic-Tac-Toe game.

(Nov'25, 5 marks) How can the game of Tic-Tac-Toe be formulated as a reinforcement learning problem?

Answer
Two kids play Tic-Tac-Toe many times. Initially they make random moves. When kid X wins, they remember: "When the board looked like THIS, I made THIS move and won — let me try this again next time." After many games, they figure out which moves lead to wins, which lead to draws, which lead to losses. That's reinforcement learning for Tic-Tac-Toe.

1. Tic-Tac-Toe as an MDP

  • States (S): all possible board configurations (3×3 grid with X, O, or empty in each cell). About 5,478 reachable states.
  • Actions (A): in each state, the legal moves are the empty cells where the current player can place their mark.
  • Transitions (P): deterministic for the player's own move; stochastic from the player's view if opponent's strategy is unknown.
  • Rewards (R):
    • +1 for winning.
    • −1 for losing.
    • 0 for a draw or intermediate move.
  • Discount factor (γ): usually 1 (episodic task — game eventually ends).

2. Simple RL algorithm for Tic-Tac-Toe (value-based)

  1. Maintain a value table V(s) for every possible board state.
  2. Initialize:
    • For winning states: V = 1.
    • For losing states: V = 0.
    • For draws and intermediate states: V = 0.5.
  3. During a game, at each turn:
    • Most of the time, play GREEDY: pick the move leading to the state with the highest V (exploitation).
    • Occasionally, pick a RANDOM move (exploration) — with probability ε.
  4. After each non-random (greedy) move, update the value of the previous state toward the value of the new state: 
    V(s_t) ← V(s_t) + α · [V(s_{t+1}) − V(s_t)]
  5. Repeat for thousands of games until V converges.
  6. The greedy policy on the converged V is the optimal strategy.

3. Why this works

  • The value of winning states "propagates backward" through V updates.
  • States leading to wins get high V; states leading to losses get low V.
  • Eventually V tells the agent which move is best from any given board.
  • This is a TEMPORAL DIFFERENCE (TD) learning approach — a simple form of model-free RL.

4. Exploration vs Exploitation in Tic-Tac-Toe

  • Exploitation: always pick the move with the highest V → may miss better options.
  • Exploration: occasionally pick a random move → discovers new strategies.
  • Balance using ε-greedy (e.g., ε = 0.1 → 10% random moves).

5. Optional improvements

  • Use Q-learning (with Q-table over (state, action) pairs) instead of V-table.
  • Train two RL agents against each other (self-play) for better learning.
  • Use a small neural network instead of a table for larger games like Connect Four.

Key points to remember

  • Tic-Tac-Toe as MDP: states = board configs; actions = legal moves; rewards = +1/0/−1.
  • Use a value table V(s); update by TD: V(s) ← V(s) + α[V(s') − V(s)].
  • Use ε-greedy: mostly greedy, sometimes random for exploration.
  • Train over many games until V converges, then play greedy.
  • This is a classic introductory RL example.

Q5 — Deep Reinforcement Learning

Asked 3 times(May'23, Nov'23, May'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23, 5 marks) What is deep reinforcement learning? Explain in detail.

(Nov'23, 6 marks) What is deep reinforcement learning? Explain in detail.

(May'25, 6 marks) Explain deep reinforcement learning.

Answer

1. Definition

Deep Reinforcement Learning (Deep RL) is a combination of Reinforcement Learning (decision-making via trial and error) with Deep Learning (function approximation using neural networks). It uses deep neural networks to approximate the value function, policy, or model in an RL algorithm.

2. Why Deep RL?

  • Traditional RL (Q-table, V-table) works only for small, discrete state spaces.
  • Real-world problems have HUGE or continuous state spaces (images, video, sensor data).
  • Deep neural networks can approximate complex value / policy functions.
  • Lets RL scale to large problems like video games, robotics, autonomous driving.

3. Building blocks of Deep RL

  • Value-based: use deep network to approximate Q-function (e.g., DQN).
  • Policy-based: use deep network to directly output a probability distribution over actions (REINFORCE, Policy Gradient).
  • Actor-Critic: uses two networks — actor (policy) and critic (value function). Examples: A2C, A3C, PPO, DDPG.
  • Model-based: learn a deep model of the environment dynamics.

4. How Deep RL works (general flow)

  1. Agent observes state s (e.g., a game-frame image).
  2. Neural network processes s to produce action a (or Q-values, or policy probabilities).
  3. Agent takes action; receives reward r and new state s'.
  4. Loss is computed based on RL objective (Bellman error, policy gradient, etc.).
  5. Backpropagation updates the network weights.
  6. Repeat over millions of interactions until policy is good.

5. Famous Deep RL milestones

  • DQN (2015): superhuman performance on Atari games (DeepMind).
  • AlphaGo (2016): beat world champion Go player.
  • AlphaZero (2017): learned chess, shogi, Go from scratch via self-play.
  • OpenAI Five (2019): beat professional Dota 2 players.
  • AlphaStar (2019): beat top StarCraft II players.
  • ChatGPT (RLHF): uses reinforcement learning with human feedback to refine language models.

6. Applications of Deep RL

  • Video and board games (Atari, Go, Chess, Dota, StarCraft).
  • Robotics (grasping, walking, manipulation).
  • Autonomous driving (lane changing, navigation).
  • Recommendation systems.
  • Healthcare (treatment optimization).
  • Finance (algorithmic trading, portfolio management).
  • Industrial control (data centre cooling, manufacturing).
  • LLM fine-tuning via RLHF (Reinforcement Learning with Human Feedback).

7. Challenges of Deep RL

  • Sample inefficient — needs millions of interactions.
  • Training instability and sensitivity to hyperparameters.
  • Sparse reward problem.
  • Hard to debug and reproduce.
  • Computationally expensive.

Key points to remember

  • Deep RL = RL + Deep Neural Networks.
  • Neural network approximates Q, policy, or model.
  • Categories: value-based (DQN), policy-based (REINFORCE), actor-critic, model-based.
  • Famous: DQN, AlphaGo, AlphaZero, OpenAI Five.
  • Applications: games, robotics, self-driving, LLM training (RLHF).

Q6 — Challenges of Reinforcement Learning

Asked 3 times(May'23, Nov'23, May'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23, 6 marks) What are the challenges of reinforcement learning? Explain any four in detail.

(Nov'23, 5 marks) What are the challenges of reinforcement learning? Explain any four in detail.

(May'25, 5 marks) What are the challenges of reinforcement learning?

Answer

The 7 main challenges of Reinforcement Learning

(1) Exploration vs Exploitation trade-off

  • Agent must EXPLORE to find new strategies, but also EXPLOIT known good strategies.
  • Too much exploration → wasted time on bad moves.
  • Too much exploitation → stuck in a local optimum.
  • Balancing this is tricky (ε-greedy, UCB, Thompson sampling).

(2) Sparse and delayed rewards

  • In many problems, rewards come only at the end (win/lose in chess, +1 for goal in soccer).
  • Hard for the agent to figure out WHICH move actually caused the reward.
  • This is called the credit assignment problem.

(3) Sample inefficiency

  • RL needs MANY interactions (millions) to learn a good policy.
  • Real-world data collection (e.g., robot in physical environment) is expensive and slow.
  • Sample efficiency is a major research bottleneck.

(4) Large state/action space (curse of dimensionality)

  • State and action spaces explode in real-world problems (e.g., images, continuous control).
  • Q-tables become infeasible.
  • Function approximation (neural networks) helps but introduces training instability.

(5) Training instability

  • Deep RL training oscillates, diverges, or collapses.
  • Small changes in hyperparameters dramatically affect performance.
  • Techniques: experience replay, target networks, gradient clipping help, but don't fully solve.

(6) Non-stationarity / Partial observability

  • Environment can change over time, or agent doesn't see the full state (POMDP).
  • Multi-agent settings make the environment non-stationary (other agents change strategies).
  • Requires RNNs or memory-augmented networks.

(7) Reward hacking and safety

  • Agent may find unintended ways to maximize reward (gaming the reward function).
  • Hard to define a good reward function (reward shaping).
  • Safety: an RL agent may take dangerous actions during exploration in a real environment.

Other challenges

  • Difficulty in evaluation — no clear single metric.
  • Computationally expensive training.
  • Transfer learning is hard — policies don't generalize well across environments.
  • Reproducibility issues — random seeds dramatically affect results.

Key points to remember

  • 7 main challenges: exploration-exploitation, sparse rewards, sample inefficiency, large state space, training instability, partial observability, reward hacking.
  • Credit assignment problem = figuring out which action caused the reward.
  • Sample inefficiency is a major real-world bottleneck.
  • Safety and reward shaping are critical for real deployment.

Q7 — RL Basics: State, Action, Reward, Policy

Asked 1 time(Nov'25 — 6 marks)
How this question appeared in past papers:

(Nov'25, 6 marks) Define the terms: state, action, reward, and policy in the context of Reinforcement Learning (RL).

Answer

1. State (s)

  • A complete description of the situation the agent is currently in.
  • Contains all the information the agent needs to make a decision (under Markov property).
  • Example: in chess, the state is the current board position. In a video game, the state could be the current screen image.
  • The set of all possible states is denoted S.

2. Action (a)

  • A choice the agent makes that affects the environment.
  • The set of all possible actions is denoted A (may depend on the state).
  • Actions can be DISCRETE (e.g., move left/right) or CONTINUOUS (e.g., steering angle).
  • Example: in chess, an action is a legal move. In driving, an action is steering + acceleration.

3. Reward (r)

  • A scalar feedback signal from the environment after the agent takes an action.
  • Indicates how good the action was in that state.
  • Positive reward = good action; negative reward = bad action.
  • Agent's goal is to maximize the CUMULATIVE reward over time.
  • Example: +10 for scoring a goal, −1 for losing a piece, −0.01 for each time step (energy cost).

4. Policy (π)

  • A strategy that tells the agent what action to take in each state.
  • Can be:
    • Deterministic: π(s) = a (always the same action in state s).
    • Stochastic: π(a | s) = probability of taking action a in state s.
  • The goal of RL is to find the OPTIMAL policy π* that maximizes total expected reward.

5. Other related terms

  • Episode: a complete sequence from start state to terminal state.
  • Return (G): total accumulated reward in an episode.
  • Value function V(s): expected return from state s under policy π.
  • Q-function Q(s, a): expected return from taking action a in state s and then following π.
  • Discount factor γ: [0, 1] — how much we value future rewards.

Key points to remember

  • State (s) = current situation; Action (a) = what agent does; Reward (r) = feedback; Policy (π) = strategy.
  • Agent's goal: find optimal policy π* maximizing total reward.
  • Policy can be deterministic π(s) = a or stochastic π(a|s).
  • Value V(s) and Q(s, a) measure expected return.

Q8 — Types of RL / Active vs Passive RL

Asked 2 times(May'24 — 6 + 5 marks)
How this question appeared in past papers:

(May'24, 6 marks) What are different types of Reinforcement Learning? Explain in brief.

(May'24, 5 marks) Compare Active and Passive Reinforcement Learning.

Answer

Part A — Types of Reinforcement Learning

(1) Based on knowledge of environment

  • Model-based RL: agent knows or learns the model (transition probabilities, rewards). Uses planning (DP).
  • Model-free RL: agent does NOT know the model; learns directly from experience. Examples: Q-Learning, SARSA.

(2) Based on what is learned

  • Value-based: learn value function V or Q. Policy is implicit (greedy). Examples: Q-Learning, DQN.
  • Policy-based: learn policy directly. Examples: REINFORCE, PPO.
  • Actor-Critic: combine both — actor learns policy, critic learns value. Examples: A2C, A3C, DDPG.

(3) Based on policy used for exploration

  • On-policy: learn the policy that is being used to explore. Example: SARSA.
  • Off-policy: learn the optimal policy while following a different (exploratory) policy. Example: Q-Learning.

(4) Based on agent's involvement (Active vs Passive)

  • Active RL: agent chooses its own actions, exploring the environment.
  • Passive RL: agent's policy is FIXED; it just observes the rewards and evaluates the given policy.

Q9 — Characteristics / Advantages-Disadvantages of RL

Asked 2 times(May'24 — 6 marks each)
How this question appeared in past papers:

(May'24, 6 marks) What is Reinforcement Learning? State and explain its advantages and disadvantages.

(May'24, 6 marks) What are different characteristics of Reinforcement Learning?

Answer

1. What is Reinforcement Learning?

RL is a type of machine learning where an agent learns to make sequential decisions by interacting with an environment, receiving rewards for good actions and penalties for bad ones, with the goal of maximizing cumulative reward.

2. Characteristics of RL

  • Trial-and-error learning: agent learns by experimenting, not from labeled data.
  • Sequential decision making: each action affects future states and rewards.
  • Delayed rewards: reward may come long after the action.
  • Goal-directed behavior: agent aims to maximize a cumulative reward.
  • Exploration vs Exploitation trade-off: must balance trying new vs sticking to known good.
  • No supervisor: no teacher providing correct actions.
  • Time matters: states are visited sequentially.
  • Feedback is evaluative (telling how good an action was), NOT instructive (which action was correct).

3. Advantages of RL

  • Can solve complex sequential decision problems with no human-provided labels.
  • Learns directly from experience — no need for explicit programming.
  • Can outperform humans (Go, Chess, video games).
  • Adapts to changing environments.
  • Discovers novel strategies not designed by humans.
  • Generalizes to new but similar problems.
  • No need for a pre-existing dataset (generates own data via interaction).

4. Disadvantages of RL

  • Needs LOTS of data / interactions to learn (sample-inefficient).
  • Training is slow and computationally expensive.
  • Sensitive to hyperparameters and reward design.
  • Sparse/delayed rewards make credit assignment hard.
  • Training instability — small changes break performance.
  • Hard to debug and reproduce results.
  • Safety issues — bad actions during exploration may be dangerous in the real world.
  • Difficult to apply in real-world due to need for simulators or huge data.

Key points to remember

  • RL = learning by trial-and-error from environment rewards.
  • Characteristics: sequential, delayed rewards, evaluative feedback, exploration-exploitation.
  • Advantages: powerful, no labels needed, can exceed human performance.
  • Disadvantages: sample-inefficient, unstable, hard to debug, safety concerns.

Q10 — Deep Recurrent Q-Networks (DQRN)

Asked 1 time(Nov'25 — 6 marks)
How this question appeared in past papers:

(Nov'25, 6 marks) How does the recurrent layer in a Deep Recurrent Q-Networks (DQRN) help in decision-making over sequences?

Answer

1. What is DQRN?

A Deep Recurrent Q-Network (DRQN / DQRN) is an extension of Deep Q-Networks (DQN) that replaces one fully-connected layer with a RECURRENT layer (usually LSTM or GRU). This allows the network to use information from PAST observations, not just the current state.

2. Why we need DQRN — limitation of plain DQN

  • Standard DQN assumes the current observation is ENOUGH (Markov property holds).
  • In many real-world problems, the current observation alone is NOT sufficient (partial observability / POMDP).
  • Example: in Atari "Pong" or "Breakout", a single frame doesn't show the BALL'S DIRECTION — you need a few previous frames.
  • The standard DQN trick is to stack 4 frames as input, but this is a hack with a fixed window.

3. How DQRN's recurrent layer helps

  • An LSTM/GRU layer maintains a hidden state that summarizes the history of all past observations.
  • This gives the agent a "memory" that persists across time steps.
  • The agent can use this memory to figure out latent variables (ball direction, opponent's pattern, etc.).
  • It is more flexible than the 4-frame stacking trick — can use ANY amount of history.
  • Particularly useful in environments where future state depends on a long sequence of past observations.

4. Architecture of DQRN

   Observation o_t → CNN layers → LSTM cell → FC layer → Q-values

   Hidden state h_{t-1} ────► LSTM ────► h_t (passed to next step)

5. Training process

  • Use experience replay just like DQN, but replay SEQUENCES (not single transitions).
  • The LSTM is unrolled over the sequence; backprop through time updates weights.
  • Hidden state is initialized at the start of each sequence (or carried over).

6. Benefits over standard DQN

  • Handles partial observability (POMDPs) gracefully.
  • No need to artificially stack frames as input.
  • Can learn long-range dependencies in observations.
  • Better performance in environments where memory matters.

7. Applications

  • Atari games with partial observability (e.g., flickering versions).
  • 3D first-person navigation tasks.
  • Robotic control with noisy or limited sensors.
  • Real-time strategy games where opponent strategy must be inferred from past moves.

Key points to remember

  • DQRN = DQN + recurrent layer (LSTM/GRU).
  • Recurrent layer adds MEMORY of past observations.
  • Handles partial observability (POMDPs).
  • Avoids the frame-stacking hack of standard DQN.
  • Useful when the optimal action depends on a LONG history.
