Preliminary Concepts (Read Once)

A normal neural network (CNN, MLP) looks at one image or one piece of data independently — it has no memory of what came before. But for sequential data like sentences, speech, stock prices, or video frames, the ORDER and CONTEXT matter — "The cat sat on the …" needs memory of earlier words to predict "mat". This is what an RNN does — it remembers past inputs while processing the current one.

1. What is "sequential data"?

  • Data where ORDER matters: text, speech, music, video, time series, DNA sequences.
  • Same word means different things in different positions ("bank of river" vs "money bank").
  • Future depends on past — needs MEMORY of previous inputs.

2. Why feed-forward networks (CNN, MLP) fail on sequences

  • They have NO memory — each input is processed independently.
  • They need a fixed-size input — but sequences vary in length.
  • They cannot share weights across time steps.

3. Solution: RNN (Recurrent Neural Network)

  • Adds a "loop" inside the network so that information from the previous time step is carried to the next.
  • Uses the SAME weights at every time step (parameter sharing across time).
  • Can handle variable-length sequences.

Q1 — Recurrent Neural Network (RNN): What, Why, Working, Types

Asked 6 times(May'23, Nov'23 twice, May'24, May'25, Nov'25 — usually 6 marks)
How this question appeared in past papers:

(May'23, 6 marks) What is RNN? What is need of RNN? Explain in brief about working of RNN.

(Nov'23, 6 marks) Explain Recurrent Neural Network with its architecture.

(Nov'23, 6 marks) Explain the types of Recurrent Neural Network.

(May'24, 6 marks) Explain RNN with its types.

(May'25, 5 marks) Explain in brief about working of RNN.

(Nov'25, 6 marks) List the types of Recurrent Neural Network and explain LSTM three gates.

Answer
A person reading a sentence remembers the words they have already read while reading the current word. This memory helps them understand the FULL context. An RNN is a neural network with a memory loop that lets it "remember" past inputs while processing the current one — just like a human reading a sentence.

1. Definition

A Recurrent Neural Network (RNN) is a type of neural network designed to process sequential data. It has a hidden state that carries information from one time step to the next, giving the network a form of memory.

2. Need / Why we need RNN

  • Many real-world problems involve sequences (text, speech, time series, video).
  • Standard neural networks have NO memory of previous inputs.
  • RNNs SHARE weights across time steps — fewer parameters for long sequences.
  • RNNs handle variable-length inputs and outputs (sentences of any length).
  • Allows the network to learn temporal patterns and dependencies.

3. Basic working of an RNN

At each time step t, the RNN receives an input xt and a hidden state ht-1 from the previous step. It produces a new hidden state ht and (optionally) an output yt.

ht = f(Wxh · xt + Whh · ht-1 + bh)
yt = g(Why · ht + by)
  • Wxh, Whh, Why are weight matrices — SHARED across all time steps.
  • f, g are activation functions (typically tanh and softmax).
  • h0 is initialized to zero or learned.

4. Folded vs Unfolded RNN diagram

Folded (compact view):              Unfolded (across time):

     ┌──→┐                          x₁    x₂    x₃    x₄
     │   │                          │     │     │     │
     │  ┌▼──┐                       ▼     ▼     ▼     ▼
   x ──►│ h │── y                  [h₁]─►[h₂]─►[h₃]─►[h₄]
        └───┘                       │     │     │     │
                                    ▼     ▼     ▼     ▼
                                   y₁    y₂    y₃    y₄

5. Types of RNN (based on input-output structure)

(a) One-to-One

  • Single input → single output.
  • Same as standard feed-forward network (no recurrence).
  • Example: image classification.

(b) One-to-Many

  • Single input → sequence of outputs.
  • Example: image captioning (image → sentence describing it).

(c) Many-to-One

  • Sequence of inputs → single output.
  • Example: sentiment analysis (sentence → positive/negative).

(d) Many-to-Many (same length)

  • Sequence of inputs → equal-length sequence of outputs.
  • Example: part-of-speech tagging (each word gets a tag).

(e) Many-to-Many (different lengths)

  • Sequence input → sequence output of different length.
  • Example: language translation (English sentence → French sentence).
  • Uses Encoder-Decoder architecture.

6. Training RNNs: Backpropagation Through Time (BPTT)

  • Unfold the RNN across time → it becomes a deep feed-forward network.
  • Apply standard backpropagation through this unfolded network.
  • Gradients flow backward through all the time steps.
  • Suffers from vanishing / exploding gradients for long sequences.

7. Variants of RNN

  • Vanilla RNN — basic version; struggles with long sequences.
  • LSTM (Long Short-Term Memory) — adds memory gates; handles long sequences.
  • GRU (Gated Recurrent Unit) — simpler than LSTM, similar performance.
  • Bidirectional RNN — runs forward AND backward; sees both past and future.
  • Deep RNN — stacks multiple RNN layers vertically.

8. Applications of RNN

  • Language modeling (predict next word).
  • Machine translation (English → French).
  • Speech recognition (audio → text).
  • Text generation (chatbots, GPT-like models historically).
  • Time-series prediction (stock prices, weather).
  • Music generation.
  • Video analysis (action recognition).

Key points to remember

  • RNN = neural network with memory loop, designed for sequences.
  • Uses same weights at every time step (parameter sharing).
  • ht = f(W·xt + U·ht-1 + b).
  • 5 types by input/output: 1-1, 1-many, many-1, many-many (same), many-many (different).
  • Trained using Backpropagation Through Time (BPTT).
  • Variants: LSTM, GRU, Bidirectional, Deep RNN.

Q2 — LSTM (Long Short-Term Memory)

Asked 5 times(May'23, Nov'23 twice, May'25, Nov'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23, 6 marks) How LSTM and Bidirectional LSTM works.

(Nov'23, 6 marks) Draw and explain architecture for Long Short-Term Memory (LSTM).

(Nov'23, 5 marks) Explain how the memory cell in the LSTM is implemented computationally.

(May'25, 6 marks) Explain the LSTM in RNN.

(Nov'25, 6 marks) Explain LSTM three gates.

Answer
A vanilla RNN's memory is like a leaky bucket — it forgets old information very quickly. LSTM adds three "gates" — like locks on the bucket — that decide what to remember, what to forget, and what to output. This lets it remember important things from MUCH earlier in the sequence.

1. Definition

LSTM (Long Short-Term Memory) is a special type of RNN designed to learn long-term dependencies in sequences. It uses a memory cell and three gates (forget, input, output) to control the flow of information.

2. Why we need LSTM (problem with vanilla RNN)

  • Vanilla RNNs suffer from vanishing/exploding gradient problems.
  • They cannot effectively learn dependencies more than 5–10 time steps apart.
  • LSTMs solve this with gating mechanisms that preserve gradients over long sequences.

3. LSTM cell components

  • Cell state (Ct) — the long-term memory; runs straight through the cell with minor modifications.
  • Hidden state (ht) — the short-term output passed to the next step.
  • Forget Gate (ft) — decides what to REMOVE from cell state.
  • Input Gate (it) — decides what NEW info to ADD to cell state.
  • Output Gate (ot) — decides what to OUTPUT from the cell state.

4. LSTM equations

ft = σ(Wf·[ht-1, xt] + bf)   # Forget gate
it = σ(Wi·[ht-1, xt] + bi)   # Input gate
Ĉt = tanh(WC·[ht-1, xt] + bC) # Candidate cell state
Ct = ft ⊙ Ct-1 + it ⊙ Ĉt   # New cell state
ot = σ(Wo·[ht-1, xt] + bo)   # Output gate
ht = ot ⊙ tanh(Ct)   # Hidden output

(⊙ = element-wise multiplication, σ = sigmoid)

5. Diagram of LSTM cell (text)

   h_{t-1}      x_t
        \      /
         \    /
          \  /
       +----------------+
       |  Forget Gate σ |── f_t ──┐
       +----------------+         ↓
       |  Input Gate σ  |── i_t ──┐
       +----------------+         ↓
       |  Candidate tanh|── Ĉ_t ──┐
       +----------------+         ↓
                              ╔══════╗
              C_{t-1} ─────► ╫  ⊙   ╬────► C_t
                              ╚══════╝
                                 │
                              ┌─tanh
                                 │
       +----------------+        ⊙────► h_t
       |  Output Gate σ |── o_t ─┘
       +----------------+

6. Working — step by step

  1. Forget Gate: looks at ht-1 and xt; outputs a number 0–1 for each cell state value. 0 = "forget completely", 1 = "keep completely".
  2. Input Gate: decides which values we will update.
  3. Candidate values (Ĉt): a tanh layer creates new candidate values to add.
  4. Update cell state: multiply old state by forget gate, then add (input gate × candidate).
  5. Output Gate: decides what part of the cell state to expose as ht.
  6. Pass ht and Ct to the next time step.

7. Bidirectional LSTM (BiLSTM)

  • Two LSTMs: one processes the sequence LEFT-TO-RIGHT (forward), other RIGHT-TO-LEFT (backward).
  • Their outputs are concatenated at each time step.
  • Allows the model to use BOTH past AND future context.
  • Used in speech recognition, machine translation, named-entity recognition.

8. Advantages of LSTM

  • Handles long-term dependencies effectively (100s of steps).
  • Avoids vanishing gradient problem.
  • Selectively remembers/forgets information using gates.
  • Widely used in NLP and time-series prediction.

9. Disadvantages

  • More parameters than vanilla RNN (slower training).
  • Computationally expensive for very long sequences.
  • Sequential nature limits parallelization (transformers are now preferred for many NLP tasks).

Key points to remember

  • LSTM = RNN with memory cell + 3 gates (forget, input, output).
  • Solves vanishing gradient problem in vanilla RNN.
  • Cell state Ct = long-term memory; hidden state ht = short-term output.
  • Forget gate = what to remove; Input gate = what to add; Output gate = what to expose.
  • Bidirectional LSTM = forward + backward LSTMs combined.

Q3 — Encoder-Decoder Sequence-to-Sequence Architecture

Asked 4 times(May'23, May'24, May'25, Nov'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23, 6 marks) Explain Encoder-Decoder Sequence to Sequence architecture with its application.

(May'24, 6 marks) Explain in brief Encoder Decoder architecture.

(May'25, 5 marks) Explain Encoder-Decoder RNN model.

(Nov'25, 6 marks) What is Encoder-Decoder architecture, and how does it work in sequence-to-sequence learning?

Answer
A translator listens to a full English sentence (compresses it into a mental summary called the "context"), then speaks the French translation word by word from that summary. The "listening" part is the ENCODER. The "speaking" part is the DECODER. The compressed mental summary is the CONTEXT VECTOR.

1. Definition

An Encoder-Decoder (Seq2Seq) architecture is a neural network framework that maps an INPUT sequence to an OUTPUT sequence, where the two sequences can be of DIFFERENT lengths. It uses two RNNs (or LSTMs): an encoder that compresses the input into a fixed-size vector, and a decoder that generates the output from this vector.

2. Architecture components

(a) Encoder

  • An RNN/LSTM/GRU that reads the input sequence ONE token at a time.
  • Updates its hidden state with every token.
  • The final hidden state summarizes the entire input → called the context vector (or thought vector).

(b) Context Vector

  • A fixed-size vector that encodes ALL information about the input sequence.
  • Bridge between encoder and decoder.

(c) Decoder

  • Another RNN/LSTM/GRU that takes the context vector as initial state.
  • Generates the output sequence ONE token at a time.
  • Each generated token is fed back as input for the next step.
  • Stops when a special END token is produced.

3. Diagram

  INPUT (English):  "I  am  a  student"
                     ↓   ↓   ↓   ↓
                  [E1]→[E2]→[E3]→[E4] ── final hidden state = context
                                                     ↓
                                              ┌──────┴──────┐
                                              │  CONTEXT     │
                                              └──────┬──────┘
                                                     ↓
                                      [D1]→[D2]→[D3]→[D4]
                                       ↓    ↓    ↓    ↓
                                     "Je  suis un  étudiant"
                                     OUTPUT (French)

4. Working step by step

  1. Tokenize input sequence (e.g., "I am a student" → [I, am, a, student]).
  2. Feed tokens one-by-one to the encoder; hidden state updates each step.
  3. After last input token, the encoder's final hidden state = CONTEXT VECTOR.
  4. Initialize the decoder with this context vector.
  5. Decoder generates first output token (e.g., "Je"); feeds it back as input.
  6. Repeat: generate next token, feed back, … until END token.

5. Limitations of basic encoder-decoder

  • The context vector is FIXED-SIZE → can't capture all information from very long inputs.
  • Decoder relies on a single summary → "information bottleneck".
  • Solution: Attention mechanism (lets decoder look at all encoder hidden states, not just the final one).

6. Applications

  • Machine translation (English → French, Hindi → English).
  • Text summarization (long article → short summary).
  • Speech recognition (audio → text).
  • Chatbots (user query → response).
  • Image captioning (image → text using CNN encoder + RNN decoder).
  • Video captioning (video sequence → text description).
  • Question answering systems.
  • Code generation from natural language descriptions.

Key points to remember

  • Encoder-Decoder = 2 RNNs that map input sequence to output sequence of different length.
  • Encoder reads input → produces context vector.
  • Decoder takes context vector → generates output one token at a time.
  • Bottleneck: fixed-size context vector. Solved by attention.
  • Applications: translation, summarization, chatbots, image captioning.

Q4 — Unfolding Computational Graphs

Asked 2 times(May'23, Nov'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23, 5 marks) Explain Unfolding computational graphs with example.

(Nov'25, 6 marks) How is the computational graph of an RNN different from that of a feedforward neural network?

Answer

1. Definition

Unfolding a computational graph means representing a recurrent (looped) network as a sequence of repeated copies of the same cell, one per time step. The CYCLE (loop) in the folded view is "unrolled" into a chain.

2. Why we unfold

  • Recurrent networks have loops → standard backpropagation doesn't work directly on loops.
  • Unfolding converts the recurrent network into an equivalent deep feed-forward network.
  • Then we can apply standard backpropagation → this is called Backpropagation Through Time (BPTT).
  • It also makes the computation easier to visualize and debug.

3. Folded vs Unfolded view

Folded (with loop):                Unfolded across 4 time steps:

     ┌──→──┐                      x₁    x₂    x₃    x₄
     │     │                       │     │     │     │
     │   ┌─▼─┐                     ▼     ▼     ▼     ▼
   x ───►│ h │─── y         h₀→[h₁]──►[h₂]──►[h₃]──►[h₄]
         └───┘                     │     │     │     │
                                   ▼     ▼     ▼     ▼
                                  y₁    y₂    y₃    y₄

4. Mathematical formulation

Folded form:

ht = f(ht-1, xt; θ)

Unfolded form (for 3 time steps):

h3 = f(f(f(h0, x1; θ), x2; θ), x3; θ)

Same parameters θ are SHARED across all time steps.

5. RNN vs Feed-forward computational graph

FeatureFeed-ForwardRNN (unfolded)
CyclesNone (DAG)Has loop in folded form
LayersDifferent at each levelSame cell repeated per time step
Parameter sharingNo (each layer has own weights)Yes (same weights at every step)
MemoryNoneHidden state carries context
Handles sequencesNo (fixed-size input)Yes (variable-length)
TrainingStandard backpropBPTT (Backpropagation Through Time)

Key points to remember

  • Unfolding = unrolling the recurrent loop into a chain across time steps.
  • Allows standard backpropagation to work (BPTT).
  • Same weights are SHARED at every time step.
  • RNN graph has loops; feed-forward graph is acyclic.

Q5 — Recurrent vs Recursive Neural Network

Asked 2 times(May'23, May'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23, 5 marks) Differentiate between Recurrent and Recursive Neural Network.

(May'25, 6 marks) Explain recursive neural network.

Answer

1. Recurrent Neural Network (RNN)

  • Processes data in a sequential / linear chain structure.
  • Hidden state passes from one time step to the next: ht = f(ht-1, xt).
  • Natural for sequences: text, speech, time series.

2. Recursive Neural Network

  • Processes data in a TREE structure (hierarchical).
  • Combines pairs of child nodes into a parent node using the same neural function recursively.
  • Natural for tree-structured data: parse trees of sentences, scene graphs, hierarchical clusters.
  • Example: combine "The" and "cat" → phrase node; combine phrase node and verb → sentence node.

3. Comparison table

FeatureRecurrent NNRecursive NN
Data structureLinear sequence (chain)Tree (hierarchical)
Hidden state flowTime step → next time stepChildren → parent node
ApplicationText, audio, time seriesParse trees, scene parsing
Computational graphLong chainTree
GeneralizationSpecial case of recursive (chain tree)General — handles arbitrary trees
TrainingBPTTBackpropagation through structure

4. Recursive NN — typical use

Sentence: "The cat sat"

Recursive NN combines pairs:

        Sentence
         /    \
        /      \
       /        \
      NP        VP
     /  \        |
    /    \       |
   The  cat     sat

Each internal node = neural combination of its children

5. Applications of Recursive NN

  • Sentiment analysis on parse trees (Stanford Sentiment Treebank).
  • Scene parsing in computer vision.
  • Compositional semantics in NLP.

Key points to remember

  • RNN = linear chain; Recursive NN = tree hierarchy.
  • RNN: ht from ht-1; Recursive: parent from children.
  • RNN for sequences (text, audio); Recursive for trees (parse trees).
  • RNN is a special case of Recursive NN (linear chain = degenerate tree).

Q6 — Challenges of Long-Term Dependencies

Asked 2 times(May'25, Nov'25 — usually 6 marks)
How this question appeared in past papers:

(May'25, 6 marks) What are the challenges of long term dependencies?

(Nov'25, 6 marks) Explain any Seven Challenges of Long-Term Dependencies.

Answer

1. What is "long-term dependency"?

In a sequence, sometimes the current output depends on information from MANY steps earlier. Example: in "I grew up in France ... [50 words later] ... I speak fluent French" — the model needs to remember "France" 50 steps ago to predict "French".

2. The 7 main challenges

(a) Vanishing Gradient Problem

  • When errors are backpropagated through many time steps, gradients become exponentially smaller.
  • Early layers learn very slowly or stop learning entirely.
  • Network fails to capture dependencies more than ~10 steps apart.

(b) Exploding Gradient Problem

  • Opposite of vanishing — gradients grow exponentially large.
  • Weights become NaN; training becomes unstable.
  • Solution: gradient clipping (cap gradient magnitude).

(c) Slow Learning

  • Even when training does happen, learning long dependencies takes many epochs.
  • The longer the dependency, the harder it is to learn.

(d) Memory Loss / Forgetting

  • Vanilla RNN's hidden state is overwritten at every step.
  • Old information gets lost / "diluted" quickly.

(e) Difficulty in Optimization

  • Long sequences create deep computational graphs.
  • Loss surface becomes very non-convex with many bad local minima.
  • Choosing the right learning rate is critical.

(f) High Computational Cost

  • BPTT requires storing all hidden states from the entire sequence.
  • Memory usage grows linearly with sequence length.
  • Training is slow for long sequences.

(g) Sequential Nature Prevents Parallelism

  • Each step depends on the previous → cannot parallelize across time steps.
  • Makes RNN slower than CNN on modern GPUs.
  • This is one reason Transformers (parallel) replaced RNNs in NLP.

3. Solutions to these challenges

  • LSTM and GRU — use gates to preserve gradients.
  • Gradient clipping — limits maximum gradient magnitude.
  • Better initialization — e.g. orthogonal weight init.
  • Skip / residual connections — give gradient a shortcut path.
  • Attention mechanisms — direct connections to past states.
  • Transformers — eliminate recurrence entirely.
  • Truncated BPTT — only backprop through a window of recent steps.

Key points to remember

  • 7 challenges: vanishing gradient, exploding gradient, slow learning, memory loss, hard optimization, high cost, no parallelism.
  • Vanishing gradient = main culprit for long-term dependency failure.
  • Solutions: LSTM/GRU, gradient clipping, attention, transformers, truncated BPTT.

Q8 — Bidirectional RNN: Limitations and Differences

Asked 1 time(Nov'25 — 6 marks)
How this question appeared in past papers:

(Nov'25, 6 marks) What are limitations of Bidirectional RNNs, and how do they differ from standard RNNs?

Answer

1. What is a Bidirectional RNN (BiRNN)?

A Bidirectional RNN uses TWO RNNs: one processes the sequence in the FORWARD direction (past → future), and another processes it BACKWARD (future → past). At each time step, the outputs from both directions are combined (usually concatenated).

2. Difference from standard RNN

FeatureStandard RNNBidirectional RNN
DirectionForward onlyBoth forward and backward
Context usedOnly pastBoth past and future
Number of hidden states1 per time step2 per time step (forward + backward)
Parameters1 set2 sets (twice as many)
Memory usageLower~2× higher
Suitable forOnline / real-time tasksOffline tasks (full sequence available)

3. Limitations of Bidirectional RNN

  1. Cannot be used for real-time / online tasks — requires the FULL sequence before processing (because backward direction needs future tokens).
  2. Twice the parameters and twice the memory — 2 RNNs instead of 1.
  3. Slower training and inference — both directions must be computed.
  4. Cannot work with streaming data — speech assistants, live translation, etc.
  5. Still suffers from vanishing gradient in vanilla form (use BiLSTM to fix this).
  6. Doubled risk of overfitting due to twice as many parameters.
  7. Higher latency — must wait for whole input before producing first output.

4. Where Bidirectional RNN is useful

  • Named-Entity Recognition (NER).
  • Part-of-Speech tagging.
  • Speech recognition (offline batch).
  • Machine translation (with encoder).
  • Sentence-level classification.

Key points to remember

  • BiRNN = forward RNN + backward RNN; uses both past and future context.
  • Standard RNN uses only past; BiRNN uses past + future.
  • Limitations: no real-time use, 2× parameters, slower, can overfit more.
  • Useful for offline tasks where full sequence is available.

Q9 — Echo State Network

Asked 1 time(Nov'25 — 6 marks)
How this question appeared in past papers:

(Nov'25, 6 marks) How does Echo State Network Differ from Traditional RNNs?

Answer

1. Definition

An Echo State Network (ESN) is a special type of RNN that uses a large, randomly initialized recurrent layer (called the reservoir) whose weights are NEVER updated. Only the output layer weights are trained.

2. Architecture

  • Input layer: feeds input into the reservoir.
  • Reservoir: a large, sparsely connected recurrent layer with random fixed weights.
  • Output layer: a simple linear layer; only this is trained (often using linear regression).

3. How ESN differs from Traditional RNN

FeatureTraditional RNNEcho State Network
Weight trainingAll weights trained via BPTTOnly output weights trained (reservoir is fixed random)
Training complexityExpensive (BPTT)Very fast (linear regression on output)
Hidden layer sizeSmall to mediumLarge (hundreds–thousands of neurons)
ConnectivityDenseSparse
Vanishing gradientSevere problemAvoided (no backprop through reservoir)
Suitable forComplex sequential learningTime-series prediction with quick training

4. Advantages of ESN

  • Very fast training (no BPTT needed).
  • Avoids vanishing/exploding gradients.
  • Good for time-series prediction and chaotic systems.

5. Limitations of ESN

  • Reservoir parameters are not optimized → may need to try many random initializations.
  • Less powerful than LSTM/GRU for complex tasks.
  • Performance highly depends on the random reservoir.

Key points to remember

  • ESN = RNN with FIXED random reservoir; only output layer trained.
  • Avoids BPTT and vanishing gradient.
  • Very fast to train but less powerful than LSTM.
  • Good for time-series, chaotic systems.

Q10 — Performance Metrics / Baseline Models / Implicit vs Explicit Memory

Asked 1 time each(May'24)
How this question appeared in past papers:

(May'24, 6 marks) WSN (Write Short Notes) on Performance Matrices.

(May'24, 6 marks) Compare implicit and explicit memory.

(May'24, 5 marks) What are default baseline models? Explain in brief.

Answer

Part A — Performance Metrics for Deep Learning Models

  • Accuracy: fraction of correct predictions. (correct / total)
  • Precision: TP / (TP + FP) — of all predicted positives, how many were actually positive.
  • Recall (Sensitivity): TP / (TP + FN) — of all actual positives, how many were caught.
  • F1 Score: 2 · (Precision · Recall) / (Precision + Recall) — harmonic mean.
  • Confusion matrix: table showing TP, FP, TN, FN for classification.
  • ROC-AUC: Area Under the Receiver Operating Characteristic curve.
  • MSE / MAE: Mean Squared/Absolute Error — for regression.
  • BLEU score: for translation tasks.
  • Perplexity: for language models.
  • Inference latency & throughput: speed metrics.

Part B — Default Baseline Models

When starting a deep learning project, a default baseline model is a simple, well-known model used as a starting point to benchmark performance. Examples:

  • For image classification: ResNet-50 or a pre-trained VGG16.
  • For text classification: a simple LSTM or BERT-base.
  • For tabular data: a small MLP.
  • For regression on time-series: ARIMA or simple LSTM.

Baseline helps determine if your custom model is actually improving over a simple, known-good model. Saves time and prevents reinventing the wheel.

Part C — Implicit vs Explicit Memory

AspectImplicit MemoryExplicit Memory
Where storedInside network weights / hidden statesSeparate external memory bank
AccessImplicit / automaticRead/write via learned addressing
CapacityLimited by hidden state sizeCan be arbitrarily large
Example architecturesRNN, LSTM, GRUNeural Turing Machine, Memory Networks, Transformers (with key-value memory)
Use caseShort sequences, statistical patternsTasks requiring large factual recall (QA, reasoning)
TrainingEnd-to-end backpropOften needs attention / addressing mechanism

Key points to remember

  • Metrics: Accuracy, Precision, Recall, F1, Confusion matrix, ROC-AUC, MSE.
  • Baseline models = simple known-good starting points (ResNet-50, BERT-base).
  • Implicit memory = stored in weights / hidden state (RNN).
  • Explicit memory = separate read/write memory (NTM, Memory Networks).

Must-remember formulas / facts

Vanilla RNN:  ht = tanh(Wx·xt + Wh·ht-1 + b)
LSTM: 3 gates (forget, input, output) + cell state Ct
Encoder-Decoder: Encoder makes context vector; Decoder generates output
Training: BPTT (Backpropagation Through Time)
Main problem: Vanishing gradient (solved by LSTM/GRU)

5 types of RNN by input-output

  1. One-to-One: image classification
  2. One-to-Many: image captioning
  3. Many-to-One: sentiment analysis
  4. Many-to-Many (same length): POS tagging
  5. Many-to-Many (different lengths): machine translation

One-liner to remember the whole chapter

RNN adds memory to neural networks for sequences; LSTM uses 3 gates (forget, input, output) to handle long sequences; Encoder-Decoder maps input sequence to output sequence of different length.
