Preliminary Concepts (Read Once)

A normal neural network treats an image as a flat list of numbers — it loses all the "shape" information (which pixel is next to which). A CNN keeps the image as a 2-D grid, and uses small "magnifying glasses" (called filters or kernels) that slide over the image looking for specific patterns — edges, corners, textures, faces, etc. This is exactly how the visual cortex of the human brain works.

1. What is a Convolutional Neural Network (CNN)?

A CNN is a type of deep neural network specially designed to process data with a grid-like structure, such as images. It uses three special types of layers — convolution, pooling, and fully-connected — to automatically learn useful features from raw pixel data.

2. Why CNN over a normal neural network for images?

  • Parameter sharing: the same small filter is used across the whole image, so the network has FAR fewer parameters than a fully-connected net.
  • Local connectivity: each neuron looks only at a small patch of the image (not the whole thing).
  • Translation invariance: a feature (e.g., an edge) is detected the same way no matter where it appears in the image.
  • Preserves spatial structure: doesn't flatten the image away.

3. The 4 key layer types in a CNN

  1. Convolution layer — slides filters across the image to detect features.
  2. Activation layer (usually ReLU) — adds non-linearity.
  3. Pooling layer — reduces the size of feature maps (down-sampling).
  4. Fully-connected layer (FC) — at the end, makes the final classification.

Q1 — Pooling Layer (with types and features)

Asked 6 times(May'23 twice, May'24, Nov'23, May'25 twice — usually 6 marks)
How this question appeared in past papers:

(May'23, 6 marks) Explain Pooling Layer with its need and different types.

(May'23, 6 marks) Explain all the features of pooling layer.

(May'24, 6 marks) Explain Pooling Layer with its different types.

(Nov'23, 6 marks) Explain Pooling layers and its types with examples.

(May'25 twice, 6 marks) Explain pooling layers and its type / Explain all the features of pooling layer.

Answer
After the convolution layer detects many small features (edges, corners), we get LOTS of feature maps — too many numbers. Pooling is like taking a high-resolution photo and shrinking it to a smaller thumbnail. We keep the most important parts and throw away the unnecessary details.

1. Definition

The Pooling layer is a layer in a CNN that down-samples (shrinks) the feature maps produced by the convolution layer. It does this by sliding a small window over the feature map and replacing each window with ONE summary value.

2. Need / Purpose of Pooling Layer

  • Reduces the spatial size of feature maps → fewer parameters → less computation.
  • Controls overfitting by reducing the number of features.
  • Provides translation invariance — small shifts in the input do not change the pooled output.
  • Increases the receptive field — each neuron in deeper layers "sees" a bigger region of the original image.
  • Speeds up training and inference by shrinking data size at each layer.

3. Features / Properties of Pooling Layer

  • Has NO learnable parameters (no weights or biases).
  • Operates independently on each channel of the input.
  • Defined by 3 hyperparameters: window size (e.g., 2×2), stride, and type (max / average / etc.).
  • Reduces both the width and height of the feature map (but keeps the depth / channels the same).
  • Typically placed AFTER convolution + ReLU.
  • Common output size after 2×2 pooling with stride 2: half the input width and half the input height.

4. Types of Pooling

(a) Max Pooling

  • For each window, takes the MAXIMUM value.
  • Captures the strongest activation in each region.
  • Most commonly used in practice (in AlexNet, VGG, ResNet).

(b) Average Pooling

  • For each window, takes the AVERAGE of all values.
  • Smooths the feature map.
  • Used in some older models (LeNet) and in Global Average Pooling at the end of modern CNNs.

(c) Min Pooling

  • For each window, takes the MINIMUM value.
  • Rarely used in practice.

(d) Global Pooling (Average / Max)

  • Window size = the entire feature map.
  • Reduces each feature map to a single number.
  • Used to replace fully-connected layers (e.g., in GoogLeNet, ResNet).

(e) Sum Pooling, L2 Pooling, Stochastic Pooling

  • Less common variants. Sum pooling = sum of values; L2 pooling = root mean square; stochastic = random sample weighted by activation.

5. Example — 2×2 Max Pooling with stride = 2

Input feature map (4×4):                  After 2×2 max pooling (stride 2):

   1   3   2   1                              max(1,3,4,2) = 4    max(2,1,1,0) = 2
   4   2   1   0                              max(0,5,1,2) = 5    max(2,3,8,4) = 8
   0   5   2   3
   1   2   8   4                              Result (2×2):
                                                  4   2
                                                  5   8

6. Pooling output size formula

Output size = ((Input size − Window size) / Stride) + 1

Example: Input = 32, Window = 2, Stride = 2 → Output = (32-2)/2 + 1 = 16.

Key points to remember

  • Pooling = down-sampling layer in CNN; no learnable parameters.
  • Need: reduce size, reduce overfitting, give translation invariance, speed up.
  • Types: Max (most common), Average, Min, Global, Sum / L2 / Stochastic.
  • Defined by window size, stride, type.
  • Common configuration: 2×2 window, stride 2 → halves the spatial size.

Q2 — ReLU Layer

Asked 5 times(May'23, May'24, Nov'23, May'25, Nov'25 — usually 6 marks)
How this question appeared in past papers:

(May'23, 6 marks) Explain ReLU Layer in detail. What are the advantages of ReLU over Sigmoid?

(May'24, 6 marks) Define ReLU. Explain disadvantages of ReLU.

(Nov'23, 6 marks) Explain ReLU Layer and its advantages.

(May'25, 6 marks) Explain ReLU Layer in detail.

(Nov'25, 6 marks) What is the ReLU activation function? Write its mathematical expression and describe how it transforms the input.

Answer
ReLU is like a one-way valve: positive water passes through unchanged; negative water is blocked completely. Simple, fast, and surprisingly very effective for deep learning.

1. Definition

ReLU (Rectified Linear Unit) is the most popular activation function used in CNNs. It outputs the input directly if it is positive; otherwise it outputs zero.

f(x) = max(0, x)

2. Graph (text description)

     f(x)
       |       /
       |      /
       |     /
       |    /
       |   /
   ────|──/──────────── x
       | /
       |/_______________
       0  (output is 0 for x ≤ 0)

3. Example transformation

If input x = [−3, −2, −1, 0, 2, 5, 8], then ReLU(x) = [0, 0, 0, 0, 2, 5, 8]. All negatives become 0; positives unchanged.

4. Why we need an activation function

  • Without an activation function, a neural network is just a chain of linear transformations — equivalent to one big linear function (useless).
  • Activation functions add NON-LINEARITY so the network can learn complex patterns.

5. Advantages of ReLU (over Sigmoid / Tanh)

  1. Very fast computation — just one max operation; no exponentials.
  2. No vanishing gradient for positive inputs — gradient is 1, so signals flow back easily in deep networks.
  3. Sparse activation — about half the neurons output 0; this acts like automatic dropout, reduces co-adaptation.
  4. Better convergence — networks with ReLU train several times faster than with sigmoid/tanh.
  5. Biologically inspired — real neurons fire in a similar one-sided way.
  6. Easier to optimize — gradient does not saturate for large positive values.

6. Disadvantages of ReLU

  1. Dying ReLU problem — once a neuron's input becomes very negative, gradient is 0 and the neuron may never recover (stays dead forever).
  2. Not zero-centered output — output is always ≥ 0; can introduce bias in gradient updates.
  3. Unbounded output — can cause exploding activations.
  4. Discontinuous derivative at x = 0 — minor mathematical inconvenience.

7. Variants of ReLU (solve some disadvantages)

  • Leaky ReLU: f(x) = max(0.01x, x) — allows small negative slope, avoids dying ReLU.
  • Parametric ReLU (PReLU): slope for negative values is learned.
  • ELU (Exponential Linear Unit): smooth, has negative values.
  • GELU: used in transformers like BERT.

8. ReLU vs Sigmoid — comparison table

AspectReLUSigmoid
Formulamax(0, x)1 / (1 + e-x)
Output range[0, ∞)(0, 1)
ComputationVery fastSlow (exponential)
Vanishing gradientNo (for x > 0)Yes (saturates at both ends)
Zero-centeredNoNo (centered at 0.5)
Dying problemYes (dying ReLU)No
Use todayDefault for hidden layersMostly for binary output

Key points to remember

  • ReLU: f(x) = max(0, x). Outputs 0 for negative, x for positive.
  • Adds non-linearity so deep networks can learn complex functions.
  • Advantages: fast, no vanishing gradient (for x > 0), sparse activation.
  • Disadvantages: dying ReLU, not zero-centered, unbounded.
  • Variants: Leaky ReLU, PReLU, ELU, GELU.

Q3 — CNN Architecture (Draw and Explain)

Asked 4 times(May'23, May'24, Nov'23, May'25 — usually 6 marks)
How this question appeared in past papers:

(May'23, 6 marks) Draw and explain CNN (Convolution Neural Network) architecture in detail.

(May'24, 6 marks) Explain CNN architecture with its application.

(Nov'23, 6 marks) Draw CNN architecture and explain its working.

(May'25, 6 marks) Draw and Explain CNN Architecture in detail.

Answer

1. Definition

A CNN (Convolutional Neural Network) is a deep neural network specialized for processing grid-like data such as images. Its architecture consists of an input layer, followed by alternating convolution and pooling layers, then one or more fully connected layers, ending with an output layer.

2. CNN Architecture diagram

Input         Conv         ReLU       Pool       Conv       ReLU      Pool      Flatten    FC       Softmax    Output
Image  ───►  Filters ───► (max 0) ──► (max)  ──► Filters ─► (max 0) ─►(max) ──► (vector)─► Dense ─► (probs) ──► Class
(32×32×3)    (28×28×16)            (14×14×16)  (10×10×32)         (5×5×32)

3. Layer-by-layer explanation

(a) Input Layer

  • Raw image (e.g., 32 × 32 × 3 for a colour image: 32 height, 32 width, 3 RGB channels).
  • Pixel values are usually normalized to [0, 1] or [−1, 1].

(b) Convolution Layer

  • Applies many small filters (e.g., 16 filters of size 3 × 3 × 3) that slide over the image.
  • Each filter detects a specific feature (edge, texture).
  • Produces feature maps (one per filter).

(c) ReLU Activation Layer

  • Applies f(x) = max(0, x) element-wise on the feature map.
  • Adds non-linearity, replaces all negatives with 0.

(d) Pooling Layer

  • Downsamples the feature map (e.g., 2 × 2 max pooling).
  • Reduces dimensions by half.

(e) Repeat (multiple conv + ReLU + pool blocks)

  • Deeper layers detect more complex features (edges → textures → shapes → objects).

(f) Flatten Layer

  • Converts the multi-dimensional feature map into a 1-D vector.

(g) Fully Connected (FC) Layer

  • Traditional dense layers; combine features for classification.

(h) Output Layer with Softmax

  • For classification: a softmax layer outputs probabilities for each class.
  • For regression: a single linear neuron.

4. Applications of CNN

  • Image classification (ImageNet, CIFAR-10).
  • Object detection (YOLO, SSD, Faster R-CNN).
  • Face recognition (FaceNet).
  • Medical imaging (tumor detection, X-ray analysis).
  • Self-driving cars (lane detection, pedestrian detection).
  • OCR (handwritten character recognition).
  • Image segmentation (U-Net).
  • Style transfer and image generation.

5. Famous CNN architectures (case studies)

  • LeNet-5 (1998) — first successful CNN, for handwritten digit recognition.
  • AlexNet (2012) — won ImageNet, kicked off the deep learning revolution.
  • VGGNet (2014) — very deep, simple 3×3 filters everywhere.
  • GoogLeNet / Inception (2014) — uses "Inception modules".
  • ResNet (2015) — residual / skip connections; allows 100+ layers.
  • EfficientNet, ConvNeXt — modern efficient designs.

Key points to remember

  • CNN = Input → (Conv + ReLU + Pool) ×N → Flatten → FC → Softmax → Output.
  • Conv detects features; ReLU adds non-linearity; Pool downsamples.
  • Deeper layers detect more complex features.
  • FC layers combine features for classification.
  • Famous: LeNet, AlexNet, VGG, GoogLeNet, ResNet.

Q4 — Padding and its Types

Asked 3 times(May'24, Nov'23, Nov'25 — usually 6 marks)
How this question appeared in past papers:

(May'24, 6 marks) What is Padding? Enlist and explain types of padding.

(Nov'23, 6 marks) Explain Padding and its types.

(Nov'25, 6 marks) How does padding help retain the spatial size?

Answer
When we slide a filter over the edge of an image, the filter "falls off" the side — the corners of the image get less coverage than the centre. Padding is like adding a picture frame around the image — extra pixels around the border — so that the filter can slide over the edge pixels evenly.

1. Definition

Padding in a CNN is the technique of adding extra rows and columns of pixels (usually zeros) around the input image before applying the convolution. It controls the output size of the feature map and allows the filter to process edge pixels fairly.

2. Need for Padding

  • Without padding, the output feature map is smaller than the input — repeatedly losing pixels at edges over many layers.
  • Pixels at the corners get fewer overlaps with the filter than central pixels — corner information is "underused".
  • Padding preserves the spatial dimensions across layers.
  • Allows for "same" output size, useful in deep architectures (ResNet, U-Net).

3. Output size formula (with padding p)

Output size = ((Input − Filter + 2 × Padding) / Stride) + 1

4. Types of padding

(a) Valid Padding (no padding, p = 0)

  • NO extra pixels added.
  • Output is SMALLER than input.
  • Used when shrinking the feature map is desired.
  • Example: input 28×28, filter 3×3, stride 1, valid → output 26×26.

(b) Same Padding (output = input size)

  • Padding is chosen so that output size = input size (with stride 1).
  • For a filter of size k×k, padding = (k−1)/2.
  • Most popular type in modern CNNs.
  • Example: input 28×28, filter 3×3, padding 1, stride 1 → output 28×28.

(c) Full Padding

  • Padding = filter_size − 1.
  • Output is LARGER than input.
  • Rare; used in certain signal-processing tasks.

(d) Causal Padding

  • Pads only on ONE side (typically the left/past).
  • Used for sequence data so the convolution does not look at future values.
  • Common in temporal convolutions and WaveNet.

5. What values are used for padding?

  • Zero padding — most common; fill with 0s.
  • Reflection padding — mirror the image edges.
  • Replication padding — repeat the edge pixel values.
  • Constant padding — fill with a user-specified constant.

6. Visual example (zero padding)

Original 4×4 image:        After zero padding (1 pixel border):

   1  2  3  4                  0  0  0  0  0  0
   5  6  7  8                  0  1  2  3  4  0
   9 10 11 12                  0  5  6  7  8  0
  13 14 15 16                  0  9 10 11 12  0
                               0 13 14 15 16  0
                               0  0  0  0  0  0

Key points to remember

  • Padding = adding extra (usually zero) pixels around the image before convolution.
  • Need: preserve size, give edge pixels fair treatment.
  • Types: Valid (no), Same (preserves size), Full, Causal.
  • Formula: Output = (Input − Filter + 2·Padding)/Stride + 1.
  • Most CNNs use ZERO padding with SAME mode.

Q5 — Working of Convolution Layer

Asked 2 times(May'23, May'25 — usually 6 marks)
How this question appeared in past papers:

(May'23, 6 marks) Explain working of Convolution Layer with its features.

(May'25, 6 marks) Explain working of Convolution Layer.

Answer
Imagine you have a small magnifying glass (filter / kernel) and a big poster (the image). You slide the magnifying glass across the poster from left to right, top to bottom, looking at one small region at a time. At each position, you do a quick calculation (multiply & add) and write down ONE number. The collection of all these numbers forms a new "feature map".

1. Definition

The Convolution layer is the core building block of a CNN. It takes an input (image or feature map) and a set of small learnable filters, and produces an output called a "feature map" by sliding each filter across the input and computing dot products at every position.

2. Step-by-step working

  1. Take a small filter (e.g., 3 × 3 × depth) with learnable weights.
  2. Place the filter at the top-left corner of the input.
  3. Compute the element-wise product of filter weights and the input pixels under it.
  4. Sum these products to get one number → this is one output pixel.
  5. Slide the filter to the right by "stride" pixels and repeat.
  6. Continue across the row, then go down to the next row, until the entire input is covered.
  7. All output values form one 2-D feature map.
  8. If there are K filters, we get K feature maps (stacked into a depth-K output).

3. Numerical example (3×3 input, 2×2 filter, stride 1)

Input:         Filter:
 1 2 3         1 0
 4 5 6         0 1
 7 8 9

Output:
  (1·1 + 2·0 + 4·0 + 5·1) = 6      (2·1 + 3·0 + 5·0 + 6·1) = 8
  (4·1 + 5·0 + 7·0 + 8·1) = 12     (5·1 + 6·0 + 8·0 + 9·1) = 14

Feature map:
  6   8
 12  14

4. Features of the Convolution Layer

  • Local connectivity: each output neuron looks only at a small region of the input (the receptive field).
  • Weight sharing: the SAME filter weights are used at every position → drastically fewer parameters.
  • Translation equivariance: if the input shifts, the output shifts the same way.
  • Hierarchical features: deeper conv layers learn higher-level features (edges → shapes → objects).
  • Learnable filters: filter weights are learned during training via backpropagation.
  • Multiple filters per layer: each detects a different feature.
  • Preserves spatial structure: output is still a 2-D grid.

5. Hyperparameters of a Convolution Layer

  • Number of filters (K) — how many feature maps to produce.
  • Filter size (F) — usually 3×3 or 5×5.
  • Stride (S) — how many pixels to skip when sliding.
  • Padding (P) — extra pixels around input border.

6. Output size formula

Output size = ((Input − F + 2P) / S) + 1

7. Number of parameters per filter

Params per filter = F × F × InputDepth + 1 (for bias)

Example: a 3×3 filter on a 64-channel input has 3·3·64 + 1 = 577 parameters.

Key points to remember

  • Convolution = sliding small filter over input, computing dot products.
  • Features: local connectivity, weight sharing, translation equivariance.
  • Output = (Input − F + 2P)/S + 1.
  • Hyperparameters: number of filters K, filter size F, stride S, padding P.
  • Each output pixel = sum of element-wise products of filter and input patch.

Q6 — Stride Convolution

Asked 2 times(May'24, Nov'23 — usually 6 marks)
How this question appeared in past papers:

(May'24, 6 marks) What is Strides in CNN? Explain in brief.

(Nov'23, 6 marks) Explain stride Convolution with example.

Answer
Stride is HOW MANY PIXELS you move the filter each time. Stride = 1 means take one tiny step at a time. Stride = 2 means take a big step, skipping every other position — your output becomes half the size.

1. Definition

Stride is the step size by which the convolution filter slides across the input. With stride 1, the filter moves 1 pixel at a time; with stride 2, it skips every other position.

2. Effect of stride on output size

Output size = ((Input − Filter + 2 · Padding) / Stride) + 1

3. Why use larger stride?

  • Reduces output size → fewer parameters and faster computation.
  • Acts like pooling — downsamples the feature map.
  • Reduces redundancy — neighbouring outputs were highly correlated; skipping some loses little.
  • Used in modern networks (e.g., ResNet's "stride-2 convolutions" replace pooling).

4. Trade-offs

  • Larger stride: faster, smaller output, but loses some detail.
  • Smaller stride: more detail, more computation.

5. Example — stride 2 on 5×5 input with 3×3 filter, no padding

Input (5×5):                Filter (3×3):
1 2 3 4 5                   1 0 1
6 7 8 9 0                   0 1 0
1 2 3 4 5                   1 0 1
6 7 8 9 0
1 2 3 4 5

With stride 2, filter is placed at positions:
   (0, 0), (0, 2)           → 2 positions in this row
   (2, 0), (2, 2)           → 2 positions in this row

Output size = ((5 − 3) / 2) + 1 = 2

Output (2×2):  values calculated by dot product at each position

6. Common stride values

  • Stride 1 — most common; keeps full spatial resolution.
  • Stride 2 — halves the spatial size; used to downsample without pooling.
  • Stride 3 or higher — rare; loses too much information.

Key points to remember

  • Stride = how many pixels the filter moves at a time.
  • Larger stride → smaller output, faster, less detail.
  • Output size = (Input − F + 2P)/S + 1.
  • Stride 2 ≈ a form of downsampling, can replace pooling.
  • Common: stride 1 (most layers), stride 2 (downsample layers).

Q7 — Dropout Layer

Asked 2 times(May'23, May'24 — usually 6 marks)
How this question appeared in past papers:

(May'23, 6 marks) Explain Dropout Layer in Convolutional Neural Network.

(May'24, 6 marks) Explain Dropout Layer in Convolutional Neural Network.

Answer
A teacher wants their students to NOT depend too much on one star pupil. So every day, the teacher randomly tells some students "stay home today, the rest of you handle everything". This forces ALL students to learn the material individually. Dropout does the same to neurons — randomly switches them off during training so all neurons learn useful features.

1. Definition

Dropout is a regularization technique in deep learning where, during training, a random subset of neurons in a layer is "dropped" (set to 0) at each forward pass. This prevents the network from over-relying on any specific neuron and reduces overfitting.

2. Why we need Dropout

  • Deep networks have millions of parameters → tend to overfit the training data.
  • Overfitting = high accuracy on training data, but poor accuracy on new (test) data.
  • Dropout forces the network to develop redundant representations and not depend on any one neuron.

3. How Dropout works

  1. Set a dropout rate p (e.g., 0.5 = 50% of neurons dropped).
  2. During EACH training step:
    • For each neuron, generate a random number.
    • If < p, set that neuron's output to 0.
    • Else, scale the output by 1/(1−p) to keep total magnitude constant.
  3. During testing/inference: all neurons are active; no dropping.
  4. Different neurons are dropped in each training step (random pattern changes).

4. Properties of Dropout

  • Only active during TRAINING; turned off during testing.
  • No learnable parameters (just the dropout rate).
  • Acts like training an ensemble of many smaller networks and averaging them.
  • Most effective when placed AFTER fully-connected layers.
  • Common rates: 0.2–0.5 (deeper layers usually higher).

5. Advantages of Dropout

  • Reduces overfitting significantly.
  • Acts as a model averaging / ensemble approximation.
  • Improves generalization to unseen data.
  • Simple to implement (just one line in modern frameworks).
  • Forces neurons to be more independent and robust.

6. Disadvantages of Dropout

  • Increases training time (more epochs needed for convergence).
  • Choosing the right dropout rate is empirical (trial and error).
  • Less commonly used in CNN convolutional layers (use Batch Normalization there instead).
  • Can hurt performance if dropout rate is too high.

7. Variants

  • Spatial Dropout — drops entire feature maps (channels) instead of individual neurons; better for CNNs.
  • DropConnect — drops individual weights rather than neurons.
  • AlphaDropout — preserves mean and variance, used with SELU activation.

Key points to remember

  • Dropout randomly switches off neurons during training to prevent overfitting.
  • Dropout rate p (typically 0.2–0.5) = fraction of neurons dropped.
  • Only active during training; disabled at test time.
  • Acts like ensemble of smaller networks.
  • Variants: Spatial Dropout (for CNN), DropConnect, AlphaDropout.

Q8 — Local Response Normalization

Asked 2 times(Nov'23, May'25 — usually 6 marks)
How this question appeared in past papers:

(Nov'23, 6 marks) Explain Local response normalization and need of it.

(May'25, 6 marks) Explain local response Normalization.

Answer

1. Definition

Local Response Normalization (LRN) is a normalization layer used in CNNs that normalizes the activations of a neuron based on the responses of NEIGHBOURING neurons (either spatially or across channels). It was introduced in AlexNet to mimic lateral inhibition in biological neurons.

2. Why we need LRN

  • Encourages competition between neurons — only the strongest activations stand out.
  • Improves generalization by spreading activations across many neurons.
  • Helps with brightness normalization in images.
  • Inspired by lateral inhibition in real neurons (one firing neuron suppresses its neighbours).

3. How it works (simplified)

  1. For each activation ai, look at neighbouring activations (e.g., across nearby channels at the same spatial position).
  2. Compute the sum of squares of these neighbours.
  3. Divide ai by this sum (with some constants to control strength).

4. Formula

bi = ai / (k + α · Σ aj²)β
  • ai = original activation
  • bi = normalized activation
  • k, α, β = hyperparameters (typical values: k=2, α=10⁻⁴, β=0.75)
  • Σ = sum over neighbouring channels j around channel i

5. Two types of LRN

  • Inter-channel LRN — normalizes across DIFFERENT channels at the same spatial position. Used in AlexNet.
  • Intra-channel LRN — normalizes within the same channel at neighbouring spatial positions.

6. Pros and cons

  • Pro: small accuracy improvement in early CNNs (AlexNet).
  • Pro: brightness invariance.
  • Con: mostly replaced by Batch Normalization (BN) in modern networks (BN is faster and more effective).
  • Con: additional hyperparameters (k, α, β) to tune.

Key points to remember

  • LRN normalizes activations using neighbouring activations.
  • Inspired by lateral inhibition in biological neurons.
  • Two types: inter-channel and intra-channel.
  • Used in AlexNet; mostly replaced by Batch Normalization now.
  • Formula: divides activation by sum of squared neighbours (with constants).

Q9 — Applications of CNN / Convolution

Asked 2 times(Nov'23, Nov'25 — usually 6 marks)
How this question appeared in past papers:

(Nov'23, 6 marks) What are the applications of Convolution with examples?

(Nov'25, 6 marks) What are Convolutional Neural Networks (CNNs) primarily used for? List at least four real-world applications.

Answer

1. Major application areas of CNN

(a) Image classification

Q10 — Interleaving between Layers in CNN

Asked 1 time(Nov'25 — 6 marks)
How this question appeared in past papers:

(Nov'25, 6 marks) What is Interleaving Between Layers in CNN? Why is Interleaving Important? Explain the role of each interleaving layer.

Answer

1. Definition

Interleaving in a CNN means stacking different types of layers in a specific repeating pattern — typically Convolution → Activation (ReLU) → Pooling — and repeating this block several times before the final fully-connected layers.

2. Why interleaving is important

  • Each layer plays a SPECIFIC role; combining them in this order produces a hierarchy of features.
  • Convolution alone is linear; ReLU adds non-linearity.
  • Pooling reduces size and gives translation invariance.
  • Repeating these allows progressively deeper, more abstract features.
  • Without interleaving, the network would be just a stack of linear filters and would not learn complex patterns.

3. Role of each interleaving layer

(a) Convolution Layer

  • Detects local patterns (edges, corners, textures) using learnable filters.
  • Preserves spatial structure.
  • Outputs a stack of feature maps.

(b) Activation Layer (ReLU)

  • Adds non-linearity → enables the network to learn complex, non-linear functions.
  • Sparsifies activations (many become 0).
  • Prevents the model from being equivalent to a simple linear function.

(c) Pooling Layer

  • Reduces the spatial size of feature maps.
  • Provides translation invariance.
  • Cuts down computation and parameter count.
  • Increases the receptive field of higher layers.

(d) (Optional) Normalization Layer (Batch Norm or LRN)

  • Stabilizes training by normalizing layer outputs.
  • Allows higher learning rates.

(e) Fully Connected Layer (at the end)

  • Combines all extracted features to make the final prediction.

4. Typical interleaving pattern

[Conv → ReLU → Pool] × N  →  Flatten  →  [FC → ReLU] × M  →  Softmax → Output

Key points to remember

  • Interleaving = alternating Conv → ReLU → Pool layers in a repeating block.
  • Each layer has a specific role; together they form a feature hierarchy.
  • Conv → detects features; ReLU → adds non-linearity; Pool → downsamples.
  • Without this interleaving, the network couldn't learn complex patterns.

Q11 — Training a CNN and Need of Normalization

Asked 1 time(Nov'25 — 6 marks)
How this question appeared in past papers:

(Nov'25, 6 marks) List the main steps involved in training a CNN for image classification. Why normalization is important before training a CNN?

Answer

1. Main steps in training a CNN

  1. Collect and prepare data: gather a labeled dataset of images (e.g., ImageNet, CIFAR-10).
  2. Preprocess images: resize to fixed dimensions, normalize pixel values, optionally apply data augmentation.
  3. Define CNN architecture: choose number of conv/pool/FC layers, filter sizes, activation, dropout.
  4. Initialize weights: random initialization (Xavier / He initialization works well).
  5. Forward pass: input image → convolution → ReLU → pooling → … → softmax output.
  6. Compute loss: compare predicted output with true label using e.g., cross-entropy loss.
  7. Backward pass (backpropagation): compute gradients of loss w.r.t. all weights.
  8. Update weights using an optimizer (SGD, Adam, RMSprop) with a chosen learning rate.
  9. Repeat steps 5–8 for many epochs over the training data.
  10. Validate on a held-out validation set; tune hyperparameters.
  11. Test on the test set to measure final accuracy.

2. Why normalization is important before training a CNN

  • Faster convergence: with normalized inputs (values in [0,1] or zero-mean), gradient descent converges much faster.
  • Prevents very large activations: raw pixel values (0–255) can cause numerical instability and exploding gradients.
  • Helps with weight initialization: weight init schemes assume inputs are roughly zero-mean and unit-variance.
  • Equal scale across features: all input channels contribute equally to learning, not just the brightest channel.
  • Reduces sensitivity to lighting: different images have different brightness; normalization makes them comparable.
  • Compatible with activation functions: sigmoid/tanh saturate for large values; ReLU works best with small positives.

3. Common normalization techniques

  • Scaling to [0, 1]: divide all pixel values by 255.
  • Mean subtraction: subtract per-channel mean of the training set.
  • Standard score: (x − mean) / std for each pixel.
  • Batch normalization (inside the network) — normalizes activations layer by layer.

Key points to remember

  • Training CNN: prepare data → preprocess → architecture → init weights → forward → loss → backprop → update → repeat.
  • Normalization is essential for faster, stable training.
  • Common: divide by 255, or (x − mean)/std.
  • Without normalization: slow training, unstable gradients, poor accuracy.

Q12 — Output Size Calculation (numerical problem)

Asked 1 time(Nov'25 — 6 marks)
How this question appeared in past papers:

(Nov'25, 6 marks) Given an input of size 64 × 64, a kernel size of 5 × 5, stride = 2, and 'same' padding: i) What will be the size of the output feature map? ii) How does padding help retain the spatial size?

Answer

(i) Output feature map size

For 'same' padding with stride > 1, the standard formula in most frameworks (TensorFlow, PyTorch) is:

Output size = ⌈ Input size / Stride ⌉

Substituting: Input = 64, Stride = 2 → Output = ⌈ 64 / 2 ⌉ = 32.

So the output feature map is 32 × 32.

Alternative general formula

Output = ((Input − Filter + 2 · Padding) / Stride) + 1

For 'same' padding with filter 5×5 and stride 2: padding is chosen as 2 on each side.

Output = ((64 − 5 + 2·2) / 2) + 1 = (63/2) + 1 = 31.5 + 1 ≈ 32. ✓

(ii) How does padding help retain spatial size?

  • Without padding, applying a 5×5 filter to a 64×64 input gives output size 60×60 → SHRINKS at every layer.
  • After many layers, the feature map would shrink to nothing.
  • By adding extra zeros around the border (padding = 2 for a 5×5 filter), the output stays at 64×64 (when stride = 1).
  • With stride > 1 + 'same' padding, the output is just Input/Stride — predictable and uniform.
  • Padding also gives edge pixels fair treatment — without padding, corner pixels are seen by the filter only once.

Key points to remember

  • For 'same' padding with stride S: Output = ⌈Input/S⌉.
  • General formula: Output = (Input − F + 2P)/S + 1.
  • For this question (64×64, 5×5, stride 2, same) → output = 32×32.
  • Padding helps: preserves size, gives edge pixels fair treatment.
