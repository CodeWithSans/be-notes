Preliminary Concepts (Read Once)

Raw data from real-world sources is MESSY — missing values, typos, duplicates, weird outliers. Before you can analyze it or train ML models, you must CLEAN and PREPARE the data. This step is called Data Preparation (or Data Preprocessing) — and it often takes 60-80% of a data scientist's time! Garbage in = garbage out.

The 5 main steps of Data Preparation

  1. Data Validation — find missing, noisy, inconsistent values.
  2. Data Transformation — convert data into a suitable form (normalize, encode).
  3. Data Reduction — reduce data size while preserving info (sampling, PCA).
  4. Data Discretization — convert continuous attributes into discrete bins.
  5. Data Exploration — univariate/bivariate/multivariate analysis.

Q1 — Data Transformation

Asked 5 times(May'23, Nov'23, May'24, May'25, Nov'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23 & May'25, 5 marks) Explain data transformation in detail with example.

(Nov'23 & Nov'25, 5 marks) What is data Transformation? Explain Data Transformation Process in Detail.

(May'24, 6 marks) What is data transformation? Why is it needed? Explain at least 3 techniques.

Answer
You have a list of salaries: ₹25,000 and ₹2,50,000. Same data, but to a machine learning algorithm, the bigger number dominates. Transforming both into a [0, 1] range makes them comparable. Data transformation is about converting data into a USEFUL form for analysis.

1. Definition

Data Transformation is the process of converting data from one format, structure, or scale to another to make it more suitable for analysis or modeling.

2. Why we need data transformation

  • Different attributes have different scales (age 0-100 vs salary 0-1,000,000).
  • Some algorithms (k-means, neural nets) are sensitive to scale.
  • Skewed distributions need normalization for accurate analysis.
  • Categorical data must be converted to numerical for ML.
  • Data from different sources must follow common format.
  • Remove dependencies (decorrelation).

3. Main techniques of data transformation

(a) Normalization / Min-Max Scaling

Scale all values into a fixed range [0, 1] or [-1, 1].

x' = (x − min) / (max − min)

Example: age 30, with min=0, max=100 → x' = 30/100 = 0.3.

(b) Standardization (Z-score normalization)

Transform to have mean = 0 and std deviation = 1.

x' = (x − μ) / σ

Example: a salary value relative to the mean salary.

(c) Decimal Scaling

Divide by a power of 10 to move decimal point.

x' = x / 10^j (where j makes max(|x'|) < 1)

(d) Log Transformation

Apply log() function — useful for skewed data.

Example: income data → log(income) to reduce skew.

(e) Aggregation

Combine multiple records into a single summary (sum, average).

Example: daily sales aggregated into monthly sales.

(f) Generalization / Concept Hierarchy

Replace low-level data with higher-level concepts.

Example: city → state → country; age 22 → "young adult".

(g) Attribute Construction (Feature Engineering)

Create new attributes from existing ones.

Example: from "date of birth" → derive "age".

(h) Encoding (for categorical data)

  • Label encoding: assign integers (Red=0, Green=1, Blue=2).
  • One-hot encoding: create binary columns (Red=[1,0,0]).

(i) Smoothing

Remove noise (binning, regression, clustering).

4. Example — Data transformation pipeline

Raw data: [Age=25, Salary=₹50,000, City="Pune", Joined="2022-05-10"]

Step 1 (Normalize Age):    25/100 = 0.25
Step 2 (Standardize Salary): (50000 − 60000) / 20000 = -0.5
Step 3 (Encode City):       Pune → [1, 0, 0]  (one-hot)
Step 4 (Attribute construction): Joined → Years_of_Service = 3

Final feature vector: [0.25, -0.5, 1, 0, 0, 3]

5. Common transformation tools

  • scikit-learn (Python): StandardScaler, MinMaxScaler, OneHotEncoder.
  • Pandas: .apply(), .map(), .pivot_table().
  • ETL tools: Talend, Informatica, SSIS.

Key points to remember

  • Data transformation = convert data into a suitable form for analysis.
  • Why: different scales, skewed data, categorical → numerical, common format.
  • Techniques: normalization, standardization, decimal scaling, log, aggregation, encoding, attribute construction.
  • Standardization: (x − μ) / σ; Normalization: (x − min)/(max − min).
  • Categorical encoding: label encoding, one-hot encoding.

Q2 — Data Validation: Incompleteness, Noise, Inconsistency

Asked 4 times(May'23, Nov'23, May'25, Nov'25 — usually 5 marks)
How this question appeared in past papers:

(May'23, Nov'23, Nov'25, 5 marks) Explain data validation, incompleteness, noise, inconsistency of quality of input data.

(May'25, 5 marks) Write short note on Data Validation.

Answer

1. Data Validation

Data Validation is the process of checking that input data is accurate, complete, consistent, and conforms to required quality standards before analysis or storage.

2. Three main quality problems in data

(a) Incompleteness

  • Data has MISSING values for some attributes.
  • Causes: not collected, sensor failure, user skipped field, data lost.
  • Example: Customer record with no email address.
  • Solutions:
    • Delete records with missing values (if few).
    • Fill with default values (e.g., "unknown").
    • Impute with mean / median / mode.
    • Predict missing value using regression or ML.
    • Hot-deck imputation (use similar record's value).

(b) Noise

  • Noise = random error or variance in measured values.
  • Causes: faulty sensors, measurement errors, data entry mistakes, transmission errors.
  • Example: temperature reading 5000°C (impossible).
  • Solutions:
    • Binning: sort values, group into bins, replace with bin mean / median.
    • Smoothing: regression to fit smooth curve.
    • Clustering: outliers in clusters are noise.
    • Outlier detection: remove values outside expected range.

(c) Inconsistency

  • Data contradicts itself or violates rules.
  • Causes: data collected from multiple sources, manual entry errors, format mismatch.
  • Examples:
    • Date of birth = 1990, age = 25 (inconsistent).
    • "NY" in one record, "New York" in another for same city.
    • Negative salary or impossible values.
  • Solutions:
    • Standardize formats (e.g., always "YYYY-MM-DD" for dates).
    • Apply integrity constraints (database level).
    • De-duplication.
    • Cross-checks against authoritative sources.

3. Other quality issues

  • Duplicates — same record appearing multiple times.
  • Outliers — extreme values that may or may not be errors.
  • Type errors — wrong data type (string in numeric field).
  • Stale data — outdated information.

4. Importance of data validation

  • Ensures GIGO (Garbage In = Garbage Out) is avoided.
  • Improves model accuracy.
  • Saves time downstream (cheaper to fix early).
  • Builds trust in BI reports.

Key points to remember

  • Data validation = check data quality before use.
  • Three main problems: Incompleteness (missing), Noise (random errors), Inconsistency (contradictions).
  • Incomplete data: impute, delete, predict.
  • Noisy data: binning, smoothing, outlier removal.
  • Inconsistent data: standardize formats, integrity constraints, de-duplicate.

Q3 — Data Reduction

Asked 3 times(May'23, May'25, Nov'25 — usually 6–7 marks)
How this question appeared in past papers:

(May'23, May'25, Nov'25, 7 marks) Explain data reduction in detail with example.

(May'24, 6 marks) What is data reduction? Explain Dimensionality Reduction and Data Compression.

Answer
You have 1 TB of customer data. Analyzing all of it takes hours. Data Reduction = make the dataset SMALLER without losing important information, so analysis is fast. Like making a thumbnail of a photo — small, but still recognizable.

1. Definition

Data Reduction is the process of reducing the SIZE of a dataset (in rows, columns, or both) while preserving the essential information needed for analysis.

2. Why we need data reduction

  • Speeds up analysis and ML training.
  • Reduces storage cost.
  • Avoids curse of dimensionality.
  • Removes irrelevant or redundant features.
  • Easier to visualize.

3. Main techniques of data reduction

(a) Sampling (reduce ROWS)

  • Select a representative subset of records.
  • Types: simple random sampling, stratified sampling, cluster sampling, systematic sampling.
  • Example: select 10,000 customers from 1 million.

(b) Feature Selection (reduce COLUMNS)

  • Keep only the most relevant attributes.
  • Methods:
    • Filter methods: rank features by correlation / chi-square.
    • Wrapper methods: use ML model to evaluate feature subsets.
    • Embedded methods: built into model (Lasso, Decision Tree importance).
  • Example: drop "phone number" if you only need demographic analysis.

(c) Principal Component Analysis (PCA) — Dimensionality Reduction

  • Transforms data into fewer NEW features (principal components) that capture most variance.
  • Each PC is a linear combination of original features.
  • Used in image compression, gene-expression analysis.
  • Reduces 100s of features to a few key dimensions.

(d) Data Compression

  • Encoded representation of data takes less space.
  • Lossless (ZIP, Huffman) or lossy (JPEG, MP3).
  • In BI: cube compression (sparse cubes stored compactly).

(e) Numerosity Reduction

  • Replace data with a smaller representation.
  • Examples: parametric models (regression instead of raw data), histograms, clustering, sampling.

(f) Discretization

  • Convert continuous to categorical (covered separately).

(g) Aggregation

  • Replace detailed records with summary (e.g., monthly aggregates instead of daily).

4. Example

Original: 1,000,000 records, 50 features per record
After data reduction:
  - Sampling: 100,000 records
  - Feature selection: 20 features
  - PCA: 10 principal components
Final: 100,000 records × 10 features = 1,000,000 data points (instead of 50M)

5. Advantages

  • Faster computation.
  • Less storage.
  • Less overfitting.
  • Easier visualization.

6. Risks

  • May lose important patterns (especially with aggressive reduction).
  • Choosing how much to reduce is non-trivial.
  • PCA may produce uninterpretable features.

Key points to remember

  • Data reduction = make dataset smaller while preserving info.
  • Techniques: Sampling, Feature Selection, PCA, Compression, Numerosity Reduction, Aggregation, Discretization.
  • Sampling = reduce rows; Feature selection = reduce columns; PCA = transform to fewer dimensions.
  • Benefits: faster analysis, less storage, less overfitting.

Q4 — Univariate, Bivariate, Multivariate Analysis

Asked 3 times(May'23, Nov'23, May'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23 & May'25, 5 marks) Difference between univariate, Bivariate, Multivariate analysis.

(Nov'23, 5 marks) Explain univariate, bivariate and multivariate analysis with example and applications.

Answer

1. Univariate Analysis

Analysis of ONE variable at a time. Describes the distribution and characteristics of a single attribute.

  • Measures: mean, median, mode, std deviation, variance, range, quartiles.
  • Visualizations: histogram, box plot, bar chart, pie chart.
  • Purpose: understand the central tendency and spread of one variable.
  • Example: "What is the average age of customers?"

2. Bivariate Analysis

Analysis of relationship between TWO variables.

  • Measures: correlation coefficient, covariance, contingency tables.
  • Visualizations: scatter plot, line chart, cross-tab.
  • Purpose: understand how two variables relate to each other.
  • Types:
    • Numerical-Numerical: correlation (Pearson, Spearman).
    • Categorical-Categorical: contingency table, chi-square test.
    • Numerical-Categorical: box plots grouped by category, ANOVA.
  • Example: "Does income relate to spending?"

3. Multivariate Analysis

Analysis of THREE OR MORE variables together.

  • Measures: correlation matrix, regression, factor analysis, PCA.
  • Visualizations: heatmap, pair plot, parallel coordinates, 3D scatter.
  • Purpose: understand complex relationships involving many variables.
  • Examples:
    • Multiple regression: predict house price from area, bedrooms, location.
    • Principal Component Analysis: reduce dimensions.
    • Cluster analysis: group customers by multiple attributes.

4. Comparison table

AspectUnivariateBivariateMultivariate
Variables123 or more
GoalDescribe one variableFind relation between twoFind pattern among many
MeasuresMean, median, std devCorrelation, covarianceCorrelation matrix, regression, PCA
VisualizationHistogram, box plotScatter plot, crosstabHeatmap, pair plot
Example questionWhat is the avg age?Does age affect income?What predicts customer churn?

5. Applications

  • Univariate: quality control, exam-score distributions.
  • Bivariate: sales vs marketing spend, height vs weight.
  • Multivariate: medical diagnostics, customer segmentation, recommendation systems.

Key points to remember

  • Univariate: 1 variable; describe.
  • Bivariate: 2 variables; find relationship.
  • Multivariate: 3+ variables; complex patterns.
  • Tools: histogram (uni), scatter plot (bi), heatmap (multi).

Q5 — Data Discretization

Asked 3 times(May'23, May'25, Nov'25 — usually 5 marks)
How this question appeared in past papers:

(May'23, May'25, Nov'25, 5 marks) Write a short note on data discretization.

Answer

1. Definition

Data Discretization is the process of converting CONTINUOUS attributes into DISCRETE intervals (bins or categories). The continuous attribute is replaced by interval labels or by representative values for each interval.

2. Why discretize?

  • Some algorithms (e.g., Naive Bayes, association rule mining) require discrete inputs.
  • Reduces noise.
  • Makes data simpler and easier to interpret.
  • Reduces data size.

3. Example

Original (continuous) AGE values: 5, 17, 23, 35, 47, 65, 80

After discretization into bins:
   "child" (0-12):    [5]
   "teen" (13-19):    [17]
   "young adult" (20-35): [23, 35]
   "middle aged" (36-60): [47]
   "senior" (60+):    [65, 80]

4. Discretization techniques

(a) Equal-Width Binning

  • Divide the range into equal-sized intervals.
  • Example: ages 0–100 split into bins of width 20.
  • Simple but sensitive to outliers.

(b) Equal-Frequency (Equal-Depth) Binning

  • Each bin contains roughly the same NUMBER of records.
  • Example: 100 customers → 5 bins of 20 customers each.
  • Better for skewed data.

(c) Clustering-Based Discretization

  • Apply clustering (e.g., k-means) on the attribute; each cluster = one bin.
  • Bins reflect natural groupings.

(d) Decision Tree-Based Discretization

  • Use entropy/information gain to find optimal split points.
  • Supervised — uses class labels.

(e) Chi-Square Based (ChiMerge)

  • Statistical test to merge adjacent intervals.
  • Bottom-up — merges similar adjacent bins.

(f) Concept Hierarchy

  • Replace numeric values with high-level concepts.
  • Example: age → "youth", "adult", "senior".

5. Advantages and limitations

  • + Easier interpretation.
  • + Robust to noise.
  • + Required by some algorithms.
  • − Loss of information (granularity).
  • − Choice of bin width / count affects results.

Key points to remember

  • Discretization = continuous → discrete (bins / categories).
  • Methods: equal-width, equal-frequency, clustering, decision tree, chi-square, concept hierarchy.
  • Used for Naive Bayes, association rules, easier interpretation.
  • Trade-off: simpler but lose granular info.

Q6 — Contingency Table & Marginal Distribution

Asked 2 times(Nov'23, Nov'25 — usually 7 marks)
How this question appeared in past papers:

(Nov'23 & Nov'25, 7 marks) What is a Contingency Table? What is Marginal Distribution? Justify with suitable example.

Answer

1. Contingency Table

A Contingency Table (also called cross-tab) is a TABLE that displays the FREQUENCY DISTRIBUTION of two (or more) categorical variables together. It shows the joint distribution of values.

2. Example

Survey of 200 students about smartphone preference:

                  Apple    Samsung   OnePlus   Total
   Male            30       40        20       90
   Female          40       50        20       110
   ─────────────────────────────────────────────
   Total           70       90        40       200

3. Marginal Distribution

The Marginal Distribution of one variable is found by SUMMING the joint frequencies over the other variable. It shows the distribution of one variable IGNORING the others.

4. Marginal distributions from the example above

  • Marginal distribution of Gender: Male = 90, Female = 110 (sums of rows).
  • Marginal distribution of Brand: Apple = 70, Samsung = 90, OnePlus = 40 (sums of columns).

The "Total" row and column in a contingency table show the marginal distributions.

5. Conditional Distribution (related concept)

  • The distribution of one variable GIVEN a specific value of the other.
  • Example: among MALES, the distribution of brand is Apple=30/90, Samsung=40/90, OnePlus=20/90.

6. Uses of Contingency Tables

  • Examining relationships between categorical variables.
  • Chi-square test of independence.
  • Market segmentation.
  • Health and medical research.
  • Survey analysis.
  • A/B testing comparisons.

7. Chi-square test (briefly)

  • Used to test whether two categorical variables are independent.
  • Compares observed frequencies with expected frequencies under independence assumption.
  • χ² = Σ (O − E)² / E.

Key points to remember

  • Contingency table = joint frequency table for 2+ categorical variables.
  • Marginal distribution = sum of joint frequencies for one variable.
  • Total row/column shows marginal distributions.
  • Conditional distribution = distribution given a fixed value of another variable.
  • Used with chi-square test to check independence.

Q7 — Sampling, Feature Selection, PCA

Asked 2 times(Nov'23, Nov'25 — usually 5 marks)
How this question appeared in past papers:

(Nov'23 & Nov'25, 5 marks) Explain following Data reduction techniques: Sampling, Feature selection, Principal component analysis.

Answer

1. Sampling

Sampling = selecting a smaller subset (sample) of records from the full dataset (population) for analysis.

  • Types:
    • Simple Random Sampling: every record has equal chance.
    • Stratified Sampling: sample from each subgroup (stratum) proportionally.
    • Cluster Sampling: divide population into clusters, sample whole clusters.
    • Systematic Sampling: every k-th record.
  • Pros: faster, cheaper analysis.
  • Cons: may miss rare patterns; sampling error.

2. Feature Selection

Feature Selection = selecting the MOST RELEVANT attributes (columns) and dropping the rest.

  • Methods:
    • Filter: rank features by statistical measures (correlation, chi-square, information gain).
    • Wrapper: evaluate feature subsets using a ML model (forward selection, backward elimination).
    • Embedded: feature selection built into the learning algorithm (Lasso, decision tree importance).
  • Pros: simpler models, less overfitting, faster training.
  • Cons: may miss feature interactions.

3. Principal Component Analysis (PCA)

PCA = a dimensionality-reduction technique that transforms data into a smaller set of NEW features (principal components) that capture most of the variance.

How PCA works

  1. Standardize data (mean=0, std=1).
  2. Compute the covariance matrix.
  3. Find eigenvectors and eigenvalues of the covariance matrix.
  4. Sort eigenvectors by eigenvalue (largest first).
  5. Pick top-k eigenvectors → these are the principal components.
  6. Project data onto these k components → reduced data.

Properties of PCA

  • Each PC is a linear combination of original features.
  • PCs are uncorrelated (orthogonal).
  • First PC captures the most variance; second PC second most; etc.
  • Common rule: keep enough PCs to capture 95% of variance.

Applications

  • Image compression (face recognition, eigenfaces).
  • Gene expression analysis.
  • Customer segmentation.
  • Visualization of high-dimensional data (project to 2D / 3D).

Key points to remember

  • Sampling = reduce rows (records); types: simple, stratified, cluster, systematic.
  • Feature Selection = reduce columns (attributes); methods: filter, wrapper, embedded.
  • PCA = transform to fewer NEW orthogonal features capturing most variance.
  • All three are data reduction techniques used in BI and ML.

Q8 — Data Exploration

Asked 2 times(May'23, May'25 — usually 7 marks)
How this question appeared in past papers:

(May'23 & May'25, 7 marks) Explain data exploration in detail with example.

Answer

1. Definition

Data Exploration (also called Exploratory Data Analysis, EDA) is the initial step of analyzing a dataset to understand its main characteristics — distributions, patterns, anomalies — usually using visual methods and summary statistics.

2. Goals of Data Exploration

  • Understand the structure and content of the dataset.
  • Identify outliers, missing values, and unusual patterns.
  • Discover relationships between variables.
  • Generate hypotheses for further analysis.
  • Choose appropriate models or transformations.

3. Steps of Data Exploration

(1) Univariate Analysis

  • For each variable: compute summary statistics (mean, median, mode, range).
  • Visualize: histogram, box plot.
  • Identify outliers and skewness.

(2) Bivariate Analysis

  • For each pair of variables: compute correlation.
  • Visualize: scatter plots, cross-tabs.
  • Identify linear and non-linear relationships.

(3) Multivariate Analysis

  • Look at multiple variables together.
  • Visualize: heatmaps, pair plots, parallel coordinates.
  • Identify clusters and groups.

(4) Outlier Detection

  • Use box plots, IQR rule, Z-score.
  • Decide whether to remove, transform, or keep.

(5) Missing Value Analysis

  • How many missing? Random or systematic?
  • Decide imputation strategy.

4. Tools for Data Exploration

  • Python: Pandas (describe(), info()), Matplotlib, Seaborn, Plotly.
  • R: ggplot2, summary().
  • BI tools: Tableau, Power BI, Excel.
  • Statistical: SPSS, SAS.

5. Example workflow

Dataset: 1000 customers with Age, Income, Spending Score
1. df.describe() → mean Age=35, Income=₹50,000, Spending=50
2. Histogram of Age → bell shape; Income → right-skewed.
3. Box plot → 5 outliers in Income (very high earners).
4. Scatter plot of Income vs Spending → no clear correlation.
5. Heatmap of correlations → Spending correlated with Age (r=0.6).
6. K-means clustering → 3 customer groups.

6. Measures used in Data Exploration

  • Central tendency: mean, median, mode.
  • Dispersion: variance, std deviation, IQR, range.
  • Shape: skewness, kurtosis.
  • Relationship: correlation coefficient, covariance.

Key points to remember

  • Data Exploration (EDA) = initial analysis to understand data structure.
  • Steps: univariate → bivariate → multivariate → outlier & missing analysis.
  • Visualizations: histogram, box plot, scatter plot, heatmap.
  • Tools: Pandas, Matplotlib, Seaborn, Tableau, Power BI.

Q9 — Other Questions (Dirty Data, Binning, Mean/Median/Mode)

How these appeared in past papers:

(May'24, 6 marks) Define dirty data. Reasons of dirty data.

(May'24, 6 marks) Explain working of binning with example.

(May'24, 6 marks) Bivariate analysis types with examples.

(May'24, 6 marks) Need for data pre-processing and any 2 techniques used.

(Nov'23, 7 marks) Compute Mean, Median and Mode for grouped data.

Answer (Quick reference)

1. Dirty Data

Dirty Data = data that has quality problems: missing values, duplicates, errors, outliers, inconsistencies.

Reasons

  • Manual data entry errors.
  • Sensor / device malfunction.
  • Data collected from multiple sources with different formats.
  • Outdated data not updated.
  • Software bugs in data collection.
  • Incorrect data integration.
  • User skipping mandatory fields.

2. Binning (Discretization technique)

Binning = sort values, group them into bins, replace with bin summary value.

Types

  • Bin by mean: replace each value with bin mean.
  • Bin by median: replace with median.
  • Bin by boundaries: replace with closest bin boundary.

Example (equal-frequency, 3 bins)

Sorted data: 4, 8, 9, 15, 21, 21, 24, 25, 26, 28, 29, 34
Partition into 3 bins of 4 elements each:
   Bin 1: 4, 8, 9, 15      → mean = 9
   Bin 2: 21, 21, 24, 25   → mean = 22.75
   Bin 3: 26, 28, 29, 34   → mean = 29.25

Smoothing by bin mean:
   Replace 4, 8, 9, 15 with 9
   Replace 21, 21, 24, 25 with 22.75
   ...

3. Bivariate Analysis Types

Variable typesAnalysisExample
Numerical × NumericalCorrelation, scatter plot, regressionHeight vs Weight
Categorical × CategoricalContingency table, chi-squareGender vs Brand preference
Numerical × CategoricalBox plot per category, ANOVAIncome vs Education level

4. Need for Data Pre-processing + 2 techniques

Need:

  • Raw data is incomplete, noisy, inconsistent.
  • ML algorithms expect clean, numerical, scaled data.
  • Better data → better predictions.

Two techniques:

  1. Data Cleaning — handle missing values, remove duplicates, smooth noise.
  2. Data Transformation — normalize, standardize, encode categorical variables.

5. Mean, Median, Mode (grouped data formulas)

  • Mean = Σ(f × midpoint) / N.
  • Median = L + ((N/2 − F) / f) × h
    • L = lower boundary of median class
    • F = cumulative frequency before median class
    • f = frequency of median class
    • h = class width
  • Mode = L + ((f₁ − f₀) / (2f₁ − f₀ − f₂)) × h
    • L = lower boundary of modal class
    • f₁ = frequency of modal class
    • f₀ = frequency of previous class
    • f₂ = frequency of next class

Apply formulas to the given grouped data in the exam question.

Key points to remember

  • Dirty data = errors, missing, duplicates; causes are manual entry, sensors, multiple sources.
  • Binning = group sorted values into bins, replace with bin mean/median/boundary.
  • Bivariate types: Num-Num (correlation), Cat-Cat (contingency), Num-Cat (box plot, ANOVA).
  • Mean/Median/Mode formulas for grouped data must be memorized.
