Preliminary Concepts (Read Once)

Machine Learning brings PREDICTIVE POWER to BI. Without ML, BI tells you what HAPPENED (sales last month). With ML, BI tells you what WILL HAPPEN (churn next month) and what TO DO (best marketing campaign). This chapter covers the main ML techniques used in BI: classification, clustering, association rules.

The 3 main ML tasks in BI

  1. Classification — predict a CATEGORY (e.g., "will this customer churn?"). Examples: Logistic Regression, Decision Tree, Naive Bayes.
  2. Clustering — group records into UNKNOWN clusters (e.g., customer segments). Examples: K-means, Hierarchical.
  3. Association Rules — find which items go TOGETHER (e.g., "people who buy bread also buy butter"). Example: Apriori.

Q1 — Logistic Regression

Asked 6 times(May'23, Nov'23, May'24, May'25, Nov'25 twice — usually 6–8 marks)
How this question appeared in past papers:

(May'23 & May'25, 8 marks) Explain logistic regression with example.

(Nov'23, 6 marks) Write a short note on Logistic Regression.

(May'24, 6 marks) What is logistic regression? Discuss the types of logistic regression.

(Nov'25, 6 marks) What is logistic regression? Discuss the types of logistic regression.

(Nov'25, 6 marks) Explain the logistic regression with example.

Answer
Linear regression draws a straight line through your data — useful when output is a number (price, temperature). But what if the output is a YES/NO question — "will the email be spam?"? Linear regression doesn't work because the answer must be a probability between 0 and 1. Logistic regression solves this by squashing the line through a "sigmoid" curve that gives a probability output.

1. Definition

Logistic Regression is a supervised machine learning algorithm used for CLASSIFICATION problems. Despite its name, it predicts the PROBABILITY of a binary (or categorical) outcome, not a continuous value.

2. The Sigmoid Function

Logistic regression uses the sigmoid function to convert any real number into a probability in [0, 1]:

σ(z) = 1 / (1 + e−z)

where z = β₀ + β₁·x₁ + β₂·x₂ + … + βn·xn (linear combination of features).

3. Sigmoid curve (text diagram)

   σ(z)
    1.0 |                    ___________
        |                __—'
    0.5 |─────────────—'
        |          __—'
    0.0 |____——'
        |———————————|———————————→  z
                    0

  If σ(z) ≥ 0.5  → predict class 1
  If σ(z) < 0.5  → predict class 0

4. Working — Steps

  1. Compute z = β₀ + β₁·x₁ + … + βn·xn for each input.
  2. Apply sigmoid: p = σ(z) = 1 / (1 + e−z).
  3. If p ≥ 0.5, predict class 1; else class 0.
  4. Coefficients β are learned during training via maximum likelihood estimation (or gradient descent).

5. Cost function (Log Loss)

J(β) = −(1/m) Σ [ yᵢ log(pᵢ) + (1 − yᵢ) log(1 − pᵢ) ]

Minimized using gradient descent or similar optimization.

6. Types of Logistic Regression

(a) Binary Logistic Regression

  • Output has 2 categories (Yes/No, Spam/Not Spam, Pass/Fail).
  • Most common form.
  • Example: predict whether customer will buy a product.

(b) Multinomial Logistic Regression

  • Output has more than 2 UNORDERED categories.
  • Example: predict customer's preferred phone brand (Apple, Samsung, OnePlus).
  • Uses softmax function instead of sigmoid.

(c) Ordinal Logistic Regression

  • Output has more than 2 ORDERED categories.
  • Example: customer satisfaction rating (1-star, 2-star, …, 5-star).

7. Example — Predicting Loan Default

Input features: Age, Income, Credit Score
Target: Default (1 = Yes, 0 = No)

Training data of 1000 customers used to learn coefficients:
   β₀ = -5.2,  β_age = -0.04,  β_income = -0.0001,  β_credit = -0.02

For new customer (age=30, income=₹50000, credit=700):
   z = -5.2 + (-0.04 × 30) + (-0.0001 × 50000) + (-0.02 × 700)
     = -5.2 − 1.2 − 5 − 14 = -25.4
   p = σ(-25.4) ≈ 0.0000000... (very close to 0)
   → predict: not likely to default

8. Advantages

  • Simple to implement and interpret.
  • Outputs probability, not just class.
  • Works well for linearly separable data.
  • Less prone to overfitting than complex models.
  • Fast training and prediction.

9. Disadvantages

  • Assumes linear relationship between features and log-odds.
  • Sensitive to outliers.
  • Cannot solve non-linear problems without feature engineering.
  • Requires no multicollinearity among features.
  • Needs reasonable sample size.

10. Applications in BI

  • Customer churn prediction.
  • Spam email detection.
  • Credit risk assessment / loan default prediction.
  • Medical diagnosis (disease yes/no).
  • Marketing campaign response prediction.
  • Fraud detection.

Key points to remember

  • Logistic regression = classification, not regression.
  • Uses sigmoid: σ(z) = 1/(1+e−z) to output probability.
  • Threshold 0.5 → class prediction.
  • Types: Binary, Multinomial, Ordinal.
  • Cost: log loss (binary cross-entropy).
  • Applications: churn, spam, fraud, medical diagnosis.

Q2 — Association Rule (Support, Confidence, Lift)

Asked 4 times(May'23, Nov'23, May'25, Nov'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23, 5 marks) What is association rule mining? Explain support, confidence, lift.

(Nov'23 & Nov'25, 6 marks) What are association rules? How to evaluate using Support and Confidence?

(May'25, 5 marks) Explain Association rule and its types.

Answer
"People who buy DIAPERS also buy BEER." This famous Walmart finding is the most-cited association rule. Stores use this to place items strategically. Association rule mining finds patterns like "if A is bought, B is also bought".

1. Definition

Association Rule Mining is a data mining technique that discovers interesting RELATIONSHIPS (associations) between variables in large datasets — typically transactional data.

An association rule has the form: X → Y, meaning "if X occurs in a transaction, then Y is likely to occur too".

2. Three key metrics

(a) Support

How OFTEN the itemset {X, Y} appears in the dataset.

Support(X → Y) = P(X ∩ Y) = (# transactions containing X and Y) / (total transactions)

Example: if "bread + butter" appears in 100 of 1000 transactions, Support = 0.1 = 10%.

(b) Confidence

How OFTEN Y occurs in transactions that contain X.

Confidence(X → Y) = P(Y | X) = Support(X ∩ Y) / Support(X)

Example: if 100 transactions have "bread + butter" and 200 have "bread", Confidence = 100/200 = 0.5 = 50%.

(c) Lift

How much MORE LIKELY Y is to occur when X is bought, vs Y's normal occurrence.

Lift(X → Y) = Confidence(X → Y) / Support(Y) = P(X ∩ Y) / (P(X) · P(Y))
  • Lift > 1 → X and Y are positively correlated (good rule).
  • Lift = 1 → independent.
  • Lift < 1 → negatively correlated (X actually reduces chance of Y).

3. Types of association rules

  • Single-dimensional: all items from same dimension (e.g., products).
  • Multi-dimensional: items from different dimensions (e.g., age + product).
  • Quantitative: involve numeric attributes (age range, price range).
  • Boolean: presence or absence of items.

4. Example

1000 supermarket transactions:
   - 300 contain {bread}
   - 200 contain {butter}
   - 150 contain {bread, butter}

Rule: {bread} → {butter}
   Support = 150 / 1000 = 0.15 = 15%
   Confidence = 150 / 300 = 0.5 = 50%
   Lift = 0.5 / (200/1000) = 0.5 / 0.2 = 2.5

Lift = 2.5 means: when bread is bought, the chance of butter being bought is 2.5×
the random chance → strong positive association.

5. Applications

  • Market basket analysis — supermarket product placement.
  • Recommendation systems — "customers who bought X also bought Y" on Amazon.
  • Web usage mining — pages visited together.
  • Healthcare — symptom/disease relationships.
  • Fraud detection — unusual co-occurrences.
  • Customer segmentation.

Key points to remember

  • Association rule: X → Y (if X bought, Y likely bought).
  • Support = how often X and Y appear together.
  • Confidence = probability of Y given X.
  • Lift = how much more likely Y is given X vs baseline.
  • Lift > 1: positive, Lift = 1: independent, Lift < 1: negative.
  • Used in market basket analysis, recommendations.

Q3 — Classification vs Clustering

Asked 4 times(May'23, Nov'23, May'24, May'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23 & May'25, 5 marks) Difference between classification and clustering.

(Nov'23, 6 marks) Difference between classification and clustering with applications.

(May'24, 6 marks) How classification and clustering are different. Discuss with example.

Answer
A teacher with a LABELED textbook is doing CLASSIFICATION — she sorts new students based on existing groups (good, average, weak). A teacher looking at a NEW class with NO labels does CLUSTERING — she groups students based on similarity. Classification = supervised; Clustering = unsupervised.

1. Classification

  • Type: Supervised learning (uses LABELED training data).
  • Goal: Predict the class label of new data points.
  • Output: Discrete class labels (e.g., spam / not spam).
  • Examples of algorithms: Logistic Regression, Decision Tree, Naive Bayes, SVM, Random Forest, k-NN.
  • Evaluation: Accuracy, Precision, Recall, F1, Confusion Matrix.

2. Clustering

  • Type: Unsupervised learning (NO labels in training data).
  • Goal: Group similar data points into clusters.
  • Output: Cluster assignments (no predefined names).
  • Examples of algorithms: K-means, Hierarchical clustering, DBSCAN, Gaussian Mixture Models.
  • Evaluation: Silhouette score, Davies-Bouldin index, within-cluster variance.

3. Side-by-side comparison

AspectClassificationClustering
Learning typeSupervisedUnsupervised
Training dataLabeled (X + y)Unlabeled (X only)
OutputPredicted class labelCluster assignment
GoalPredict known categoriesDiscover unknown groups
ExamplesSpam detection, churn predictionCustomer segmentation, market segmentation
AlgorithmsLogistic Reg, Decision Tree, SVM, k-NNK-means, Hierarchical, DBSCAN
EvaluationAccuracy, F1, ROC-AUCSilhouette, DB index
Domain knowledgeNeeded for labelingUsed to interpret clusters

4. Examples

Classification ExampleClustering Example
Email: spam vs not spamGroup emails by topic (no labels)
Loan: default vs no defaultGroup customers by buying behavior
Disease: positive vs negativeFind disease subtypes from symptoms
Product review: positive / negativeGroup similar reviews together

5. Applications in BI

  • Classification: credit scoring, fraud detection, churn prediction, marketing response prediction.
  • Clustering: customer segmentation, product grouping, market segmentation, anomaly detection.

Key points to remember

  • Classification = supervised, uses labels, predicts class.
  • Clustering = unsupervised, no labels, finds groups.
  • Classification: predict known classes (spam, churn).
  • Clustering: discover unknown groups (segments).

Q4 — Apriori Algorithm (Numerical Problem)

Asked 4 times(May'23, Nov'23, May'24, May'25 — usually 6–8 marks)
How this question appeared in past papers:

(May'23, Nov'23, May'25, 6-8 marks) Find frequent itemsets and generate association rules using Apriori algorithm. min_support = 2, min_confidence = 60-70%.

(May'24, 6 marks) What is the Apriori Algorithm? Discuss applications with example.

Answer

1. Definition

The Apriori Algorithm is a classic algorithm for mining FREQUENT ITEMSETS and generating ASSOCIATION RULES from transactional data. Proposed by Agrawal & Srikant (1994). Named for the "a priori" principle.

2. The Apriori Principle

"If an itemset is FREQUENT, then ALL its SUBSETS must also be frequent. Conversely, if a subset is infrequent, the larger itemset cannot be frequent either."

This allows aggressive pruning of candidates.

3. Steps of Apriori

  1. Find all frequent 1-itemsets (single items meeting min_support).
  2. Use them to generate candidate 2-itemsets.
  3. Scan database, count support, keep only frequent ones.
  4. Use frequent k-itemsets to generate candidate (k+1)-itemsets.
  5. Repeat until no more frequent itemsets can be generated.
  6. Generate association rules from frequent itemsets and filter by min_confidence.

4. SOLVED EXAMPLE

Transactions:

T1: I1, I2, I5
T2: I2, I4
T3: I2, I3
T4: I1, I2, I4
T5: I1, I3
T6: I2, I3
T7: I1, I3
T8: I1, I2, I3, I5
T9: I1, I2, I3

min_support = 2,  min_confidence = 60%

Step 1: Count 1-itemsets and filter by min_support

Item   Count
I1     6     ✓ (≥ 2)
I2     7     ✓
I3     6     ✓
I4     2     ✓
I5     2     ✓

All items meet min_support → L1 = {I1, I2, I3, I4, I5}

Step 2: Generate and count 2-itemsets

Itemset    Count
{I1,I2}    4    ✓
{I1,I3}    4    ✓
{I1,I4}    1    ✗
{I1,I5}    2    ✓
{I2,I3}    4    ✓
{I2,I4}    2    ✓
{I2,I5}    2    ✓
{I3,I4}    0    ✗
{I3,I5}    1    ✗
{I4,I5}    0    ✗

L2 = {(I1,I2), (I1,I3), (I1,I5), (I2,I3), (I2,I4), (I2,I5)}

Step 3: Generate and count 3-itemsets

Candidates generated from L2 (must have all subsets frequent):

Itemset       Count
{I1,I2,I3}    2    ✓
{I1,I2,I5}    2    ✓
{I1,I3,I5}    1    ✗ (and {I3,I5} is not frequent → pruned)
{I2,I3,I4}    0    ✗

L3 = {(I1,I2,I3), (I1,I2,I5)}

Step 4: Try 4-itemsets

{I1,I2,I3,I5} — its subset {I3,I5} is infrequent → pruned.

L4 is empty. Stop.

Step 5: Generate strong association rules

For each frequent itemset, generate rules and check confidence.

Take {I1,I2,I5} (support = 2):

  • {I1,I2} → {I5}: conf = 2/4 = 50% ✗ (below 60%)
  • {I1,I5} → {I2}: conf = 2/2 = 100% ✓
  • {I2,I5} → {I1}: conf = 2/2 = 100% ✓
  • {I1} → {I2,I5}: conf = 2/6 = 33% ✗
  • {I2} → {I1,I5}: conf = 2/7 = 29% ✗
  • {I5} → {I1,I2}: conf = 2/2 = 100% ✓

Strong rules (confidence ≥ 60%):

  • {I1, I5} → {I2} (confidence 100%)
  • {I2, I5} → {I1} (confidence 100%)
  • {I5} → {I1, I2} (confidence 100%)

5. Applications

  • Market basket analysis (supermarket product placement).
  • Web log analysis (which pages visited together).
  • Recommendation systems (Amazon "Frequently bought together").
  • Healthcare (which medications taken together).
  • Fraud detection (unusual combinations).

6. Advantages and limitations

  • + Simple and easy to implement.
  • + Effective use of apriori principle for pruning.
  • − Scans database multiple times (slow for large data).
  • − Generates huge candidates with low min_support.
  • − Improved variants: FP-Growth, ECLAT.

Key points to remember

  • Apriori finds frequent itemsets bottom-up (L1 → L2 → L3 …).
  • Apriori principle: all subsets of frequent itemset are also frequent.
  • Steps: count → filter by min_support → generate next level → repeat.
  • Then generate rules and filter by min_confidence.
  • Applications: market basket, recommendations, web analysis.

Q5 — Bayes Theorem

Asked 2 times(May'23, May'25 — usually 5 marks)
How this question appeared in past papers:

(May'23 & May'25, 5 marks) Explain Bayes theorem in detail.

Answer

1. Statement

Bayes' Theorem describes how to UPDATE the probability of a hypothesis based on new evidence:

P(A | B) = P(B | A) · P(A) / P(B)

2. Terms

  • P(A | B): POSTERIOR probability — probability of A given B.
  • P(B | A): LIKELIHOOD — probability of B given A.
  • P(A): PRIOR probability of A.
  • P(B): Marginal probability of B (evidence).

3. Intuition

You start with a PRIOR belief about something. When new EVIDENCE arrives, you UPDATE your belief using Bayes' theorem to get the POSTERIOR.

4. Example

A disease affects 1% of population. A test is 99% accurate (sensitivity)
and has 5% false positive rate. Patient tests positive. What is the probability they
have the disease?

P(Disease) = 0.01
P(Positive | Disease) = 0.99
P(Positive | No Disease) = 0.05

P(Positive) = P(+|D)·P(D) + P(+|¬D)·P(¬D)
            = 0.99 × 0.01 + 0.05 × 0.99
            = 0.0099 + 0.0495 = 0.0594

P(Disease | Positive) = (0.99 × 0.01) / 0.0594 = 0.167 = 16.7%

Surprising! Even with positive test, only 16.7% probability of disease.

5. Applications

  • Naive Bayes classification (spam, sentiment).
  • Medical diagnosis.
  • Fraud detection.
  • Document classification.
  • A/B testing.
  • Bayesian networks.

6. Naive Bayes Classifier

Apply Bayes' theorem with the "naive" assumption that features are INDEPENDENT given the class:

P(class | features) ∝ P(class) · Π P(feature_i | class)

Used widely for text classification.

Key points to remember

  • Bayes' theorem: P(A | B) = P(B | A) · P(A) / P(B).
  • Updates beliefs based on evidence.
  • Posterior ∝ Likelihood × Prior.
  • Naive Bayes = Bayes' theorem with independence assumption.
  • Used in spam filtering, medical diagnosis, classification.

Q6 — K-means Clustering

Asked 2 times(Nov'23, May'24 — usually 6 marks)
How this question appeared in past papers:

(Nov'23, 6 marks) Given group of visitors by age, find clusters for K=2.

(May'24, 6 marks) What is K-mean clustering? Explain step-by-step working.

Answer

1. Definition

K-means is a popular unsupervised clustering algorithm that partitions n data points into k clusters, where each point belongs to the cluster with the NEAREST mean (centroid).

2. Algorithm steps

  1. Initialize: randomly pick k points as initial centroids.
  2. Assign: each data point to the nearest centroid (using Euclidean distance).
  3. Update: recompute each centroid as the MEAN of its assigned points.
  4. Repeat steps 2–3 until centroids do not change (converged) OR max iterations reached.

3. SOLVED EXAMPLE (1-D data, K=2)

Ages: 16, 16, 17, 20, 20, 21, 21, 22, 23, 29, 36, 41, 42, 43, 44, 45, 61, 62, 66

Step 1 (initialize): pick 2 random centroids: say C1 = 16, C2 = 66.

Step 2 (assign to nearest centroid):
Cluster 1 (closer to 16): 16, 16, 17, 20, 20, 21, 21, 22, 23, 29
Cluster 2 (closer to 66): 36, 41, 42, 43, 44, 45, 61, 62, 66

Step 3 (update centroids):
   New C1 = mean of cluster 1 = (16+16+17+20+20+21+21+22+23+29)/10 = 20.5
   New C2 = mean of cluster 2 = (36+41+42+43+44+45+61+62+66)/9 = 48.9

Step 4: re-assign each point to nearer of (20.5, 48.9).
   Cluster 1: 16, 16, 17, 20, 20, 21, 21, 22, 23, 29
   Cluster 2: 36, 41, 42, 43, 44, 45, 61, 62, 66
   (Same as before → CONVERGED)

Final clusters:
   Cluster 1 (young visitors): 16, 16, 17, 20, 20, 21, 21, 22, 23, 29  (mean 20.5)
   Cluster 2 (older visitors): 36, 41, 42, 43, 44, 45, 61, 62, 66       (mean 48.9)

4. Advantages

  • Simple and fast.
  • Easy to interpret.
  • Scales well to large datasets.
  • Works well with spherical clusters.

5. Limitations

  • Must specify k in advance.
  • Sensitive to initial centroid choice (use k-means++ to fix).
  • Assumes spherical, equal-sized clusters.
  • Sensitive to outliers.
  • Only works with numerical data (need encoding for categorical).

6. Choosing K — Elbow Method

  • Run k-means for k = 1, 2, …, 10.
  • For each k, compute Within-Cluster Sum of Squares (WCSS).
  • Plot WCSS vs k → look for the "elbow" where adding more clusters gives diminishing returns.

7. Applications

  • Customer segmentation.
  • Image segmentation (color quantization).
  • Document clustering.
  • Anomaly detection.
  • Market basket grouping.

Key points to remember

  • K-means: partition n points into k clusters by nearest centroid.
  • 4 steps: initialize → assign → update → repeat.
  • Pick k via elbow method.
  • Sensitive to initial centroids; use k-means++.
  • Applications: segmentation, image compression, anomaly detection.

Q7 — Hierarchical vs Partitioning Clustering

Asked 2 times(May'23, Nov'25 — usually 5–6 marks)
How this question appeared in past papers:

(May'23 & Nov'25, 5-6 marks) What is the difference between hierarchical clustering and partitioning method?

Answer

1. Partitioning Clustering

  • Divides data into K non-overlapping FLAT clusters.
  • Each data point belongs to exactly ONE cluster.
  • Iterative — improves cluster assignments through repeated iterations.
  • K must be specified in advance.
  • Examples: K-means, K-medoids (PAM), CLARA.

2. Hierarchical Clustering

  • Builds a TREE-LIKE hierarchy of clusters (called a dendrogram).
  • No need to specify K in advance.
  • Two types:
    • Agglomerative (bottom-up): start with each point as own cluster; iteratively merge closest pairs.
    • Divisive (top-down): start with all points in one cluster; iteratively split.
  • To get K clusters, cut the dendrogram at desired level.

3. Comparison table

AspectPartitioningHierarchical
StructureFlat (single level)Tree (dendrogram)
K specificationRequired upfrontNot required (cut tree later)
Algorithm typeIterative refinementMerge or split iteratively
ExamplesK-means, K-medoidsAgglomerative, Divisive
ResultSingle set of clustersMultiple levels of clusters
ReversibilityIterative until convergenceDecisions are final (no merge undo)
ScalabilityO(n) per iteration (fast)O(n²) or O(n³) — slow for big data
VisualizationScatter plot of clustersDendrogram
Best forLarge datasets, known KSmall datasets, exploratory

4. Linkage methods (Hierarchical)

  • Single linkage: minimum distance between two clusters.
  • Complete linkage: maximum distance between two clusters.
  • Average linkage: average distance.
  • Ward's method: minimize within-cluster variance.

Key points to remember

  • Partitioning: flat clusters, K fixed in advance (K-means, K-medoids).
  • Hierarchical: tree of clusters; agglomerative (bottom-up) or divisive (top-down).
  • Partitioning is faster; hierarchical gives more insight via dendrogram.
  • Hierarchical doesn't require K upfront.

Q8 — Decision Tree

Asked 2 times(May'24, Nov'25 — usually 6 marks)
How this question appeared in past papers:

(May'24 & Nov'25, 6 marks) What is a decision tree? Explain with a case study.

Answer

1. Definition

A Decision Tree is a supervised ML algorithm that models decisions as a TREE structure: each internal node = test on an attribute, each branch = outcome, each leaf = class label (or value).

2. Structure

                  [ Age < 30? ]
                    /     \
                  Yes      No
                  /         \
            [Income > 50K?]   [Approve loan]
              /        \
            Yes         No
            /            \
       [Approve]      [Reject]

3. How it works

  1. Start at root node.
  2. At each node, apply the test on an attribute.
  3. Follow the branch that matches the data point's value.
  4. Reach a leaf → return the class label.

4. Building a decision tree

  1. For each attribute, calculate how well it SPLITS the data (purity measures: Entropy, Information Gain, Gini Index).
  2. Choose the attribute with the best split.
  3. Recursively build sub-trees for each branch.
  4. Stop when leaves are pure or threshold reached.

5. Splitting criteria

  • Entropy: measure of impurity. H(S) = −Σ pᵢ · log₂(pᵢ).
  • Information Gain: reduction in entropy after split.
  • Gini Index: Σ pᵢ(1 − pᵢ); lower = purer.
  • Chi-square: statistical significance of split.

6. Famous algorithms

  • ID3 — uses Information Gain.
  • C4.5 — extension of ID3 with handling of continuous attributes.
  • CART (Classification and Regression Trees) — uses Gini Index.
  • CHAID — uses chi-square.

7. Case Study — Loan Approval

Data of 1000 customers with attributes (Age, Income, Credit Score, Employment) and target (Approve / Reject).

Decision tree learns:
   Credit Score > 700?
      Yes → Income > 50K?
              Yes → APPROVE
              No  → Employment stable?
                       Yes → APPROVE
                       No  → REJECT
      No → REJECT

A new customer's loan decision is taken by walking down the tree.

8. Advantages

  • Easy to understand and visualize.
  • Handles both numerical and categorical data.
  • Requires little data preparation.
  • Performs feature selection implicitly.
  • Can model non-linear relationships.

9. Disadvantages

  • Prone to overfitting (deep trees).
  • Small changes in data can produce very different trees.
  • Biased toward attributes with many values.
  • Pruning needed to avoid overfitting.

10. Applications

  • Loan approval, credit scoring.
  • Medical diagnosis.
  • Customer churn prediction.
  • Fraud detection.
  • Marketing target selection.

Key points to remember

  • Decision tree = tree of attribute tests; leaves are class labels.
  • Build using Entropy / Information Gain / Gini Index.
  • Algorithms: ID3, C4.5, CART, CHAID.
  • Pros: interpretable, handles mixed data.
  • Cons: overfits, unstable.
  • Pruning controls overfitting.

Q9 — Evaluation of Classification Models

Asked 1 time(Nov'23 — 6 marks)
How this question appeared in past papers:

(Nov'23, 6 marks) State different formulae for Evaluation of classification models.

Answer

1. Confusion Matrix

                  Predicted: Positive    Predicted: Negative
   Actual: Positive    TP                       FN
   Actual: Negative    FP                       TN
  • TP = True Positive (correctly predicted positive).
  • TN = True Negative.
  • FP = False Positive (type I error).
  • FN = False Negative (type II error).

2. Key Metrics

MetricFormulaMeaning
Accuracy(TP + TN) / (TP + TN + FP + FN)% correct overall
PrecisionTP / (TP + FP)Of predicted positives, % truly positive
Recall (Sensitivity)TP / (TP + FN)Of actual positives, % caught
SpecificityTN / (TN + FP)Of actual negatives, % caught
F1 Score2 · P · R / (P + R)Harmonic mean of precision and recall
FPRFP / (FP + TN)False positive rate
FNRFN / (FN + TP)False negative rate

3. ROC and AUC

  • ROC Curve: Plot of True Positive Rate vs False Positive Rate at different thresholds.
  • AUC (Area Under ROC Curve): single number from 0 to 1 measuring overall performance. Higher is better.

4. When to use which

  • Accuracy: when classes are balanced.
  • Precision: when false positives are costly (e.g., spam filtering).
  • Recall: when false negatives are costly (e.g., disease detection).
  • F1: when you need a balance and classes are imbalanced.
  • AUC: for threshold-independent evaluation.

Key points to remember

  • Confusion matrix: TP, FP, TN, FN.
  • Accuracy = (TP + TN) / Total.
  • Precision = TP / (TP + FP).
  • Recall = TP / (TP + FN).
  • F1 = 2 · P · R / (P + R).
  • AUC = area under ROC curve.

Q10 — Definitions: Frequent Itemset, Min Support, etc.

Asked 2 times(May'24, Nov'25 — usually 6 marks)
How this question appeared in past papers:

(May'24 & Nov'25, 6 marks) Define: Frequent item set, Minimum Support Count, Hierarchical Clustering, Regression.

Answer

1. Frequent Itemset

An itemset = a collection of one or more items. A frequent itemset is one whose SUPPORT is greater than or equal to the user-defined minimum support threshold.

Example: in supermarket transactions, {bread, butter} appearing in 30% of transactions is a frequent itemset if min_support = 20%.

2. Minimum Support Count

The Minimum Support Count = minimum NUMBER of transactions an itemset must appear in to be considered frequent. Specified by the user as a threshold.

Example: if there are 1000 transactions and min_support is 5%, the minimum support count = 50.

3. Hierarchical Clustering

A clustering algorithm that builds a HIERARCHY of clusters represented as a tree (dendrogram). Two types:

  • Agglomerative (bottom-up): start with each point as own cluster, iteratively merge.
  • Divisive (top-down): start with one big cluster, iteratively split.

No need to specify K in advance; cut the dendrogram at desired level.

4. Regression

A supervised learning technique that predicts a CONTINUOUS NUMERICAL output based on input features. Examples:

  • Linear Regression: y = β₀ + β₁x₁ + … + βnxn.
  • Polynomial Regression: includes powers of x.
  • Logistic Regression: for classification (despite the name).
  • Ridge / Lasso Regression: regularized versions.

Used for: predicting house prices, sales forecasting, temperature prediction.

Key points to remember

  • Frequent itemset = itemset with support ≥ min_support.
  • Min support count = the threshold number of transactions.
  • Hierarchical clustering = tree of clusters (agglomerative or divisive).
  • Regression = predict continuous numeric output.
